# 001.5 - Config/Middleware Loading Divergence

## Severity: 🟠 HIGH

## The Problem

```
Virtual Filesystem:  Content → esbuild → temp file → import
Local Filesystem:    file:// URL → native Deno import
```

Two completely different code paths.

## Differences

| Aspect | Virtual FS | Local FS |
|--------|-----------|----------|
| Transform | esbuild | Deno native |
| Error messages | esbuild format | Deno format |
| Relative imports | May break (`bundle: false`) | Work |
| Debug info | Temp file paths | Real file paths |

## Impact

- Syntax errors look different locally vs production
- Configs with relative imports work locally, break in production
- "Can't reproduce" bugs

## Fix

Use **one loading strategy** for all adapters:

```typescript
async function loadConfig(configPath: string, adapter: RuntimeAdapter) {
  // Always use esbuild OR always use native - not both
  return await loadConfigWithEsbuild(configPath, adapter);
}
```

## Value

| Before | After |
|--------|-------|
| Different errors local vs prod | Same errors everywhere |
| Imports work locally, break in prod | Consistent import behavior |
| Two code paths to maintain | One code path |
