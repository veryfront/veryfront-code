# 018.3: Event Listener Accumulation

## Severity: LOW-MEDIUM

## Location

**Files:** Component lifecycle and cleanup handlers throughout codebase

```typescript
// LEAK - addEventListener without matching removeEventListener
function setupComponent(element: HTMLElement): void {
  window.addEventListener("resize", handleResize);
  document.addEventListener("click", handleClick);
  // No cleanup on component unmount
}
```

## The Problem

Event listeners attached to global objects (window, document) hold references to handler functions and their closures. Without removal, they:
1. Continue executing after component removal
2. Prevent garbage collection of component state
3. Accumulate on repeated mount/unmount

### Growth Pattern

```
Component mount/unmount cycle:
├── Mount 1: +2 listeners (resize, click)
├── Unmount 1: listeners remain
├── Mount 2: +2 listeners (now 4 total)
├── Unmount 2: listeners remain
├── Mount 10: +2 listeners (now 20 total)
└── Each handler holds component closure
```

### Memory Reference Chain

```
Global listener registry
└── Handler function
    └── Closure scope
        └── Component state
            └── Rendered DOM (detached)
                └── All child references
```

## Proposed Fix

### Option A: AbortController Pattern (Recommended)

```typescript
function setupComponent(element: HTMLElement): () => void {
  const controller = new AbortController();
  const { signal } = controller;

  window.addEventListener("resize", handleResize, { signal });
  document.addEventListener("click", handleClick, { signal });

  // Return cleanup function
  return () => controller.abort();
}

// Usage in React
useEffect(() => {
  const cleanup = setupComponent(ref.current);
  return cleanup;
}, []);
```

### Option B: Explicit Remove

```typescript
function setupComponent(element: HTMLElement): () => void {
  const handleResize = () => { /* ... */ };
  const handleClick = () => { /* ... */ };

  window.addEventListener("resize", handleResize);
  document.addEventListener("click", handleClick);

  return () => {
    window.removeEventListener("resize", handleResize);
    document.removeEventListener("click", handleClick);
  };
}
```

### Option C: WeakRef for Handlers

```typescript
const handlers = new WeakMap<HTMLElement, Set<() => void>>();

function setupComponent(element: HTMLElement): void {
  const elementRef = new WeakRef(element);

  const handleResize = () => {
    const el = elementRef.deref();
    if (!el || !document.contains(el)) {
      // Element gone, remove listener
      window.removeEventListener("resize", handleResize);
      return;
    }
    // Handle resize
  };

  window.addEventListener("resize", handleResize);
}
```

## Affected Patterns

Look for:
```typescript
window.addEventListener(...)
document.addEventListener(...)
element.addEventListener(...) // on long-lived elements
```

Without corresponding `removeEventListener` or cleanup.

## Blast Radius

| Affected | Impact |
|----------|--------|
| SPA navigation | Listeners from previous pages |
| Modal/dialog components | Accumulate on open/close |
| Scroll/resize handlers | Memory + CPU waste |

## Migration Checklist

- [ ] Audit global event listener registrations
- [ ] Add cleanup to all component effects
- [ ] Prefer AbortController pattern
- [ ] Add ESLint rule for cleanup verification
