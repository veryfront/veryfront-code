# 018.2: WebSocket Timer Leaks

## Severity: MEDIUM

## Location

**File:** `src/server/ws/`

```typescript
// LEAK - Timer not cleared on close
function setupHeartbeat(ws: WebSocket): void {
  const interval = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
      ws.ping();
    }
  }, 30000);

  // Missing: clearInterval when ws closes
}
```

## The Problem

`setInterval` timers for WebSocket heartbeats are never cleared. Each connection accumulates timers that continue running after disconnect.

### Timer Accumulation

```
Connection 1: setInterval created → never cleared
Connection 2: setInterval created → never cleared
...
Connection 100: setInterval created → never cleared

Result: 100 intervals running, each firing every 30s
→ 3.3 timer callbacks per second
→ CPU waste checking dead sockets
```

### Memory Impact

```
Each timer holds reference to:
├── Callback function
├── Closure variables (ws reference)
├── Timer metadata
└── ~1-5KB per timer

1000 connections over time:
→ 1-5MB leaked in timer references
→ 1000 dead WebSocket objects not GC'd
```

## Proposed Fix

### Track and Clear Timers

```typescript
function setupHeartbeat(ws: WebSocket): void {
  const intervalId = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
      ws.ping();
    }
  }, 30000);

  ws.addEventListener("close", () => {
    clearInterval(intervalId);
  });

  ws.addEventListener("error", () => {
    clearInterval(intervalId);
  });
}
```

### Using AbortController Pattern

```typescript
function setupHeartbeat(ws: WebSocket, signal: AbortSignal): void {
  const intervalId = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
      ws.ping();
    }
  }, 30000);

  signal.addEventListener("abort", () => {
    clearInterval(intervalId);
  });
}

// Usage
const controller = new AbortController();
setupHeartbeat(ws, controller.signal);

ws.onclose = () => controller.abort();
```

### Connection Manager Pattern

```typescript
class WebSocketConnection {
  private ws: WebSocket;
  private timers: Set<number> = new Set();

  constructor(ws: WebSocket) {
    this.ws = ws;
    this.ws.addEventListener("close", () => this.cleanup());
  }

  addTimer(callback: () => void, interval: number): void {
    const id = setInterval(callback, interval);
    this.timers.add(id);
  }

  private cleanup(): void {
    for (const id of this.timers) {
      clearInterval(id);
    }
    this.timers.clear();
  }
}
```

## Blast Radius

| Affected | Impact |
|----------|--------|
| Dev server | Timer accumulation |
| Production WS | CPU waste, memory leak |
| GC pressure | Objects held by timers |
