# 004.0: RFC - Content-Addressed Dependency Tracking

## TL;DR

**Problem**: Cache keys use only the main file's content hash. When dependencies change, stale bundles are served.

**Solution**: Content-addressed caching with full dependency graph hashing. Cache key = hash(file + deps + config).

---

## The Problem

```typescript
// Current cache key generation - DEPENDENCIES NOT TRACKED
export function buildTransformCacheKey(
  filePath: string,
  contentHash: string,  // Only the main file's hash!
  ssr: boolean = false,
  studioEmbed: boolean = false,
): string {
  return `v${TRANSFORM_CACHE_VERSION}:${filePath}:${contentHash}:${ssrKey}`;
}
```

When `helper.ts` changes but `page.tsx` (which imports it) doesn't, the cache returns a stale bundle with the old `helper.ts` code.

**Real bug**: User updates a shared component. Some pages show the new component, others show the old version depending on cache state.

## Solution

### Content-Addressed Cache Keys

**New cache key formula:**
```
cacheKey = v{VERSION}:{projectId}:{filePath}:{contentHash}:{depsHash}:{configHash}:{target}
```

Where:
- `contentHash` = hash of the source file content
- `depsHash` = hash of all transitive dependency contents
- `configHash` = hash of transform-affecting configuration

### Core Algorithm

```typescript
interface DependencyGraph {
  // Map from file path to set of files it imports
  imports: Map<string, Set<string>>;
  // Map from file path to its content hash
  hashes: Map<string, string>;
}

async function computeDependencyHash(
  filePath: string,
  adapter: RuntimeAdapter,
  graph: DependencyGraph = { imports: new Map(), hashes: new Map() },
  visited: Set<string> = new Set(),
): Promise<string> {
  // Circular dependency check
  if (visited.has(filePath)) return graph.hashes.get(filePath) ?? "";
  visited.add(filePath);

  // Read and hash the file
  const content = await adapter.fs.readFile(filePath);
  const contentHash = await computeHash(content);
  graph.hashes.set(filePath, contentHash);

  // Parse imports
  const imports = await parseLocalImports(content, filePath, projectDir, adapter);
  const depPaths = new Set(imports.imports.map(i => i.absolutePath));
  graph.imports.set(filePath, depPaths);

  // Recursively hash dependencies
  const depHashes: string[] = [contentHash];
  for (const depPath of depPaths) {
    const depHash = await computeDependencyHash(depPath, adapter, graph, visited);
    depHashes.push(depHash);
  }

  // Stable sort for deterministic hash
  depHashes.sort();
  return computeHash(depHashes.join(":"));
}
```

### Phase 7: Inverse Dependency Index

For targeted cache invalidation on file changes:

```typescript
// Build inverse index: what files depend on this file?
function buildInverseIndex(graph: DependencyGraph): Map<string, Set<string>> {
  const inverse = new Map<string, Set<string>>();
  for (const [file, deps] of graph.imports) {
    for (const dep of deps) {
      if (!inverse.has(dep)) inverse.set(dep, new Set());
      inverse.get(dep)!.add(file);
    }
  }
  return inverse;
}

// Invalidate all caches affected by a file change
async function invalidateAffected(changedPath: string, inverse: Map<string, Set<string>>) {
  const affected = new Set<string>();
  const queue = [changedPath];
  while (queue.length > 0) {
    const path = queue.pop()!;
    if (affected.has(path)) continue;
    affected.add(path);
    const dependents = inverse.get(path) ?? new Set();
    for (const dep of dependents) queue.push(dep);
  }
  // Invalidate cache entries for all affected files
  for (const file of affected) {
    await invalidateTransformCache(file);
  }
}
```

## Risk Considerations

### Performance Impact

Computing dependency hashes requires reading all transitive dependencies. Mitigations:
1. **Lazy computation**: Only compute when cache miss
2. **Memoization**: Cache dependency graph per render
3. **Parallel reads**: Read all deps in parallel
4. **Incremental updates**: Update graph on file change, not full rebuild

### Cache Invalidation Storm

When a widely-used file changes (e.g., `utils.ts`), many cache entries become invalid. Mitigations:
1. **Lazy invalidation**: Don't proactively clear, let cache miss on next request
2. **Warm-up script**: Pre-warm caches after deployment
3. **Graceful degradation**: Serve stale while recomputing (with headers)

### Circular Dependencies

Import cycles create infinite loops in dependency hash computation. Mitigation:
1. **Visited set**: Track visited files, return cached hash on re-visit
2. **Cycle detection**: Warn on circular dependencies in dev mode
