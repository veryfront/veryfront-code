# 007.0: RFC - Config Normalization Layer

## Target State

```
User Config          Normalize          Internal Config
(liberal)               │               (strict, frozen)
───────────             │               ────────────────

"app"                   │
"app-router"   ────────▶├──────────────▶ router: "app" | "pages"
"pages-router"          │

true                    │
{ origin: "*" } ───────▶├──────────────▶ NormalizedCorsConfig
["*.com"]               │                (strict interface)

"path"                  │
false          ────────▶├──────────────▶ { enabled: bool, path: string | null }
undefined               │

                        ▼
               ┌─────────────────────────────────┐
               │ Deep clone per request          │
               │ No shared mutable state         │
               │ Object.freeze() on all configs  │
               └─────────────────────────────────┘
```

**Key Invariant**: Config normalized ONCE at boundary. Internal code uses strict types.

---

## TL;DR

**Problem**: Config values accept multiple formats (e.g., `"app"` vs `"app-router"`) with scattered normalization logic = inconsistent behavior and multi-tenancy risks from shared DEFAULT_CONFIG mutation.

**Solution**: Single normalization boundary after validation. Accept liberal input, emit strict internal types.

## The Problem

```typescript
// This scattered pattern appears throughout the codebase
const pref = cfg?.generate?.preferredRouter ?? cfg?.router;
if (pref === "app-router" || pref === "pages-router") return pref;
// BUT route-discovery.ts only handles "app" | "pages"
// FORMAT MISMATCH - behavior differs based on which path is taken
```

**Real bugs**:
1. `generate.preferredRouter: "app-router"` doesn't match `router: "app"` comparison
2. CORS schema validates `origin?: string` but runtime accepts `string[] | Function`
3. `DEFAULT_CONFIG` is shared and potentially mutated by the mergeConfigs spread pattern

## Multi-Tenancy Risk: Shared DEFAULT_CONFIG

### Current Code (loader.ts:26-72, 111-162)

```typescript
// Line 26: This is a MODULE-LEVEL constant object
const DEFAULT_CONFIG: Partial<VeryfrontConfig> = {
  title: "Veryfront App",
  // ...nested objects...
  cache: {
    dir: DEFAULT_CACHE_DIR,
    render: {
      type: "memory",
      // ...
    },
  },
  resolve: {
    importMap: getDefaultImportMapForConfig(), // FUNCTION CALL - executed once at module load
  },
  // ...
};

// Line 111: mergeConfigs uses shallow spread
function mergeConfigs(userConfig: Partial<VeryfrontConfig>): VeryfrontConfig {
  const merged = {
    ...DEFAULT_CONFIG,        // Spreads the shared object
    ...userConfig,
    cache: {
      ...DEFAULT_CONFIG.cache,  // Nested spreads from shared object
      ...userConfig.cache,
    },
    // ...
  };
```

### The Risk

While the current spread pattern creates new objects, there are subtle issues:

1. **`getDefaultImportMapForConfig()` called at module load** - not per-request
2. **If any code mutates nested properties** of the merged config, and that config is cached, the mutation persists
3. **The `configCacheByProject` Map stores merged configs** - if the merge is shallow, nested object references could be shared

### Scenario: Import Map Contamination

```
Timeline:
=========
t0: Module loads, DEFAULT_CONFIG.resolve.importMap created ONCE

t1: Project A request - getConfig returns merged config with importMap reference
    → Code adds custom import: merged.resolve.importMap.imports["custom"] = "..."
    → IF this mutates the cached config, it affects the shared reference

t2: Project B request - getConfig returns cached config
    → Project B's config now contains Project A's custom imports!
```

## Solution: Normalized Config Layer

### Architecture

```
User Input          →  Validation  →  Normalization  →  Internal Config
(types.ts)             (schema.ts)    (normalize.ts)     (normalized-types.ts)
                                           ↓
                                    DEEP COPY + FORMAT CONVERSION
                                           ↓
                                    Single canonical format
```

### Normalized Types (normalized-types.ts)

```typescript
/**
 * Internal normalized config types.
 * All code outside src/config/ should use these types.
 * IMMUTABLE after creation - never mutate a NormalizedConfig.
 */

export type RouterType = "app" | "pages";

export interface NormalizedCorsConfig {
  readonly enabled: boolean;
  readonly origin: string | readonly string[] | ((origin: string) => boolean | string);
  readonly credentials: boolean;
  readonly methods: readonly string[];
  readonly allowedHeaders: readonly string[];
  readonly exposedHeaders: readonly string[];
  readonly maxAge: number;
}

export interface NormalizedLayoutConfig {
  readonly enabled: boolean;
  readonly path: string | null;
}

export interface NormalizedCacheConfig {
  readonly dir: string;
  readonly bundleManifest: {
    readonly enabled: boolean;
    readonly type: "redis" | "kv" | "memory";
    readonly redisUrl: string | null;
    readonly keyPrefix: string;
    readonly ttl: number;
  };
  readonly render: {
    readonly enabled: boolean;
    readonly type: "memory" | "filesystem" | "kv" | "redis";
    readonly ttl: number;
    readonly maxEntries: number;
  };
}

export interface NormalizedConfig {
  readonly projectSlug: string | null;
  readonly title: string;
  readonly description: string;
  readonly router: RouterType;
  readonly layout: NormalizedLayoutConfig;
  readonly app: NormalizedLayoutConfig;
  readonly cors: NormalizedCorsConfig;
  readonly cache: NormalizedCacheConfig;
  // ... all fields are readonly
}
```

### Normalizer Functions (normalize.ts)

```typescript
import type { VeryfrontConfig } from "./types.ts";
import type { NormalizedConfig, RouterType } from "./normalized-types.ts";
import { structuredClone } from "#veryfront/platform/compat/clone.ts";

// DEEP CLONE defaults to prevent shared reference issues
const BASE_DEFAULTS = Object.freeze({
  title: "Veryfront App",
  description: "Built with Veryfront",
  router: "pages" as RouterType,
  // ... all base values
});

function createFreshDefaults(): typeof BASE_DEFAULTS {
  return structuredClone(BASE_DEFAULTS);
}

/**
 * Normalize router format.
 * Accepts: "app", "pages", "app-router", "pages-router"
 * Emits: "app" | "pages"
 */
function normalizeRouter(
  router?: "app" | "pages",
  preferredRouter?: "app-router" | "pages-router"
): RouterType {
  // preferredRouter takes precedence (more specific)
  if (preferredRouter === "app-router") return "app";
  if (preferredRouter === "pages-router") return "pages";
  return router ?? "pages";
}

/**
 * Normalize CORS configuration.
 * Accepts: boolean | { origin?: string }
 * Emits: Full immutable CORSConfig
 */
function normalizeCors(
  cors?: boolean | { origin?: string }
): NormalizedCorsConfig {
  const defaults: NormalizedCorsConfig = Object.freeze({
    enabled: true,
    origin: "*",
    credentials: false,
    methods: Object.freeze(["GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"]),
    allowedHeaders: Object.freeze(["Content-Type", "Authorization"]),
    exposedHeaders: Object.freeze([]),
    maxAge: 86400,
  });

  if (cors === undefined || cors === true) return defaults;
  if (cors === false) return Object.freeze({ ...defaults, enabled: false });

  return Object.freeze({
    ...defaults,
    enabled: true,
    origin: cors.origin ?? "*",
  });
}

/**
 * Normalize layout configuration.
 * Accepts: string | false | undefined
 * Emits: { enabled: boolean, path: string | null }
 */
function normalizeLayout(
  layout?: string | false
): NormalizedLayoutConfig {
  if (layout === false) return Object.freeze({ enabled: false, path: null });
  if (typeof layout === "string") return Object.freeze({ enabled: true, path: layout });
  return Object.freeze({ enabled: true, path: null }); // undefined = auto-discover
}

/**
 * Main normalization entry point.
 * Called ONCE after validation, result is cached.
 * Returns a deeply frozen, immutable config.
 */
export function normalizeConfig(input: VeryfrontConfig): NormalizedConfig {
  const normalized: NormalizedConfig = {
    projectSlug: input.projectSlug ?? null,
    title: input.title ?? BASE_DEFAULTS.title,
    description: input.description ?? BASE_DEFAULTS.description,
    router: normalizeRouter(input.router, input.generate?.preferredRouter),
    layout: normalizeLayout(input.layout),
    app: normalizeLayout(input.app),
    cors: normalizeCors(input.security?.cors),
    cache: normalizeCacheConfig(input.cache),
    // ... normalize all fields
  };

  // Deep freeze to prevent any mutation
  return deepFreeze(normalized);
}

function deepFreeze<T>(obj: T): T {
  if (obj === null || typeof obj !== "object") return obj;

  Object.freeze(obj);
  for (const key of Object.keys(obj)) {
    const value = (obj as Record<string, unknown>)[key];
    if (value !== null && typeof value === "object" && !Object.isFrozen(value)) {
      deepFreeze(value);
    }
  }
  return obj;
}
```

### Integration with Loader (loader.ts)

```typescript
import { normalizeConfig } from "./normalize.ts";
import type { NormalizedConfig } from "./normalized-types.ts";

// Cache stores NORMALIZED configs (immutable)
const configCacheByProject = new Map<string, {
  revision: number;
  config: NormalizedConfig;  // Changed from VeryfrontConfig
}>();

function validateAndCacheConfig(
  userConfig: unknown,
  cacheKey: string
): NormalizedConfig {
  // 1. Validate shape (throws on invalid)
  validateConfigShape(userConfig);
  validateCorsConfig(userConfig);

  // 2. Normalize to internal format (creates fresh, frozen object)
  const normalized = normalizeConfig(userConfig as VeryfrontConfig);

  // 3. Cache normalized config
  configCacheByProject.set(cacheKey, {
    revision: cacheRevision,
    config: normalized,  // Immutable - safe to cache
  });

  return normalized;
}
```
