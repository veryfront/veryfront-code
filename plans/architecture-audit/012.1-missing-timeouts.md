## Why It's Bad for Multi-Tenancy

### Resource Exhaustion Scenario

When external services slow down or hang:

```
Timeline:
=========
t0: ProjectA request starts, calls GitHub API
    → fetch() hangs waiting for response
    → Worker thread blocked

t1: ProjectB request starts, calls GitHub API
    → fetch() hangs waiting for response
    → Another worker thread blocked

t2-t99: More requests arrive
    → All worker threads become blocked
    → Request queue builds up

t100: Connection pool exhausted
    → ALL projects start timing out at load balancer level
    → User-visible errors across the entire platform

Root Cause: One slow external service (GitHub, esm.sh, GCS)
            blocks all tenants because no fetch timeouts.
```

### Real-World Scenarios

| Scenario | Impact |
|----------|--------|
| GitHub API rate limit (slow 403 responses) | All GitHub-backed projects hang |
| esm.sh CDN slowdown | All SSR with npm packages hang |
| Veryfront API overload | All projects hang |
| GCS regional outage | All blob storage operations hang |

## Solution

See **[012.0 - HTTP Clients RFC](./012.0-http-clients-rfc.md)** for the unified client solution.

### Quick Fix (Before Full Migration)

For each affected client, add timeout handling:

```typescript
// Pattern to add to each client
const DEFAULT_TIMEOUT_MS = 30000;

async function fetchWithTimeout(
  url: string,
  options: RequestInit & { timeout?: number } = {},
): Promise<Response> {
  const { timeout = DEFAULT_TIMEOUT_MS, ...fetchOptions } = options;

  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeout);

  try {
    const response = await fetch(url, {
      ...fetchOptions,
      signal: controller.signal,
    });
    return response;
  } catch (error) {
    if (error instanceof Error && error.name === "AbortError") {
      throw new Error(`Request timed out after ${timeout}ms: ${url}`);
    }
    throw error;
  } finally {
    clearTimeout(timeoutId);
  }
}
```

| Client | Recommended Timeout | Rationale |
|--------|---------------------|-----------|
| Veryfront API | 30s | API calls can be slow for large file ops |
| Token Storage | 10s | Simple key-value operations |
| GitHub API | 30s | Tree operations can be large |
| Domain Lookup | 5s | Simple lookup, critical path |
| GCS Storage | 60s | Large file uploads/downloads |
