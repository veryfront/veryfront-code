# 005.2 - SSG getAllPages() Silently Misses App Router Pages

## Severity: CRITICAL

## Location

**File:** `/Users/mattboon/Sites/veryfront-renderer/src/rendering/page-resolution/page-resolver.ts`
**Lines:** 89-113

## The Problem

```typescript
// Lines 89-113 - ONLY scans pages/ directory!
async getAllPages(): Promise<string[]> {
  const pages = new Set<string>();
  const pagesDirName = this.config.directories?.pages ?? "pages";

  // Line 93-100: Only checks pages/ directory
  const pagesDir = join(this.projectDir, pagesDirName);
  if (await this.adapter.fs.exists(pagesDir)) {
    for await (const entry of this.adapter.fs.readDir(pagesDir)) {
      if (entry.isFile && isPageFile(entry.name)) {
        pages.add(fileToSlug(entry.name));
      }
    }
  }

  // Line 102-107: Also checks project root (legacy support)
  for await (const entry of this.adapter.fs.readDir(this.projectDir)) {
    if (!entry.isFile || !isPageFile(entry.name) || entry.name.includes("config")) {
      continue;
    }
    pages.add(fileToSlug(entry.name));
  }

  // MISSING: No scan of app/ directory!
  // App Router pages are NEVER discovered!

  return Array.from(pages);
}
```

## Why It's Bad for Multi-Tenancy

This bug affects ALL projects using App Router, regardless of multi-tenancy. However, in a multi-tenant environment:

1. **Silent failures are harder to debug**: When SSG skips pages, there's no error - pages just don't get pre-rendered
2. **Inconsistent behavior**: Some tenants (Pages Router) get full SSG, others (App Router) get partial/no SSG
3. **Resource waste**: App Router projects may fall back to SSR for all pages, increasing server load

### Scope of Missing Code

The function checks:
- `pages/*.tsx` files
- `{projectRoot}/*.tsx` files (legacy)

The function **NEVER** checks:
- `app/page.tsx` (root route)
- `app/*/page.tsx` (nested routes)
- `app/**/page.tsx` (deeply nested routes)

## Failure Scenario

**Scenario: App Router project with SSG enabled**

```
Project structure:
  app/
    page.tsx           ← Home page, MISSED
    about/
      page.tsx         ← About page, MISSED
    blog/
      page.tsx         ← Blog index, MISSED
      [slug]/
        page.tsx       ← Blog posts (dynamic, expected to need getStaticPaths)
  veryfront.config.ts  ← ssg: { enabled: true }

What happens:
  1. SSG build starts
  2. getAllPages() called
  3. Scans pages/ directory → empty (doesn't exist)
  4. Scans project root → only finds config file
  5. Returns: []  ← EMPTY!
  6. SSG builds zero pages
  7. All routes fall back to SSR
  8. No error, no warning - silent failure!

User impact:
  - Slower page loads (SSR vs static)
  - Higher server costs
  - No pre-rendered HTML for SEO crawlers
```

## Contrast with build-routes.ts

Interestingly, `build-routes.ts` correctly handles both routers:


But this is only used for certain build scenarios, not the main `getAllPages()` used by SSG.

## Solution

See **[005.0 - Router Unification RFC](./005.0-router-unification-rfc.md)** for the complete solution.

### Immediate Fix

```typescript
async getAllPages(): Promise<string[]> {
  const pages = new Set<string>();
  const pagesDirName = this.config.directories?.pages ?? "pages";
  const appDirName = this.config.directories?.app ?? "app";

  // Scan pages/ directory (existing)
  const pagesDir = join(this.projectDir, pagesDirName);
  if (await this.adapter.fs.exists(pagesDir)) {
    for await (const entry of this.adapter.fs.readDir(pagesDir)) {
      if (entry.isFile && isPageFile(entry.name)) {
        pages.add(fileToSlug(entry.name));
      }
    }
  }

  // NEW: Scan app/ directory for page.tsx files
  const appDir = join(this.projectDir, appDirName);
  if (await this.adapter.fs.exists(appDir)) {
    await this.scanAppDirectory(appDir, "", pages);
  }

  // Check project root (legacy support)
  for await (const entry of this.adapter.fs.readDir(this.projectDir)) {
    if (!entry.isFile || !isPageFile(entry.name) || entry.name.includes("config")) {
      continue;
    }
    pages.add(fileToSlug(entry.name));
  }

  return Array.from(pages);
}

private async scanAppDirectory(dir: string, prefix: string, pages: Set<string>): Promise<void> {
  try {
    for await (const entry of this.adapter.fs.readDir(dir)) {
      // Skip hidden, private, route groups, parallel routes
      if (entry.name.startsWith(".") || entry.name.startsWith("_")) continue;
      if (entry.name.startsWith("(") || entry.name.startsWith("@")) continue;

      const fullPath = join(dir, entry.name);

      if (entry.isDirectory) {
        // Skip dynamic segments for static discovery
        // (they need getStaticPaths)
        if (entry.name.startsWith("[")) continue;

        const newPrefix = prefix ? `${prefix}/${entry.name}` : entry.name;
        await this.scanAppDirectory(fullPath, newPrefix, pages);
        continue;
      }

      // Only page.tsx files create routes in App Router
      if (/^page\.(tsx?|jsx?|mdx?)$/.test(entry.name)) {
        const slug = prefix || "/";
        pages.add(slug);
      }
    }
  } catch {
    // Directory read failed, skip
  }
}
```

### Proper Fix (from RFC)

Use the unified RouteRegistry which discovers all routes at once:

```typescript
async getAllPages(): Promise<string[]> {
  const registry = await getRouteRegistry(
    this.projectId,
    this.projectDir,
    this.adapter,
    this.config
  );

  // Registry already has all routes from both app/ and pages/
  return registry.routes
    .filter(route => !route.isDynamic)
    .map(route => route.pattern);
}
```

## Testing

```typescript
// tests/ssg-app-router.test.ts
Deno.test("getAllPages discovers App Router pages", async () => {
  const tmpDir = await Deno.makeTempDir();

  // Create App Router structure
  await Deno.mkdir(join(tmpDir, "app", "about"), { recursive: true });
  await Deno.mkdir(join(tmpDir, "app", "blog"), { recursive: true });

  await Deno.writeTextFile(join(tmpDir, "app", "page.tsx"), "export default () => <h1>Home</h1>");
  await Deno.writeTextFile(join(tmpDir, "app", "about", "page.tsx"), "export default () => <h1>About</h1>");
  await Deno.writeTextFile(join(tmpDir, "app", "blog", "page.tsx"), "export default () => <h1>Blog</h1>");

  const resolver = new PageResolver({
    projectDir: tmpDir,
    config: { router: "app" },
    adapter: createLocalAdapter(tmpDir),
  });

  const pages = await resolver.getAllPages();

  // Should find all App Router pages
  assertEquals(pages.sort(), ["/", "/about", "/blog"].sort());

  await Deno.remove(tmpDir, { recursive: true });
});

Deno.test("getAllPages discovers both App Router and Pages Router", async () => {

  // Create mixed structure

  const pages = await resolver.getAllPages();

  // Should find routes from both routers
  assertEquals(pages.sort(), ["/", "/dashboard", "/legacy"].sort());

  await Deno.remove(tmpDir, { recursive: true });
});
```
