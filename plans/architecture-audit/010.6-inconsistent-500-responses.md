## The Problem

500 error responses are handled inconsistently across the codebase:

| Handler | Format | Dev Mode | Custom Page | Security Headers |
|---------|--------|----------|-------------|------------------|
| SSRHandler | HTML (ErrorOverlay/ErrorPages) | Yes | Yes | Yes |
| UniversalHandler | Plain text | No | No | No |
| Deno adapter | Plain text | No | No | No |
| Bun adapter | Plain text | No | No | No |
| RSC handler | Status only | No | Partial | Varies |
| API routes | JSON | No | No | No |

This causes:
1. **Inconsistent UX**: Same error shows different content depending on code path
2. **Missing security headers**: Some responses bypass ResponseBuilder
3. **No custom error pages**: Only SSRHandler tries user's custom error page
4. **Dev mode confusion**: Error overlays only in some handlers

## Why It's Bad for Multi-Tenancy

### Scenario 1: Different Error Experiences

```
Project A has custom _error.tsx page with branded error UI
Project B uses default error pages

User request to Project A:
- SSRHandler path: Shows branded custom error page (correct!)

The same project gets different error experiences depending on
which code path the error occurs in.
```

## Solution

See **[010.0 - Error Handling RFC](./010.0-error-handling-rfc.md)** for the complete solution.

**Summary: Centralized Error Response Builder**

```typescript
// src/server/utils/error-response.ts
export interface ErrorResponseOptions {
  statusCode: number;
  error?: Error;
  pathname?: string;
  isDev?: boolean;
  isPreview?: boolean;
  nonce: string;
  securityConfig?: SecurityConfig;
  corsConfig?: CORSConfig;
  request?: Request;  // For CORS origin checking
}

/**
 * Build consistent error responses regardless of which handler catches the error.
 */
export async function buildErrorResponse(
  options: ErrorResponseOptions,
  ctx?: HandlerContext,
): Promise<Response> {
  const { statusCode, error, pathname, isDev, isPreview, nonce, securityConfig, corsConfig, request } = options;

  // 1. Try custom error page (if context available)
  if (ctx && statusCode >= 400) {
    const builder = createResponseBuilder(nonce);
    const customResponse = await tryErrorPageFallback(request!, ctx, builder, {
      statusCode,
      error,
      pathname,
    });
    if (customResponse) {
      return addSecurityAndCors(customResponse, securityConfig, corsConfig, request);
    }
  }

  // 2. Dev/Preview: Show detailed error overlay
  if ((isDev || isPreview) && error && statusCode >= 500) {
    const body = ErrorOverlay.createHTML({ error, type: "runtime", nonce });
    return createSecureHtmlResponse(body, statusCode, nonce, securityConfig, corsConfig, request);
  }

  // 3. Production: Styled error page
  const body = statusCode === 404
    ? ErrorPages.notFound(pathname)
    : statusCode === 503
    ? ErrorPages.memoryPressure()
    : ErrorPages.serverError();

  return createSecureHtmlResponse(body, statusCode, nonce, securityConfig, corsConfig, request);
}

function createSecureHtmlResponse(
  body: string,
  status: number,
  nonce: string,
  securityConfig?: SecurityConfig,
  corsConfig?: CORSConfig,
  request?: Request,
): Response {
  const headers = new Headers({
    "Content-Type": "text/html; charset=utf-8",
    "Cache-Control": "no-cache, no-store, must-revalidate",
  });

  // Always add security headers
  if (securityConfig) {
    addSecurityHeaders(headers, securityConfig, nonce);
  }

  // Always add CORS if configured
  if (corsConfig && request) {
    addCorsHeaders(headers, corsConfig, request);
  }

  return new Response(body, { status, headers });
}
```

## Quick Fix

If full implementation is delayed, at least standardize the format:

```typescript
// Create helper used by all handlers
function createBasicErrorResponse(status: number, isDev: boolean, error?: Error): Response {
  const headers = new Headers({
    "Content-Type": "text/html; charset=utf-8",
    "Cache-Control": "no-cache",
    "X-Content-Type-Options": "nosniff",  // Minimal security
  });

  let body: string;
  if (isDev && error) {
    body = `<!DOCTYPE html><html><body><h1>Error ${status}</h1><pre>${error.stack}</pre></body></html>`;
  } else {
    body = ErrorPages.serverError();
  }

  return new Response(body, { status, headers });
}
```
