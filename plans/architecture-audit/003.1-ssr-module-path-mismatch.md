# 003.1 - SSR Module Cache: Cross-Pod Path Mismatch

## Severity: CRITICAL - Production 500 Errors

## The Problem

SSR modules are cached in Redis with absolute `file://` paths that reference temp directories specific to the pod that created them. When another pod retrieves the cached transform, the paths don't exist locally, causing "Module not found" errors.

### Location: `src/modules/react-loader/ssr-module-loader/loader.ts:493-540`

```typescript
// Redis cache hit path (lines 493-540)
if (isSSRDistributedCacheEnabled()) {
  const redisCode = await getFromRedis(contentCacheKey);
  if (redisCode) {
    // CRITICAL: redisCode contains file:// paths from Pod A
    // Example: import "file:///tmp/veryfront-abc123/node_modules/react.mjs"
    // But /tmp/veryfront-abc123/ only exists on Pod A, not Pod B

    // PARTIAL MITIGATION: HTTP bundle path check (lines 500-514)
    const bundlePaths = extractHttpBundlePaths(redisCode);
    if (bundlePaths.length > 0) {
      const cacheDir = getHttpBundleCacheDir();
      const failed = await ensureHttpBundlesExist(bundlePaths, cacheDir);
      if (failed.length > 0) {
        // Falls through to re-transform
        httpBundlesOk = false;
      }
    }

    // BUG: Only checks HTTP bundle paths, NOT local import paths!
    // If httpBundlesOk is true, code with stale local file:// paths is used

    if (httpBundlesOk) {
      // Writes redisCode (with potentially stale paths) to local temp file
      const tempPath = await this.getTempPath(filePath, transformedHash);
      await this.fs.writeTextFile(tempPath, redisCode);
      // ...
    }
  }
}
```

## Cache HIT vs MISS Divergence

| Aspect | Cache HIT (Redis) | Cache MISS |
|--------|-------------------|------------|
| HTTP bundle validation | Partial - only `veryfront-http-bundle` paths | Full - bundles created locally |
| Local import validation | **NONE** | Full - imports resolved locally |
| Cross-project import validation | **NONE** | Full - imports fetched and transformed |
| Dependency graph freshness | **NONE** | Full - all deps re-parsed |
| Path portability | Absolute paths from source pod | Absolute paths match local pod |

## Code Evidence

### 1. HTTP Bundle Extraction (Only Partial)

```typescript
const FILE_PATH_PATTERN = /file:\/\/([^"'\s]+)/g;

// Local imports like file:///tmp/veryfront-abc/lib/utils.mjs are NOT checked
```

### 2. What extractHttpBundlePaths Matches

```typescript
// http-cache.ts:590-610
export function extractHttpBundlePaths(code: string): string[] {
  const pattern = /file:\/\/[^"'\s]+veryfront-http-bundle[^"'\s]+\.mjs/g;
  //                            ^^^^^^^^^^^^^^^^^^^^
  //                            Only matches HTTP bundles!
  return [...code.matchAll(pattern)].map((m) => m[0].slice(7)); // Remove file://
}
```

### 3. What's NOT Validated on Cache Hit

```typescript
// A cached transform might contain:
import { useState } from "file:///tmp/veryfront-abc123/http-bundle/react.mjs"; // VALIDATED
import { utils } from "file:///tmp/veryfront-abc123/project/lib/utils.mjs";    // NOT VALIDATED!
import Component from "file:///tmp/veryfront-abc123/project/components/Foo.mjs"; // NOT VALIDATED!
```

## Failure Scenario

1. **Pod A** transforms `pages/index.tsx` which imports `lib/utils.ts`
2. Transformed code contains: `import { helper } from "file:///tmp/veryfront-A/lib/utils.mjs"`
3. Transform stored in Redis with key: `v10:proj123:pages/index.tsx:{hash}`
4. **Pod B** receives request for same page
5. Pod B gets Redis cache hit
6. Pod B writes cached code to `/tmp/veryfront-B/pages/index.mjs`
7. Runtime import: `import "file:///tmp/veryfront-A/lib/utils.mjs"` **FAILS**
   - Path `/tmp/veryfront-A/` doesn't exist on Pod B
   - Error: "Module not found: file:///tmp/veryfront-A/lib/utils.mjs"

## When This Breaks

- **Pod restarts**: New pod gets different temp directory
- **Horizontal scaling**: New pods don't have old pod's temp files
- **Rolling deployments**: Old pods' temp paths become invalid
- **Memory pressure**: LRU eviction clears local modules but Redis keeps stale transforms

## Root Cause Analysis

### Why HTTP Bundles Are Validated But Not Local Imports

```typescript
// The code has TWO separate validation systems that don't share logic:

// 1. HTTP bundle validation (implemented correctly)
const bundlePaths = extractHttpBundlePaths(redisCode);  // Only HTTP bundles
const failed = await ensureHttpBundlesExist(bundlePaths, cacheDir);

// 2. Local import validation (NOT IMPLEMENTED for Redis hits)
// The miss path does full local import resolution:
const parseResult = await parseLocalImports(code, filePath, projectDir, adapter);
// But the hit path skips this entirely!
```

## The Fix

### Option 1: Validate ALL file:// Paths (Quick Fix)

```typescript
// Add to loader.ts Redis cache hit path:
function extractAllFilePaths(code: string): string[] {
  // Match ALL file:// paths, not just HTTP bundles
  const pattern = /file:\/\/([^"'\s]+\.mjs)/g;
  return [...code.matchAll(pattern)].map(m => m[1]);
}

// Then validate each path exists locally before using cached code
const allPaths = extractAllFilePaths(redisCode);
const missingPaths = [];
for (const path of allPaths) {
  if (!await this.fs.exists(path)) {
    missingPaths.push(path);
  }
}
if (missingPaths.length > 0) {
  // Invalidate and re-transform
}
```

### Option 2: Store Relative Paths in Distributed Cache (Correct Fix)

```typescript
// Before storing in Redis, convert absolute to relative:
function toPortablePaths(code: string, baseTmpDir: string): string {
  return code.replace(
    new RegExp(`file://${escapeRegex(baseTmpDir)}`, 'g'),
    'file://${TMP_DIR}'  // Placeholder
  );
}

// On cache hit, resolve placeholders to local paths:
function fromPortablePaths(code: string, localTmpDir: string): string {
  return code.replace(
    /file:\/\/\$\{TMP_DIR\}/g,
    `file://${localTmpDir}`
  );
}
```

## Solution

See **[003.0 - Cache Consistency RFC](./003.0-cache-consistency-rfc.md)** for the complete solution.

The key insight is that distributed cache should store **portable representations** that can be resolved on any pod, not absolute paths tied to the source pod's filesystem.
