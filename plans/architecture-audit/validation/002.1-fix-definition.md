# Fix Definition: 002.1 Head Collector Leakage

## Problem Statement

The head collector uses module-level global state causing concurrent SSR requests to overwrite each other's metadata. This results in wrong SEO titles, descriptions, and OG tags being served to users.

## Scope

### In Scope
- Modify `src/react/head-collector.ts` to use AsyncLocalStorage
- Update `src/rendering/orchestrator/ssr-orchestrator.ts` caller
- Update `src/react/head-collector.test.ts` unit tests
- Add reproduction test for concurrent scenarios

### Out of Scope
- Other global state issues (002.2, 002.3, etc.)
- Performance optimization
- API surface changes beyond deprecation

## Dependencies

### Blocks
- None - this is a foundational fix

### Blocked By

## Implementation Plan

### Step 1: Update head-collector.ts

```typescript
// Add AsyncLocalStorage
import { AsyncLocalStorage } from "node:async_hooks";

const headStorage = new AsyncLocalStorage<CollectedHead>();

// New API
export function runWithHeadCollector<T>(
  fn: () => T | Promise<T>
): Promise<{ result: T; head: CollectedHead }>;

// Update collectHead to use storage
export function collectHead(data: Partial<CollectedHead>): void {
  const collected = headStorage.getStore();
  if (!collected) return; // Not in SSR context
  // ... merge data
}

// Deprecate old API
/** @deprecated Use runWithHeadCollector instead */
export function resetHeadCollector(): void;

/** @deprecated Use runWithHeadCollector instead */
export function flushHeadCollector(): CollectedHead;
```

### Step 2: Update ssr-orchestrator.ts

```typescript
// Before
resetHeadCollector();
const { html, stream } = await this.config.ssrRenderer.renderToHTML(...);
const collectedHead = flushHeadCollector();

// After
const { result: renderResult, head: collectedHead } = await runWithHeadCollector(
  () => this.config.ssrRenderer.renderToHTML(validatedElement, options)
);
const { html, stream } = renderResult;
```
