## Why It's Bad for Multi-Tenancy

### Problem 1: detectAppRouter() Called Multiple Times

Each call to `detectAppRouter()` is a potential cache hit/miss. In a multi-tenant environment with the global cache issue (see 005.1), this creates multiple opportunities for incorrect router detection:

```
Request lifecycle:
  1. PageResolver.resolvePage() → detectAppRouter()
  2. LayoutCollector.collectNestedLayouts() → detectAppRouter()  // Could differ!
  3. LayoutApplicator.applyLayouts() → detectAppRouter()         // Could differ!
```

If the cache is invalidated or corrupted between calls, the same request could have inconsistent router detection results.

### Problem 2: Veryfront API Path Skips Nested Layouts

The `collectAPILayoutConfiguration()` path completely skips nested layout discovery:

```typescript
// This path only checks:
// - config.layout
// - components/layout.{ext}

// It NEVER discovers:
// - app/dashboard/layout.tsx
// - app/settings/layout.tsx
// - pages/_app.tsx (for Pages Router API projects)
```

This is documented in Chapter 001 as the root cause of "layouts work locally, break in production".

### Problem 3: Different Wrapping Behavior

| Feature | Pages Router | App Router |
|---------|-------------|------------|
| Root wrapper | `_app.tsx` | None (use `layout.tsx`) |
| Error handling | Manual | `error.tsx` auto-wrapped |
| Loading states | Manual | `loading.tsx` auto-wrapped |
| Suspense boundary | Manual | Auto-created if `loading.tsx` exists |

This means the React component tree structure differs based on router type, affecting hydration and client-side behavior.

## Failure Scenario

**Scenario: Concurrent requests with cache race**

```
Timeline:
=========
t0: Request A (project-alpha, App Router) starts
    → PageResolver: detectAppRouter() returns true
    → Cache: { "alpha-dir": true }

t1: Request B (project-beta, Pages Router) starts
    → PageResolver: detectAppRouter() returns false
    → Cache: { "alpha-dir": true, "beta-dir": false }

t2: Cache invalidated globally (file watcher, deploy, etc.)

t3: Request A continues, LayoutCollector
    → detectAppRouter() called again
    → Cache miss, re-scans filesystem
    → BUT: Veryfront API adapter, takes different path!
    → collectAPILayoutConfiguration() called
    → Returns only root layout, misses dashboard/layout.tsx

t4: Request A renders
    → User sees page WITHOUT dashboard layout
    → "Works locally, breaks in production"
```

| Affected | Impact |
|----------|--------|

## Solution

See **[005.0 - Router Unification RFC](./005.0-router-unification-rfc.md)** for the complete solution.

### Pre-compute Layouts During Route Discovery

```typescript
interface UnifiedRoute {
  pattern: string;
  filePath: string;
  source: "app" | "pages";

  // Layouts pre-resolved during discovery
  layouts: string[];

  // Reserved files pre-resolved during discovery
  reservedFiles?: {
    error?: string;
    loading?: string;
    notFound?: string;
  };
}

// During route discovery, resolve ALL layout information
async function discoverRoute(
  pageFile: string,
  source: "app" | "pages",
  projectDir: string,
  adapter: RuntimeAdapter,
): Promise<UnifiedRoute> {
  const layouts = await discoverLayoutsForPage(pageFile, source, projectDir, adapter);
  const reservedFiles = source === "app"
    ? await discoverReservedFiles(pageFile, projectDir, adapter)
    : undefined;

  return {
    pattern: fileToPattern(pageFile, source),
    filePath: pageFile,
    source,
    layouts,
    reservedFiles,
  };
}
```

### Unified Layout Application

```typescript
async function applyLayouts(
  element: ReactElement,
  route: UnifiedRoute,
  options: LayoutOptions,
): Promise<ReactElement> {
  let wrapped = element;

  // Apply nested layouts (from innermost to outermost)
  for (const layoutPath of route.layouts.reverse()) {
    const Layout = await loadLayout(layoutPath, options);
    wrapped = createElement(Layout, { children: wrapped });
  }

  // Apply reserved components if present (App Router only)
  if (route.reservedFiles?.loading) {
    const Loading = await loadComponent(route.reservedFiles.loading, options);
    wrapped = createElement(Suspense, { fallback: createElement(Loading) }, wrapped);
  }

  if (route.reservedFiles?.error) {
    const ErrorComp = await loadComponent(route.reservedFiles.error, options);
    const Boundary = createErrorBoundary(ErrorComp);
    wrapped = createElement(Boundary, {}, wrapped);
  }

  return wrapped;
}
```

### Benefits

1. **No runtime router detection**: Layout info pre-computed during discovery
2. **Consistent behavior**: Same code path for all adapters
3. **Cacheable**: Route registry includes all layout info
4. **Predictable**: No branching based on adapter type or router detection
