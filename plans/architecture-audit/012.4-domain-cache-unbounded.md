# 012.4: Domain Lookup Cache Unbounded Growth

## Severity: LOW

## Location

**File:** `/Users/mattboon/Sites/veryfront-renderer/src/server/utils/domain-lookup.ts`
**Lines:** 22-26, 32-49

## The Problem

```typescript
// Lines 22-24: Configuration
const DOMAIN_CACHE_TTL_MS = 60_000;
const DOMAIN_CACHE_MAX_ENTRIES = 1000;

// Lines 25-26: Module-level caches
const domainCache = new Map<string, CacheEntry>();
const inFlightRequests = new Map<string, Promise<DomainLookupResult | null>>();

// Lines 32-49: Cleanup and eviction
function cleanupExpiredEntries(): void {
  const now = Date.now();
  for (const [key, entry] of domainCache) {
    if (entry.expiresAt < now) domainCache.delete(key);
  }
}

function evictOldestEntries(): void {
  if (domainCache.size < DOMAIN_CACHE_MAX_ENTRIES) return;

  const entries = [...domainCache.entries()].sort((a, b) => a[1].expiresAt - b[1].expiresAt);
  const toRemove = Math.ceil(entries.length * 0.1);

  for (let i = 0; i < toRemove; i++) {
    const entry = entries[i];
    if (entry) domainCache.delete(entry[0]);
  }
}
```

## Analysis

### Why It's LOW Severity

1. **Bounded by MAX_ENTRIES**: The cache has a hard limit of 1000 entries
2. **Cleanup Triggers**: Cleanup runs when cache exceeds 50% capacity
3. **TTL Expiration**: Entries expire after 60 seconds
4. **Eviction Strategy**: Oldest entries are evicted first

### Memory Impact

With 1000 entries and ~500 bytes per entry (domain string + result object):
- Maximum memory: ~500KB per pod
- Acceptable for production workloads

### What This Cache Stores

```typescript
interface CacheEntry {
  result: DomainLookupResult | null;
  expiresAt: number;
}

interface DomainLookupResult {
  project_id: string;
  project_slug: string;
  project_name: string;
  environment: { id: string; name: string } | null;
  release_id: string | null;
}
```

Key format: normalized domain (lowercase, port stripped)

### Is This Safe for Multi-Tenancy?

**YES** - Domain lookups are:
1. **Deterministic**: Same domain always maps to same project
2. **Public Information**: Domain-to-project mapping is not sensitive
3. **Per-Request Token**: The lookup uses a per-request token, not a cached one

## Potential Improvements

### 1. Missing Timeout (More Critical)

The fetch has no timeout - covered in **[012.1 - Missing Timeouts](./012.1-missing-timeouts.md)**.

### 2. Consider LRU Instead of Map

Current eviction is O(n log n) due to sorting:

```typescript
// Current: O(n log n) eviction
const entries = [...domainCache.entries()].sort((a, b) => a[1].expiresAt - b[1].expiresAt);
```

Could be improved with LRU wrapper:

```typescript
// Better: O(1) eviction with LRU
import { LRUCache } from "#veryfront/utils/lru-wrapper.ts";

const domainCache = new LRUCache<string, DomainLookupResult | null>({
  maxEntries: DOMAIN_CACHE_MAX_ENTRIES,
  ttlMs: DOMAIN_CACHE_TTL_MS,
});
```

### 3. Add Cache Hit Metrics

```typescript
// Track cache effectiveness
let cacheHits = 0;
let cacheMisses = 0;

export function getDomainCacheStats(): {
  size: number;
  maxSize: number;
  hitRate: number;
} {
  const total = cacheHits + cacheMisses;
  return {
    size: domainCache.size,
    maxSize: DOMAIN_CACHE_MAX_ENTRIES,
    hitRate: total > 0 ? cacheHits / total : 0,
  };
}
```

### 4. In-Flight Deduplication Safety

```typescript
// Lines 71-75: In-flight deduplication
const inFlight = inFlightRequests.get(cacheKey);
if (inFlight) {
  logger.debug("[DomainLookup] Waiting for in-flight request", { domain });
  return inFlight;

```

This is good practice - prevents thundering herd on the same domain. No changes needed.

## Comparison with Other Caches

| Cache | Type | Max Size | TTL | Eviction |
|-------|------|----------|-----|----------|
| domainCache | Map + manual | 1000 | 60s | Sort + slice (O(n log n)) |
| cachedPaths (http-cache) | LRUCache | 10000 | N/A | LRU (O(1)) |
| moduleCache | LRUCache | configurable | configurable | LRU (O(1)) |

The domain cache is the only one using plain Map with manual eviction. Could benefit from LRU wrapper but low priority.

## Solution

See **[012.0 - HTTP Clients RFC](./012.0-http-clients-rfc.md)** for the overall architecture.

For the domain lookup specifically, the main improvement needed is adding a timeout (see 012.1). The cache itself is correctly bounded and not a multi-tenancy risk.
