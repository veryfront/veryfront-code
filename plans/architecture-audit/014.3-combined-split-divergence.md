# 014.3: Combined vs Split Mode Behavior Divergence

## Severity: MEDIUM

## Location

**Combined Mode Entry:** `/Users/mattboon/Sites/veryfront-renderer/scripts/server.ts`
**Split Mode Proxy:** `/Users/mattboon/Sites/veryfront-renderer/proxy/main.ts`
**Split Mode Renderer:** `/Users/mattboon/Sites/veryfront-renderer/src/server/production-server.ts`
**Proxy Handler Logic:** `/Users/mattboon/Sites/veryfront-renderer/proxy/handler.ts`

## The Problem

Veryfront has two deployment modes:

### Combined Mode (Local Dev)
```
Client --> [Proxy Logic + Renderer] --> API
           (single process)
```
- Started with `deno task start`
- Proxy logic runs in-process via `requestInterceptor`
- Headers injected directly into Request object

### Split Mode (Production)
```
Client --> Proxy --> Renderer --> API
           (HTTP)    (HTTP)
```
- Proxy runs as separate container
- Renderer runs as separate container
- Headers passed via HTTP

The behavior **should** be identical, but there are subtle divergences.

## Divergence 1: Error Handling

### Combined Mode (scripts/server.ts)
```typescript
// scripts/server.ts:268-271
const requestInterceptor = async (req: Request): Promise<Request> => {
  const ctx = await proxyHandler.processRequest(req);
  return injectContextHeaders(req, ctx);
  // ^^^^^ Returns Request regardless of ctx.error
};
```

**Problem**: If `ctx.error` is set, the request is still forwarded with error context in headers. The renderer then sees a malformed request.

### Split Mode (proxy/main.ts)
```typescript
// proxy/main.ts:238-258
if (ctx.error) {
  const ms = Math.round(performance.now() - startTime);
  proxyLogger.error(`${ctx.error.status} ${req.method} ${url.pathname}`, {
    ms, domain: ctx.host,
  });

  // Handle redirect for protected environments
  if (ctx.error.redirectUrl) {
    return new Response(null, {
      status: 302,
      headers: { Location: ctx.error.redirectUrl },
    });
  }

  return jsonErrorResponse(ctx.error.status, {
    error: ctx.error.message,
    status: ctx.error.status,
  });
}
```

**Behavior**: Errors are handled immediately, returning proper error responses.

### Impact

In combined mode:
- Error responses are not returned immediately
- Request continues to renderer with incomplete context
- Renderer may produce unexpected results or crash

## Divergence 3: Request Timeout

### Combined Mode
```typescript
// scripts/server.ts - No explicit timeout on request interceptor
const requestInterceptor = async (req: Request): Promise<Request> => {
  const ctx = await proxyHandler.processRequest(req);  // No timeout!
  return injectContextHeaders(req, ctx);
};
```

### Split Mode (proxy/main.ts)
```typescript
// proxy/main.ts:52-53, 283-311
const RENDERER_REQUEST_TIMEOUT_MS = parseInt(
  getEnv("RENDERER_REQUEST_TIMEOUT_MS") || "90000"
);

// Create abort controller for timeout
const abortController = new AbortController();
const timeoutId = setTimeout(() => {
  abortController.abort();
}, RENDERER_REQUEST_TIMEOUT_MS);

try {
  response = await fetch(rendererUrl.toString(), {
    ...options,
    signal: abortController.signal,
  });
} finally {
  clearTimeout(timeoutId);
}

// Handle timeout specifically
if (error instanceof Error && error.name === "AbortError") {
  return jsonErrorResponse(504, {
    error: "Gateway Timeout",
    message: `Renderer request timed out after ${RENDERER_REQUEST_TIMEOUT_MS}ms`,
  });
}
```

**Impact**: Combined mode has no timeout protection at the proxy layer, relying solely on renderer-level timeouts.

## Evidence of Behavioral Differences

### Issue: Different error responses for same condition

**Combined mode with missing releaseId:**
- Request continues to renderer
- Renderer returns 502 JSON
- No logging at proxy layer

**Split mode with missing releaseId:**
- Proxy catches error immediately
- Returns 502 JSON with `domain` field
- Logs error with timing

### Issue: Different timeout behavior

**Combined mode with slow project:**
- Token fetch blocks indefinitely
- Only renderer timeout applies

**Split mode with slow project:**
- 90s timeout at proxy layer
- Clear 504 Gateway Timeout response

## Concrete Fix

### Option A: Unify Request Interceptor Behavior

```typescript
// scripts/server.ts - Updated request interceptor

const requestInterceptor = async (req: Request): Promise<Request | Response> => {
  const startTime = performance.now();
  const url = new URL(req.url);

  // Add timeout to proxy processing
  const timeoutMs = parseInt(Deno.env.get("PROXY_REQUEST_TIMEOUT_MS") || "30000");
  const abortController = new AbortController();
  const timeoutId = setTimeout(() => abortController.abort(), timeoutMs);

  try {
    const ctx = await proxyHandler.processRequest(req);

    // Handle errors immediately (like split mode)
    if (ctx.error) {
      const ms = Math.round(performance.now() - startTime);
      console.error(`[Proxy] ${ctx.error.status} ${req.method} ${url.pathname}`, { ms });

      if (ctx.error.redirectUrl) {
        return new Response(null, {
          status: 302,
          headers: { Location: ctx.error.redirectUrl },
        });
      }

      return new Response(
        JSON.stringify({ error: ctx.error.message, status: ctx.error.status }),
        { status: ctx.error.status, headers: { "Content-Type": "application/json" } },
      );
    }

    return injectContextHeaders(req, ctx);
  } catch (error) {
    if (error instanceof Error && error.name === "AbortError") {
      return new Response(
        JSON.stringify({ error: "Proxy request timeout", timeoutMs }),
        { status: 504, headers: { "Content-Type": "application/json" } },
      );
    }
    throw error;
  } finally {
    clearTimeout(timeoutId);
  }
};

// Update dev server to accept Response from interceptor
const devServer = await createDevServer({
  ...options,
  requestInterceptor,  // Now returns Request | Response
});
```

### Option C: Extract Common Proxy Logic

```typescript
// src/proxy/core.ts - Shared proxy logic

export async function handleProxyRequest(
  req: Request,
  handler: ProxyHandler,
  options: { timeout?: number } = {},
): Promise<{ type: "forward"; request: Request } | { type: "response"; response: Response }> {
  const timeoutMs = options.timeout ?? 30000;
  const startTime = performance.now();

  // Add timeout
  const ctx = await withTimeout(
    handler.processRequest(req),
    timeoutMs,
    () => ({
      error: { status: 504, message: "Proxy request timeout" },
    } as ProxyContext),
  );

  // Handle errors
  if (ctx.error) {
    const ms = Math.round(performance.now() - startTime);
    logger.error(`Proxy error: ${ctx.error.status}`, { ms, path: new URL(req.url).pathname });

    if (ctx.error.redirectUrl) {
      return {
        type: "response",
        response: new Response(null, {
          status: 302,
          headers: { Location: ctx.error.redirectUrl },
        }),
      };
    }

    return {
      type: "response",
      response: new Response(
        JSON.stringify({ error: ctx.error.message, status: ctx.error.status }),
        { status: ctx.error.status, headers: { "Content-Type": "application/json" } },
      ),
    };
  }

  // Forward request
  return {
    type: "forward",
    request: injectContextHeaders(req, ctx),
  };
}
```

Then use in both modes:
```typescript
// Combined mode
const result = await handleProxyRequest(req, proxyHandler);
if (result.type === "response") return result.response;
return result.request;

// Split mode
const result = await handleProxyRequest(req, proxyHandler, { timeout: RENDERER_REQUEST_TIMEOUT_MS });
if (result.type === "response") return result.response;
const response = await fetch(rendererUrl, { /* forward result.request */ });
```
