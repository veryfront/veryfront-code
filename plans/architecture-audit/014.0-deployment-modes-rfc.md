# RFC 014.0: Unified Deployment Mode Architecture

## Problem Statement

The veryfront-renderer has **three orthogonal axes** of deployment mode configuration that interact in complex, undocumented ways:

1. **Runtime Environment** (`NODE_ENV`): development | production | test
2. **Content Source** (`productionMode`/`PRODUCTION_MODE`): Draft (branch) vs Published (release)
3. **Deployment Topology** (`PROXY_MODE`): Combined (single process) vs Split (proxy + renderer)

These axes create a **2 x 2 x 2 = 8 possible mode combinations**, but only a subset are valid. The current implementation has:

1. **No central authority**: Mode determination is scattered across 20+ files
2. **Implicit fallbacks**: Missing env vars default to "development", which is unsafe in production
3. **Header/domain conflicts**: When proxy headers and domain parsing disagree, behavior is undefined
4. **Cache TTL misclassification**: Production requests misclassified as preview get 36x shorter cache TTLs
5. **Multi-tenant risks**: Different projects on the same pod could be classified into different modes

## Scope

This RFC covers:
- Unifying mode determination into a single `ModeResolver` service
- Explicit validation of mode combinations at startup
- Header propagation contract between proxy and renderer
- Cache key isolation by mode

Out of scope:
- Proxy OAuth token management (handled by proxy)
- HMR implementation details
- Specific cache eviction strategies

## Current State: Mode Determination Sources

| Source | Location | What It Determines |
|--------|----------|-------------------|
| `NODE_ENV` | Process env | `isLocalDev` flag |
| `PROXY_MODE` | Process env | Cache backend selection |
| `PRODUCTION_MODE` | Process env | Distributed cache TTLs |
| `x-environment` header | Proxy header | `resolvedEnvironment` |
| `x-release-id` header | Proxy header | Content source (release vs branch) |
| Domain parsing | `domain-parser.ts` | `isDraft`, `environment` |
| Config file | `veryfront.config.ts` | `productionMode` override |
| `x-content-source-id` | Proxy header | Cache key prefix |

### Precedence Conflicts

The current precedence is:
1. Config file (`productionMode: true`) overrides everything
2. Proxy headers override domain parsing
3. Domain parsing provides fallback
4. Environment variables provide defaults

**Problem**: This precedence is implicit and not enforced. Different code paths may use different precedence orders.

## Solution: ModeResolver Service

### Single Source of Truth

```typescript
// src/core/mode/resolver.ts
import { AsyncLocalStorage } from "node:async_hooks";

export interface ResolvedMode {
  // Immutable for the request lifetime
  readonly environment: "preview" | "production";
  readonly isLocalDev: boolean;
  readonly contentSourceId: string;
  readonly releaseId: string | null;
  readonly branch: string | null;
  readonly projectSlug: string;
  readonly projectId: string | null;

  // Computed properties
  readonly cacheTTLMultiplier: number;
  readonly shouldEnableCache: boolean;
  readonly shouldShowErrorDetails: boolean;
}

const modeStorage = new AsyncLocalStorage<ResolvedMode>();

export function getResolvedMode(): ResolvedMode | null {
  return modeStorage.getStore() ?? null;
}

export function requireResolvedMode(): ResolvedMode {
  const mode = getResolvedMode();
  if (!mode) {
    throw new Error(
      "No mode context available. " +
      "Ensure request is wrapped in runWithMode() or mode middleware."
    );
  }
  return mode;
}

export function runWithMode<T>(
  mode: ResolvedMode,
  fn: () => T | Promise<T>,
): T | Promise<T> {
  return modeStorage.run(mode, fn);
}
```

### Mode Resolution Logic

```typescript
// src/core/mode/resolver.ts (continued)

export interface ModeResolverInput {
  // From proxy headers (highest precedence)
  proxyEnvironment?: "preview" | "production";
  proxyReleaseId?: string;
  proxyContentSourceId?: string;
  proxyProjectSlug?: string;
  proxyProjectId?: string;

  // From domain parsing (fallback)
  parsedDomain: ParsedDomain;

  // From config (override)
  configProductionMode?: boolean;

  // From environment
  nodeEnv: string;
  proxyMode: boolean;

  // From request context
  isLocalProject: boolean;
}

export function resolveMode(input: ModeResolverInput): ResolvedMode {
  // Step 1: Determine base environment
  let environment: "preview" | "production";

  if (input.configProductionMode === true) {
    // Config override takes absolute precedence
    environment = "production";
  } else if (input.proxyEnvironment) {
    // Proxy header is authoritative in proxy mode
    environment = input.proxyEnvironment;
  } else if (input.parsedDomain.environment) {
    // Fall back to domain parsing
    environment = input.parsedDomain.environment;
  } else {
    // Default to preview for safety
    environment = "preview";
  }

  // Step 2: Determine isLocalDev
  const isLocalDev = input.nodeEnv !== "production" || input.isLocalProject;

  // Step 3: Validate production requirements
  if (environment === "production" && !isLocalDev) {
    if (!input.proxyReleaseId && !input.configProductionMode) {
      throw new ModeValidationError(
        "Production environment requires releaseId. " +
        "Either provide x-release-id header or set config.fs.veryfront.productionMode=true"
      );
    }
  }

  // Step 4: Compute contentSourceId
  const contentSourceId = input.proxyContentSourceId ??
    computeContentSourceId(
      isLocalDev,
      environment,
      input.parsedDomain.branch,
      input.proxyReleaseId,
    );

  // Step 5: Compute derived properties
  const cacheTTLMultiplier = environment === "production" ? 1 : 0.028; // 36x shorter for preview
  const shouldEnableCache = !isLocalDev && environment === "production";
  const shouldShowErrorDetails = isLocalDev || environment === "preview";

  return Object.freeze({
    environment,
    isLocalDev,
    contentSourceId,
    releaseId: input.proxyReleaseId ?? null,
    branch: input.parsedDomain.branch ?? null,
    projectSlug: input.proxyProjectSlug ?? input.parsedDomain.slug ?? "",
    projectId: input.proxyProjectId ?? null,
    cacheTTLMultiplier,
    shouldEnableCache,
    shouldShowErrorDetails,
  });
}
```

### Startup Validation

```typescript
// src/core/mode/validator.ts

export function validateEnvironmentConfiguration(): void {
  const nodeEnv = process.env.NODE_ENV;
  const proxyMode = process.env.PROXY_MODE === "1";

  // Critical: NODE_ENV must be set in proxy mode
  if (proxyMode && !nodeEnv) {
    throw new ConfigurationError(
      "NODE_ENV must be set when PROXY_MODE=1. " +
      "This prevents accidental dev behavior in production."
    );
  }

  // Warning: PROXY_MODE=1 without NODE_ENV=production
  if (proxyMode && nodeEnv !== "production") {
    logger.warn(
      "[ModeValidator] PROXY_MODE=1 but NODE_ENV is not 'production'. " +
      "This may cause unexpected behavior with caching and error handling.",
      { nodeEnv, proxyMode }
    );
  }

  // Info: Log effective configuration at startup
  logger.info("[ModeValidator] Mode configuration validated", {
    nodeEnv: nodeEnv ?? "(unset, defaulting to development)",
    proxyMode,
    productionMode: process.env.PRODUCTION_MODE === "1",
  });
}
```

### Header Contract

The proxy MUST inject these headers for every request to renderer:

| Header | Required | Description |
|--------|----------|-------------|
| `x-environment` | Yes | `"preview"` or `"production"` |
| `x-project-slug` | Yes | Project identifier |
| `x-content-source-id` | Yes | Cache isolation key |
| `x-release-id` | If production | Release identifier |
| `x-project-id` | No | Project UUID (for metrics) |
| `x-branch-name` | If preview | Branch name |
| `x-token` | Yes | API token for fetching content |

Renderer MUST validate:
- `x-environment` is present and valid
- `x-release-id` is present when `x-environment=production`
- `x-content-source-id` format matches expected pattern

## Migration Plan

### Phase 1: Add ModeResolver (Week 1)

1. Create `src/core/mode/resolver.ts` with new API
2. Add startup validation in `src/server/bootstrap.ts`
3. Keep existing scattered logic as fallback

### Phase 2: Migrate Consumers (Week 2-3)

1. Update `universal-handler/index.ts` to use ModeResolver
2. Migrate cache key builders to use `requireResolvedMode()`
3. Migrate error handlers to use `shouldShowErrorDetails`
4. Migrate cache TTL lookups to use `cacheTTLMultiplier`

### Phase 3: Remove Legacy Paths (Week 4)

1. Remove direct `NODE_ENV` checks (use `isLocalDev`)
2. Remove direct `PROXY_MODE` checks (use ModeResolver)
3. Remove scattered `productionMode` checks
4. Add deprecation warnings to legacy APIs

```typescript
// tests/mode-resolver.test.ts

Deno.test("resolveMode throws for production without releaseId", () => {
  assertThrows(() => {
    resolveMode({
      proxyEnvironment: "production",
      proxyReleaseId: undefined,  // Missing!
      parsedDomain: { slug: "test", branch: null, environment: "production" },
      nodeEnv: "production",
      proxyMode: true,
      isLocalProject: false,
    });
  }, ModeValidationError);
});

Deno.test("resolveMode allows production without releaseId for local projects", () => {
  const mode = resolveMode({
    proxyEnvironment: "production",
    proxyReleaseId: undefined,
    parsedDomain: { slug: "test", branch: null, environment: "production" },
    nodeEnv: "production",
    proxyMode: true,
    isLocalProject: true,  // Local project is exempt
  });

  assertEquals(mode.environment, "production");
  assertEquals(mode.isLocalDev, true);
});

Deno.test("proxy environment overrides domain parsing", () => {
  const mode = resolveMode({
    proxyEnvironment: "preview",  // Proxy says preview
    proxyReleaseId: undefined,
    parsedDomain: {
      slug: "test",
      branch: null,
      environment: "production"  // Domain says production
    },
    nodeEnv: "production",
    proxyMode: true,
    isLocalProject: false,
  });

  // Proxy wins
  assertEquals(mode.environment, "preview");
});

Deno.test("config override beats everything", () => {
  const mode = resolveMode({
    proxyEnvironment: "preview",  // Proxy says preview
    parsedDomain: {
      slug: "test",
      branch: null,
      environment: "preview"  // Domain says preview
    },
    configProductionMode: true,  // Config says production
    nodeEnv: "development",
    proxyMode: false,
    isLocalProject: false,
  });

  // Config wins
  assertEquals(mode.environment, "production");
});
```
