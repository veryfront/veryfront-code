# 018.1: HMR Client Map Unbounded

## Severity: MEDIUM (Dev only)

## Location

**File:** `src/server/hmr/`

```typescript
// LEAK - Map grows without cleanup
const clients = new Map<string, WebSocket>();

function addClient(id: string, ws: WebSocket): void {
  clients.set(id, ws);
}

// Missing: No removal when WebSocket closes
```

## The Problem

HMR client connections are tracked but never removed. Each browser refresh or reconnect adds an entry that's never cleaned up.

### Growth Pattern

```
Dev session (8 hours):
├── Initial load: 1 entry
├── Browser refresh: +1 entry (old not removed)
├── Code change HMR: connection stays
├── 50 refreshes: 50 entries
└── Memory: 50 * (socket + metadata) ≈ 5-50MB
```

### Why It Grows

```typescript
// WebSocket close event exists but...
ws.onclose = () => {
  // Entry not removed from map
  console.log("Client disconnected");
};
```

## Proposed Fix

### Option A: Cleanup on Close

```typescript
const clients = new Map<string, WebSocket>();

function addClient(id: string, ws: WebSocket): void {
  // Remove old connection if exists
  const existing = clients.get(id);
  if (existing) {
    existing.close();
  }

  clients.set(id, ws);

  ws.addEventListener("close", () => {
    // Only remove if this is still the active connection
    if (clients.get(id) === ws) {
      clients.delete(id);
    }
  });
}
```

### Option B: WeakRef-Based Tracking

```typescript
const clients = new Map<string, WeakRef<WebSocket>>();

function addClient(id: string, ws: WebSocket): void {
  clients.set(id, new WeakRef(ws));
}

function broadcast(message: string): void {
  for (const [id, ref] of clients) {
    const ws = ref.deref();
    if (ws && ws.readyState === WebSocket.OPEN) {
      ws.send(message);
    } else {
      clients.delete(id);  // Cleanup stale refs
    }
  }
}
```

### Option C: Periodic Cleanup

```typescript
const clients = new Map<string, WebSocket>();

// Run every 5 minutes
setInterval(() => {
  for (const [id, ws] of clients) {
    if (ws.readyState !== WebSocket.OPEN) {
      clients.delete(id);
    }
  }
}, 5 * 60 * 1000);
```

## Blast Radius

| Affected | Impact |
|----------|--------|
| Dev server | Memory growth during long sessions |
| HMR broadcasts | Sending to dead sockets |
| Resource cleanup | Leaked file handles |

## Migration Checklist

- [ ] Add `close` event handler to remove clients
- [ ] Add periodic cleanup sweep
- [ ] Log client count for monitoring
- [ ] Test memory during extended dev session
