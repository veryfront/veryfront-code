## The Problem

Two completely different things are named `VeryfrontError`:

| Aspect | veryfront-error.ts | types.ts |
|--------|-------------------|----------|
| **Kind** | Type alias (union) | Class |
| **Runtime** | Plain object | Error instance |
| **Stack trace** | NONE | Yes |
| **Type guard** | `isBuildError(e)` etc. | `e instanceof VeryfrontError` |
| **Error codes** | `type: "build"` | `code: ErrorCode.BUILD_ERROR` |
| **Throwable** | No (must wrap with `toError()`) | Yes (directly) |

This causes:

1. **Import confusion**: Which `VeryfrontError` to use?
2. **Type check failures**: `instanceof` doesn't work on type union
3. **Lost stack traces**: Type union version has no stack
4. **Inconsistent catching**: Some code catches class, some checks `type` property

## Solution

See **[010.0 - Error Handling RFC](./010.0-error-handling-rfc.md)** for the complete solution.

**Summary of fixes:**

1. **Rename type union** to `VeryfrontErrorData` (plain object for serialization)
2. **Keep class** as canonical `VeryfrontError` (for throwing)
3. **Deprecate** `createError()` + `toError()` pattern
4. **Add factory functions** that return proper Error instances:

```typescript
// NEW: Factory that returns Error instance with stack
export function createBuildError(
  message: string,
  context?: BuildContext,
  cause?: Error,
): VeryfrontError {
  return new VeryfrontError(message, ErrorCode.BUILD_ERROR, context, { cause });
}

// Usage:
throw createBuildError(
  `Failed to transform module: ${filePath}`,
  { file: filePath, phase: "transform" },
  originalError,  // Preserves original stack via cause
);
```

## Quick Fix

If full refactor is delayed, at least fix stack trace capture:

```typescript
// In veryfront-error.ts, modify toError() to capture stack at call site
export function toError(veryfrontError: VeryfrontError): Error {
  const error = new Error(veryfrontError.message);
  error.name = `VeryfrontError[${veryfrontError.type}]`;

  // Capture stack at this point (not perfect but better)
  if (Error.captureStackTrace) {
    Error.captureStackTrace(error, toError);  // Exclude toError from stack
  }

  Object.defineProperty(error, "context", {
    value: veryfrontError,
    enumerable: false,
    configurable: true,
  });
  return error;
}
```
