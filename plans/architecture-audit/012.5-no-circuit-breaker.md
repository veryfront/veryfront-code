## The Problem

When an external service becomes unhealthy (slow, returning errors, or completely down), the current HTTP clients will:

1. Continue sending requests
2. Wait for timeouts (or forever if no timeout)
3. Retry failed requests
4. Consume resources waiting for doomed requests

This creates a cascade failure pattern where one unhealthy service can bring down the entire renderer.

### Current Behavior

```typescript
// Example: retry-handler.ts
for (let attempt = 0; attempt <= maxRetries; attempt++) {
  try {
    const response = await fetch(url, { headers });
    // ...
  } catch (error) {
    // Wait and retry regardless of how many times this service has failed
    const delay = Math.min(initialDelay * Math.pow(2, attempt), maxDelay);
    await new Promise((resolve) => setTimeout(resolve, delay));
  }
}
// No memory of past failures - next request starts fresh
```

## Why It's Bad for Multi-Tenancy

### Cascade Failure Scenario

```
Timeline:
=========
t0: esm.sh experiences high latency (10s+ per request)

t1-t50: 50 concurrent SSR requests from various projects
        Each request needs esm.sh modules
        Each waits 10s+ for slow responses
        Worker pool exhausted

t51: New requests start queueing
     ALL projects affected - not just those needing esm.sh

t52-t100: Queue grows, timeouts at load balancer level
          User-visible errors for ALL tenants

Without Circuit Breaker:
- Every request to esm.sh will wait and fail
- Resources wasted on requests that will certainly fail
- No fast-fail mechanism

With Circuit Breaker:
- After N failures, circuit "opens"
- Subsequent requests fail immediately with CircuitBreakerOpenError
- Resources preserved for healthy services
- Circuit "half-opens" after reset period to test recovery
```

### Impact by Service

| External Service | Failure Impact |
|------------------|----------------|
| Veryfront API | All file operations, metadata |
| GitHub API | All GitHub-backed projects |
| esm.sh | All SSR with npm packages |
| OAuth endpoint | All token refresh operations |
| GCS | All blob/artifact operations |

## Circuit Breaker Pattern

### State Machine

```
         ┌─────────────────────────────────────────┐
         │                                         │
         ▼                                         │
    ┌─────────┐                              ┌─────────┐
    │ CLOSED  │────────────────────────────▶│  OPEN   │
    │         │    N failures in window      │         │
    └─────────┘                              └─────────┘
         ▲                                         │
         │                                         │
         │       Success on test request           │
         │                                         ▼
         │                                   ┌───────────┐
         └───────────────────────────────────│HALF-OPEN  │
                                             │           │
                                             └───────────┘
                                                   │
                                                   │ Failure on test
                                                   ▼
                                              Back to OPEN
```

### States

| State | Behavior |
|-------|----------|
| **CLOSED** | Normal operation, requests pass through |
| **OPEN** | Fast-fail all requests, no external calls |
| **HALF-OPEN** | Allow one test request to check recovery |

## Solution

See **[012.0 - HTTP Clients RFC](./012.0-http-clients-rfc.md)** for the complete implementation.

### Circuit Breaker Middleware

```typescript
// src/http/middleware/circuit-breaker.ts
export interface CircuitBreakerConfig {
  /** Number of failures before opening circuit */
  failureThreshold: number;
  /** Time in ms before attempting to close circuit */
  resetTimeout: number;
  /** Time window in ms for counting failures */
  rollingWindowMs: number;
}

interface CircuitState {
  state: "closed" | "open" | "half-open";
  failures: number[];
  nextAttemptTime: number;
}

export function createCircuitBreakerMiddleware(
  config: CircuitBreakerConfig,
): HttpMiddleware {
  // Per-hostname circuit state
  const circuits = new Map<string, CircuitState>();

  function getCircuit(hostname: string): CircuitState {
    let circuit = circuits.get(hostname);
    if (!circuit) {
      circuit = {
        state: "closed",
        failures: [],
        nextAttemptTime: 0,
      };
      circuits.set(hostname, circuit);
    }
    return circuit;
  }

  function recordFailure(circuit: CircuitState): void {
    const now = Date.now();
    circuit.failures.push(now);

    // Remove failures outside rolling window
    const windowStart = now - config.rollingWindowMs;
    circuit.failures = circuit.failures.filter((t) => t > windowStart);

    // Check if we should open the circuit
    if (circuit.failures.length >= config.failureThreshold) {
      circuit.state = "open";
      circuit.nextAttemptTime = now + config.resetTimeout;
      logger.warn("[CircuitBreaker] Circuit opened", {
        failures: circuit.failures.length,
        threshold: config.failureThreshold,
        resetAt: new Date(circuit.nextAttemptTime).toISOString(),
      });
    }
  }

  function recordSuccess(circuit: CircuitState): void {
    if (circuit.state === "half-open") {
      circuit.state = "closed";
      circuit.failures = [];
      logger.info("[CircuitBreaker] Circuit closed after successful test");
    }
  }

  return async (request, next) => {
    const url = new URL(request.url);
    const circuit = getCircuit(url.hostname);

    // Check circuit state
    if (circuit.state === "open") {
      const now = Date.now();
      if (now >= circuit.nextAttemptTime) {
        // Transition to half-open for test request
        circuit.state = "half-open";
        logger.info("[CircuitBreaker] Circuit half-open, testing recovery", {
          hostname: url.hostname,
        });
      } else {
        // Fast-fail
        throw new CircuitBreakerOpenError(
          `Circuit breaker open for ${url.hostname}`,
          {
            hostname: url.hostname,
            resetAt: circuit.nextAttemptTime,
            failures: circuit.failures.length,
          },
        );
      }
    }

    try {
      const response = await next(request);

      // Count non-2xx as failure for circuit breaker
      if (!response.ok) {
        recordFailure(circuit);
      } else {
        recordSuccess(circuit);
      }

      return response;
    } catch (error) {
      recordFailure(circuit);
      throw error;
    }
  };
}

export class CircuitBreakerOpenError extends Error {
  constructor(
    message: string,
    public readonly context: {
      hostname: string;
      resetAt: number;
      failures: number;
    },
  ) {
    super(message);
    this.name = "CircuitBreakerOpenError";
  }
}
```

### Recommended Thresholds

| Service | Failure Threshold | Reset Timeout | Window |
|---------|-------------------|---------------|--------|
| Veryfront API | 5 | 60s | 60s |
| GitHub API | 5 | 120s | 60s |
| esm.sh | 3 | 30s | 30s |
| OAuth | Disabled | - | - |
| GCS | 5 | 60s | 60s |

OAuth is disabled because:
- Critical path - can't fast-fail authentication
- Failures are usually client errors (invalid credentials)
- Better to let individual requests fail than block all auth

### Graceful Degradation

When circuit is open, callers should handle gracefully:

```typescript
// Example: SSR with fallback
async function renderWithModules(url: string): Promise<string> {
  try {
    const module = await fetchModule(url);
    return renderWithModule(module);
  } catch (error) {
    if (error instanceof CircuitBreakerOpenError) {
      // Circuit is open - esm.sh is down
      logger.warn("Module fetch failed due to circuit breaker", {
        url,
        resetAt: error.context.resetAt,
      });
      // Fallback: render without client-side interactivity
      return renderStaticFallback();
    }
    throw error;
  }
}
```

## Testing

```typescript
// tests/circuit-breaker.test.ts
Deno.test("circuit breaker opens after threshold failures", async () => {
  const middleware = createCircuitBreakerMiddleware({
    failureThreshold: 3,
    resetTimeout: 1000,
    rollingWindowMs: 5000,
  });

  const failingNext = async () => {
    throw new Error("Connection refused");
  };

  const request = new Request("https://example.com");

  // First 3 failures should pass through
  for (let i = 0; i < 3; i++) {
    try {
      await middleware(request, failingNext);
    } catch {
      // Expected
    }
  }

  // 4th request should fail fast with circuit breaker
  await assertRejects(
    () => middleware(request, failingNext),
    CircuitBreakerOpenError,
    "Circuit breaker open",
  );
});

Deno.test("circuit breaker resets after timeout", async () => {
  const middleware = createCircuitBreakerMiddleware({
    failureThreshold: 1,
    resetTimeout: 100, // 100ms for testing
    rollingWindowMs: 5000,
  });

  const failingNext = async () => {
    throw new Error("Connection refused");
  };

  const request = new Request("https://example.com");

  // Trigger circuit open
  try {
    await middleware(request, failingNext);
  } catch {
    // Expected
  }

  // Should be open
  await assertRejects(
    () => middleware(request, failingNext),
    CircuitBreakerOpenError,
  );

  // Wait for reset
  await new Promise((r) => setTimeout(r, 150));

  // Should be half-open, allowing test request
  const successNext = async () => new Response("ok");
  const response = await middleware(request, successNext);
  assertEquals(response.status, 200);
});
```
