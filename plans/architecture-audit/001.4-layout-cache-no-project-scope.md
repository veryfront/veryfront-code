# 001.4 - Layout Discovery Cache Without Project Scoping

## Severity: 🔴 CRITICAL - Multi-Tenant Data Leakage

## The Bug

**File**: `src/rendering/layouts/utils/discovery.ts`

```typescript
// Line 8 - GLOBAL cache, no project isolation
const layoutDiscoveryCache = new Map<string, LayoutItem[]>();

export async function discoverNestedLayouts(
  pageFilePath: string,
  rootDir: string,
  projectDir: string,
  adapter: RuntimeAdapter,
): Promise<LayoutItem[]> {
  // Line 27 - Key does NOT include projectId
  const key = simpleHash(pageFilePath, rootDir);
  const cached = layoutDiscoveryCache.get(key);
  if (cached) return cached;

  const result = await discoverNestedLayoutsImpl(pageFilePath, rootDir, projectDir, adapter);
  layoutDiscoveryCache.set(key, result);
  return result;
}
```

## Why This Is Critical

### Multi-Tenant Collision Scenario

**Project A** has:
```
/virtual/projectA/app/dashboard/layout.tsx  → renders Sidebar A
/virtual/projectA/app/dashboard/page.tsx
```

**Project B** has:
```
/virtual/projectB/app/dashboard/layout.tsx  → renders Sidebar B
/virtual/projectB/app/dashboard/page.tsx
```

### The Collision

1. Request for Project A's `/dashboard` page arrives
2. `discoverNestedLayouts` called with:
   - `pageFilePath`: `/virtual/projectA/app/dashboard/page.tsx`
   - `rootDir`: `/virtual/projectA/app`
3. Cache key: `simpleHash("/virtual/projectA/app/dashboard/page.tsx", "/virtual/projectA/app")`
4. Result: `[{ path: "/virtual/projectA/app/dashboard/layout.tsx" }]`
5. Cached under that key

Later:

6. Request for Project B's `/dashboard` page arrives
7. `discoverNestedLayouts` called with:
   - `pageFilePath`: `/virtual/projectB/app/dashboard/page.tsx`
   - `rootDir`: `/virtual/projectB/app`
8. **IF the hash collides** (same relative structure), wrong layouts returned

## Wait - Does simpleHash Prevent This?

Let's check `simpleHash`:

```typescript
// From #veryfront/utils
export function simpleHash(...args: string[]): string {
  return hashString(args.join(':'));
}
```

The key would be:
- Project A: `hash("/virtual/projectA/app/dashboard/page.tsx:/virtual/projectA/app")`
- Project B: `hash("/virtual/projectB/app/dashboard/page.tsx:/virtual/projectB/app")`

**These ARE different**. So the immediate collision doesn't happen.

## But The Real Problems Remain

### Problem 1: Unbounded Memory Growth

```typescript
const layoutDiscoveryCache = new Map<string, LayoutItem[]>();
```

- No LRU eviction
- No max size limit
- No TTL
- Grows forever as more projects are accessed

With 1000 projects × 100 pages = 100,000 cache entries. Each entry contains:
- `LayoutItem[]` with path strings
- Never cleaned up

**Memory leak in long-running renderer pods.**

### Problem 2: No Cache Invalidation on File Changes

```typescript
export function clearLayoutDiscoveryCache(): void {
  layoutDiscoveryCache.clear();
}
```

This clears the **ENTIRE** cache, for **ALL** projects.

If Project A's layout changes:
1. HMR triggers
2. `clearLayoutDiscoveryCache()` called
3. **ALL projects** lose their cache
4. Cold start penalty for everyone

### Problem 3: Stale Entries After Deployment

When a project is redeployed:
1. New layout files may exist
2. But old cache entries remain
3. Old layout paths returned
4. **Stale layouts served until pod restart**

### Problem 4: No Project-Scoped Invalidation

There's no way to clear cache for just one project:

```typescript
// We need this, but it doesn't exist:
export function clearLayoutCacheForProject(projectId: string): void {
  // Can't do this because cache keys don't include projectId
}
```

## The Fix

### Option A: Include Project ID in Cache Key

```typescript
export async function discoverNestedLayouts(
  pageFilePath: string,
  rootDir: string,
  projectDir: string,
  adapter: RuntimeAdapter,
  projectId: string,  // 🟢 NEW PARAMETER
): Promise<LayoutItem[]> {
  const key = simpleHash(projectId, pageFilePath, rootDir);  // 🟢 Include projectId
  // ...
}
```

### Option B: Use LRU Cache with TTL

```typescript
import { LRUCache } from 'lru-cache';

const layoutDiscoveryCache = new LRUCache<string, LayoutItem[]>({
  max: 10000,           // Max entries
  ttl: 1000 * 60 * 5,   // 5 minute TTL
});

export function clearLayoutCacheForProject(projectId: string): void {
  for (const key of layoutDiscoveryCache.keys()) {
    if (key.startsWith(`${projectId}:`)) {
      layoutDiscoveryCache.delete(key);
    }
  }
}
```

### Option C: Request-Scoped Cache

```typescript
import { AsyncLocalStorage } from 'async_hooks';

const requestLayoutCache = new AsyncLocalStorage<Map<string, LayoutItem[]>>();

export async function discoverNestedLayouts(...): Promise<LayoutItem[]> {
  const cache = requestLayoutCache.getStore();
  if (!cache) {
    // No request context, compute without caching
    return await discoverNestedLayoutsImpl(...);
  }

  const key = simpleHash(pageFilePath, rootDir);
  const cached = cache.get(key);
  if (cached) return cached;

  const result = await discoverNestedLayoutsImpl(...);
  cache.set(key, result);
  return result;
}
```
