# 001.4 - Layout Cache Without Project Scoping

## Severity: 🔴 CRITICAL

## The Bug

```typescript
// discovery.ts:8 - GLOBAL cache, no project isolation
const layoutDiscoveryCache = new Map<string, LayoutItem[]>();

// Key does NOT include projectId
const key = simpleHash(pageFilePath, rootDir);
```

## Problems

| Issue | Impact |
|-------|--------|
| Unbounded growth | Memory leak - never evicts |
| No project-scoped invalidation | `clearLayoutDiscoveryCache()` clears ALL projects |
| Stale after deploy | Old layouts served until pod restart |

## Fix

```typescript
const layoutDiscoveryCache = new LRUCache<string, LayoutItem[]>({
  max: 10000,
  ttl: 1000 * 60 * 5,  // 5 min TTL
});

// Include projectId in key
const key = simpleHash(projectId, pageFilePath, rootDir);
```

## Value

| Before | After |
|--------|-------|
| Memory grows forever | Bounded cache with eviction |
| One project clears all | Project-scoped invalidation |
| Stale layouts after deploy | TTL ensures freshness |
