# 001.2 - Unsafe Type Casting

## Severity: 🟠 HIGH

## The Pattern

```typescript
const wrappedAdapter = fs.getUnderlyingAdapter() as {
  getProjectData?: () => { updated_at?: string } | undefined;
};
```

Ad-hoc types, no interface, no compile-time safety.

## Locations

| File | Cast |
|------|------|
| `pipeline.ts:651` | `as { getProjectData? }` |
| `html.ts:320` | `as unknown as` (double cast!) |
| `getEntityInfo.ts:96` | `as { getEntityIdForPath? }` |
| `layout-collector.ts:304` | `wrappedAdapter: unknown` |

## Impact

- Methods may not exist at runtime
- Silent fallback to `undefined`
- No IDE autocompletion
- No TypeScript protection

## Fix

Add optional methods to the adapter interface:

```typescript
interface ExtendedFSAdapter {
  getProjectMetadata?(): ProjectMetadata | undefined;
  getAllSourceFiles?(): Promise<SourceFile[]>;
}
```

## Value

| Before | After |
|--------|-------|
| Runtime surprises | Compile-time checks |
| No documentation | Self-documenting interface |
| Copy-paste casts | Single source of truth |
