# 013.2 Agent Cache Missing Project Isolation

## Summary

The agent cache in `src/agent/middleware/cache/cache.ts` generates cache keys using only input hash, without project context. This could cause cross-project data leakage if the same prompt is used in different projects.

## Value Table

| Issue | Severity | Effort | Priority |
|-------|----------|--------|----------|
| Agent cache cross-project leakage | Medium | Low | P2 |

## Bug Location

**File**: `/Users/mattboon/Sites/veryfront-renderer/src/agent/middleware/cache/cache.ts`
**Line**: 204

```typescript
// src/agent/middleware/cache/cache.ts:204
const keyGenerator = config.keyGenerator ?? ((input: string) => `cache_${hashString(input)}`);
```

## Problem Analysis

### Current Behavior

The default key generator creates keys based solely on input hash:

```typescript
// Line 204: Default key generator - NO PROJECT CONTEXT
const keyGenerator = config.keyGenerator ?? ((input: string) => `cache_${hashString(input)}`);

// Line 206-208: Key is used without project scoping
function keyFor(input: string, context?: Record<string, unknown>): string {
  return keyGenerator(input, context);
}
```

The `cacheMiddleware` function passes context, but the default generator ignores it:

```typescript
// Line 244-291: Cache middleware
export function cacheMiddleware(
  config: CacheConfig,
): (
  context: Record<string, unknown>,
  next: () => Promise<AgentResponse>,
) => Promise<AgentResponse> {
  const cache = createCache(config);

  return (
    context: Record<string, unknown>,
    next: () => Promise<AgentResponse>,
  ): Promise<AgentResponse> =>
    withSpan(
      "agent.middleware.cache",
      async () => {
        const inputString = typeof context.input === "string"
          ? context.input
          : JSON.stringify(context.input);

        // Context is available but ignored by default keyGenerator
        const cached = cache.get(inputString, context);
        // ...
      },
      { "cache.strategy": config.strategy },
    );
}
```

### Risk Scenario

1. Project A sends prompt "Generate a button component" to agent
2. Agent response is cached with key `cache_${hash("Generate a button component")}`
3. Project B sends identical prompt "Generate a button component"
4. **BUG**: Project B receives Project A's cached response

This is problematic because:
- Agent responses may include project-specific context
- Agent may have access to project-specific tools/data
- Different projects may have different agent configurations

## Impact Assessment

**Severity: Medium**
- Agents are not widely used in production yet
- Risk increases as agent adoption grows
- Information leakage could include project-specific data

**Likelihood: Medium**
- Requires same prompt in different projects
- More likely with common prompts (e.g., "summarize", "generate")

## Recommended Fix

### Option 1: Update Default Key Generator (Recommended)

```typescript
// BEFORE (line 204)
const keyGenerator = config.keyGenerator ?? ((input: string) => `cache_${hashString(input)}`);

// AFTER
const keyGenerator = config.keyGenerator ?? ((input: string, context?: Record<string, unknown>) => {
  const projectId = (context?.projectId as string) ?? 'global';
  return `agent:${projectId}:${hashString(input)}`;
});
```

### Option 2: Require Project Context in CacheConfig

```typescript
interface CacheConfig {
  strategy: "memory" | "lru" | "ttl";
  maxSize?: number;
  ttl?: number;
  keyGenerator?: (input: string, context?: Record<string, unknown>) => string;
  requireProjectScope?: boolean; // NEW: Enforce project scoping
}
```

### Option 3: Project-Aware Cache Instance

```typescript
export function createProjectScopedCache(projectId: string, config: CacheConfig) {
  const baseCache = createCache(config);

  return {
    get(input: string, context?: Record<string, unknown>) {
      const scopedKey = `${projectId}:${input}`;
      return baseCache.get(scopedKey, context);
    },
    set(input: string, response: AgentResponse, context?: Record<string, unknown>) {
      const scopedKey = `${projectId}:${input}`;
      return baseCache.set(scopedKey, response, context);
    },
    // ... other methods
  };
}
```

## Verification Test

Add to test suite:

```typescript
// tests/agent/cache-isolation.test.ts
describe("Agent cache isolation", () => {
  it("isolates cache entries between projects", async () => {
    const cache = createCache({ strategy: "memory" });

    const responseA: AgentResponse = {
      output: "Project A response",
      metadata: { projectId: "project-a" }
    };
    const responseB: AgentResponse = {
      output: "Project B response",
      metadata: { projectId: "project-b" }
    };

    // Same input, different projects
    cache.set("test prompt", responseA, { projectId: "project-a" });
    cache.set("test prompt", responseB, { projectId: "project-b" });

    // Should return project-specific responses
    const resultA = cache.get("test prompt", { projectId: "project-a" });
    const resultB = cache.get("test prompt", { projectId: "project-b" });

    assertEquals(resultA?.output, "Project A response");
    assertEquals(resultB?.output, "Project B response");
  });
});
```
