## Precedence Matrix

| Source | Checked By | Precedence Order |
|--------|-----------|------------------|
| Config `productionMode` | `isProductionMode()` | 1 (highest) |
| `x-environment` header | `universal-handler` | 2 |
| Domain `.preview.` | `domain-parser.ts` | 3 |
| `x-forwarded-host` | `request-context.ts` | 3 (same as domain) |
| Default | Various | 4 (lowest) |

**Problem**: Different code paths use different precedence orders!

## Conflict Scenarios

### Scenario 1: Header Says Preview, Domain Says Production

```
Request: https://example.veryfront.com/page
Headers: x-environment: preview

Domain Parser: environment = "production" (no .preview. in domain)
Proxy Header: environment = "preview"
```

**Actual Behavior (universal-handler):**

## Concrete Fix

### Option A: Single Precedence Function (Recommended)

```typescript
// src/core/mode/precedence.ts

export interface EnvironmentSources {
  configProductionMode?: boolean;
  proxyHeader?: "preview" | "production";
  domainEnvironment?: "preview" | "production";
  forwardedHostEnvironment?: "preview" | "production";
}

/**
 * Resolve environment from multiple sources with explicit precedence.
 *
 * Precedence (highest to lowest):
 * 1. Config productionMode override (always wins)
 * 2. Proxy x-environment header (authoritative in proxy mode)
 * 3. Domain parsing (fallback)
 * 4. Default: "preview" (safe default)
 */
export function resolveEnvironment(
  sources: EnvironmentSources,
  options: { isProxyMode: boolean } = { isProxyMode: false },
): { environment: "preview" | "production"; source: string } {
  // 1. Config override (highest precedence)
  if (sources.configProductionMode === true) {
    return { environment: "production", source: "config.productionMode" };
  }

  // 2. Proxy header (in proxy mode)
  if (options.isProxyMode && sources.proxyHeader) {
    return { environment: sources.proxyHeader, source: "x-environment header" };
  }

  // 3. Domain parsing
  if (sources.domainEnvironment) {
    return { environment: sources.domainEnvironment, source: "domain parsing" };
  }

  // 4. Forwarded host (for reverse proxy scenarios)
  if (sources.forwardedHostEnvironment) {
    return { environment: sources.forwardedHostEnvironment, source: "x-forwarded-host" };
  }

  // 5. Safe default
  return { environment: "preview", source: "default" };
}
```

### Option B: Validate Consistency at Resolution Time

```typescript
// src/core/mode/validator.ts

export function validateEnvironmentConsistency(sources: EnvironmentSources): void {
  const environments = [
    sources.proxyHeader,
    sources.domainEnvironment,
    sources.forwardedHostEnvironment,
  ].filter(Boolean);

  // Check for conflicts
  const unique = new Set(environments);
  if (unique.size > 1) {
    logger.warn("[EnvironmentValidator] Multiple conflicting environments detected", {
      proxyHeader: sources.proxyHeader,
      domainEnvironment: sources.domainEnvironment,
      forwardedHostEnvironment: sources.forwardedHostEnvironment,
      resolution: "using highest precedence source",
    });
  }

  // Check config vs runtime conflict
  if (sources.configProductionMode === true && sources.proxyHeader === "preview") {
    logger.warn(
      "[EnvironmentValidator] Config forces production but proxy header says preview",
      { action: "config wins, serving production content" }
    );
  }
}
```

## Current Precedence Documentation

Add to CLAUDE.md:

```markdown
## Environment Resolution Precedence

When multiple sources provide environment information, precedence is:

1. **Config file** (`fs.veryfront.productionMode: true`) - Always wins
2. **Proxy headers** (`x-environment`) - Authoritative in proxy mode
3. **Domain parsing** (`.preview.` in host) - Fallback
4. **Default** - `preview` (safe default)

If sources conflict, the higher precedence source wins and a warning is logged.
```
