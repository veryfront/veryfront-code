# 001.1 - The Layout Bug

## Severity: 🔴 CRITICAL

## The Bug

Nested layouts (`app/dashboard/layout.tsx`) work locally but **disappear in production**.

## Root Cause

```typescript
// layout-collector.ts:296
if (isVeryfrontAPI) {
  return await this.collectAPILayoutConfiguration(fs);  // Only checks config + components/
} else {
  return await this.collectFilesystemLayouts(pageFilePath); // Walks app/ directory
}
```

API adapter **never walks directories** - it only checks `config.layout` and `components/layout.*`.

## Impact

```
User's project:
  app/layout.tsx              ← Works
  app/dashboard/layout.tsx    ← IGNORED in production
  app/dashboard/page.tsx

Result: Sidebars, nested navs, section layouts all missing in production.
```

## Fix

Delete `collectAPILayoutConfiguration`. Use `collectFilesystemLayouts` for ALL adapters.

## Value

| Before | After |
|--------|-------|
| Layouts break in prod | Layouts work everywhere |
| Hours debugging "missing sidebar" | Zero adapter-related layout bugs |
