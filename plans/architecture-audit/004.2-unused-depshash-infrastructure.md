# 004.2 - Unused depsHash Infrastructure

## Severity: MEDIUM - Dead Code / Incomplete Feature

## The Problem

The codebase contains **infrastructure for dependency hash tracking** that was designed but **never connected**. This creates confusion about whether dependency tracking works and makes the code harder to understand.

## The Disconnected Infrastructure

### 1. BundleMetadata.meta.depsHash

**Location:** `/Users/mattboon/Sites/veryfront-renderer/src/utils/bundle-manifest.ts:3-17`

```typescript
export interface BundleMetadata {
  hash: string;
  codeHash: string;
  size: number;
  compiledAt: number;
  source: string;
  mode: "development" | "production";
  meta?: {
    type?: "mdx" | "component" | "layout" | "provider";
    depsHash?: string;           // <-- DEFINED
    reactVersion?: string;
    headings?: Array<{ id: string; text: string; level: number }>;
  };
}
```

**Problem:** This field is never set when creating metadata, and never read when validating cache entries.

### 2. NestedLayoutsResult.depsHash

**Location:** `/Users/mattboon/Sites/veryfront-renderer/src/rendering/layouts/types.ts:7-10`

```typescript
export interface NestedLayoutsResult {
  nestedLayouts: LayoutItem[];
  depsHash: string;              // <-- DEFINED
}
```

**Problem:** This is computed by `computeDepsHash()` but the result is not used for cache invalidation.

### 3. computeDepsHash() Function

**Location:** `/Users/mattboon/Sites/veryfront-renderer/src/rendering/layouts/utils/hash-calculator.ts:5-45`

```typescript
export async function computeDepsHash(
  layoutBundle: MdxBundle | undefined,
  nestedLayouts: LayoutItem[],
  adapter: RuntimeAdapter,
): Promise<string> {
  try {
    const hashPromises: Promise<string>[] = [];

    if (layoutBundle) {
      hashPromises.push(computeHash(String(layoutBundle.compiledCode ?? "")));
    }

    for (const item of nestedLayouts) {
      if (!item) continue;

      if (item.componentPath) {
        hashPromises.push(
          adapter.fs
            .readFile(item.componentPath)
            .then((src) => computeHash(src))
            // ...
        );
        continue;
      }

      const compiledCode = item.bundle?.compiledCode;
      if (compiledCode) {
        hashPromises.push(computeHash(String(compiledCode)));
      }
    }

    const depParts = await Promise.all(hashPromises);
    return depParts.filter(Boolean).join(":");
  } catch (e) {
    // ...
    return "";
  }
}
```

**Problem:** This function is **exported but never imported** elsewhere in the codebase for cache key generation.

## Evidence of Intentional Design

The presence of this infrastructure in multiple files suggests **intentional design** that was never completed:

1. **Interface defines field** - `BundleMetadata.meta.depsHash`
2. **Result type includes field** - `NestedLayoutsResult.depsHash`
3. **Implementation exists** - `computeDepsHash()` function
4. **But no wiring** - Nothing calls `computeDepsHash()` and uses the result

## Options

### Option A: Complete the Feature

Connect `computeDepsHash()` to cache key generation:

```typescript
// In layout caching code
const depsHash = await computeDepsHash(layoutBundle, nestedLayouts, adapter);
const cacheKey = buildLayoutComponentCacheKey(projectId, componentPath, hash, depsHash);
```

### Option B: Remove Dead Code

If dependency tracking will be implemented differently (see RFC), remove this infrastructure:

1. Delete `depsHash` from `BundleMetadata.meta`
2. Delete `depsHash` from `NestedLayoutsResult`
3. Delete `computeDepsHash()` function
4. Add proper implementation per RFC

## Recommendation

**Option A is preferred** if the existing implementation is sufficient for layouts. However, the RFC proposes a more comprehensive solution that may supersede this.

For now:
1. Keep the infrastructure (it's a good design)
2. Connect it as part of RFC implementation
3. Extend the pattern to all transform caching, not just layouts

## Solution

See **[004.0 - Dependency Tracking RFC](./004.0-dependency-tracking-rfc.md)** for the complete solution.

The RFC proposes a unified dependency tracking system that will either:
- Leverage and extend this existing infrastructure
- Replace it with a more comprehensive solution
