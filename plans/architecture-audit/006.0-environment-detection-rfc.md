# 006.0: RFC - Unified Environment Detection

## TL;DR

**Problem**: 5 different SSR/browser detection patterns + 3 runtime detection approaches = inconsistent behavior and multi-tenancy risks from shared globals

**Solution**: Single source of truth for environment detection via explicit context passing, not global state inspection

---

## The Problem

### Current State: Detection Pattern Chaos

The codebase uses **5 different patterns** to detect SSR vs browser:

| Pattern | Location | Risk |
|---------|----------|------|
| `typeof window !== "undefined"` | 12+ files | Fails when SSR globals stub `window` |
| `typeof document !== "undefined"` | 5+ files | Same problem |
| `globalThis.__VERYFRONT_SSR__` | `Head.tsx` | Relies on mutable global state |
| `isBrowser(ctx: TransformContext)` | Transform pipeline | Correct - uses explicit context |
| Inline `const isBrowser = ...` | `markdown.tsx`, `mdx-module-loader.ts` | Module-level evaluation, not request-scoped |

### Why This Matters for Multi-Tenancy

**Critical Bug Scenario**: In a multi-tenant renderer serving Project A and Project B concurrently:

```
Timeline:
T1: Project A request starts SSR → setupSSRGlobals() sets globalThis.__VERYFRONT_SSR__ = true
T2: Project B request starts SSR (concurrent) → sees __VERYFRONT_SSR__ already true, skips setup
T3: Project A SSR completes → cleanup removes globalThis.window
T4: Project B SSR tries typeof window !== "undefined" → FAILS (window gone)
```

The root issue: **global state is not request-scoped**.

### Detection Inconsistency Examples

**1. `Head.tsx` vs `LivePageContextProvider.tsx`**

```typescript
// Head.tsx:5-6 - Uses global flag
function isServerEnvironment(): boolean {
  const ssrFlag = (globalThis as Record<string, unknown>).__VERYFRONT_SSR__;
  return ssrFlag === true || typeof window === "undefined";
}

// LivePageContextProvider.tsx:21-25 - Uses typeof window directly
const context: PageContext = pageContext ?? {
  path: typeof window !== "undefined" ? globalThis.location.pathname : "/",
  query: typeof window !== "undefined"
    ? Object.fromEntries(new URLSearchParams(globalThis.location.search))
    : {},
};
```

**Problem**: These can disagree! When `setupSSRGlobals()` creates a window stub:
- `Head.tsx` sees `__VERYFRONT_SSR__ = true` and correctly identifies SSR
- `LivePageContextProvider.tsx` sees `window` exists and thinks it's browser

**2. Module-Level `isBrowser` Evaluation**

```typescript
// markdown.tsx:32 - Evaluated ONCE at module load
const isBrowser = typeof window !== "undefined" && typeof document !== "undefined";

// Later in MermaidDiagram - uses stale value
if (!isBrowser) return null;  // May be wrong for this specific request
```

**Problem**: Module-level constants are evaluated once, not per-request. In long-running server:
- Module loads during first request (SSR) - `isBrowser = false`
- HMR or test reloads module in browser context - `isBrowser = true`
- But the server is still serving SSR requests with `isBrowser = true`

## Solution: Request-Scoped Environment Context

### Design Principles

1. **No global state for environment detection** - use explicit context
2. **Single source of truth** - one function/type for all environment checks
3. **Request-scoped** - environment is passed through render context, not inspected
4. **Browser-only files exempt** - files in `client/`, `spa/`, hydration code can use `typeof window`

### New API

```typescript
// ═══════════════════════════════════════════════════════════════
// src/rendering/environment.ts - Single source of truth
// ═══════════════════════════════════════════════════════════════

export type RenderEnvironment = "ssr" | "browser" | "rsc-server";

export interface RenderContext {
  /** Where this code is executing */
  environment: RenderEnvironment;
  /** Project being rendered (for multi-tenancy) */
  projectSlug?: string;
  /** Request ID for tracing */
  requestId?: string;
}

/** Check if we're in SSR context */
export function isSSR(ctx: RenderContext): boolean {
  return ctx.environment === "ssr" || ctx.environment === "rsc-server";
}

/** Check if we're in browser context */
export function isBrowserEnv(ctx: RenderContext): boolean {
  return ctx.environment === "browser";
}

/**
 * DEPRECATED: Use isSSR(ctx) instead.
 * This fallback exists only for gradual migration.
 */
export function isServerEnvironmentLegacy(): boolean {
  // Only for files that haven't been migrated yet
  const ssrFlag = (globalThis as Record<string, unknown>).__VERYFRONT_SSR__;
  if (ssrFlag === true) return true;

  // Final fallback - but log a warning in dev
  const hasWindow = typeof globalThis.window !== "undefined";
  if (process.env.NODE_ENV === "development" && !hasWindow) {
    console.warn(
      "[Veryfront] Using legacy isServerEnvironmentLegacy(). " +
      "Migrate to isSSR(ctx) for multi-tenant safety."
    );
  }
  return !hasWindow;
}
```

### Migration Examples

**1. Head.tsx - Before/After**

```typescript
// BEFORE: Global state inspection
function isServerEnvironment(): boolean {
  const ssrFlag = (globalThis as Record<string, unknown>).__VERYFRONT_SSR__;
  return ssrFlag === true || typeof window === "undefined";
}

export function Head({ children }: { children: React.ReactNode }) {
  const isSSR = isServerEnvironment();
  // ...
}

// AFTER: Context-based (via React context)
import { useRenderContext } from "#veryfront/rendering/environment.ts";

export function Head({ children }: { children: React.ReactNode }) {
  const ctx = useRenderContext();
  const isSSR = ctx.environment !== "browser";
  // ...
}
```

**2. LivePageContextProvider.tsx - Before/After**

```typescript
// BEFORE: typeof window check
const context: PageContext = pageContext ?? {
  path: typeof window !== "undefined" ? globalThis.location.pathname : "/",
  query: typeof window !== "undefined"
    ? Object.fromEntries(new URLSearchParams(globalThis.location.search))
    : {},
};

// AFTER: Explicit context
export function LivePageContextProvider({
  children,
  pageContext,
  renderEnv,  // NEW: explicit environment
}: {
  children: React.ReactNode;
  pageContext?: TypedPageContext;
  renderEnv: RenderEnvironment;
}) {
  const context: PageContext = pageContext ?? {
    slug: "",
    path: renderEnv === "browser" ? globalThis.location.pathname : "/",
    params: {},
    query: renderEnv === "browser"
      ? Object.fromEntries(new URLSearchParams(globalThis.location.search))
      : {},
  };
  // ...
}
```

**3. markdown.tsx - Before/After**

```typescript
// BEFORE: Module-level constant (WRONG)
const isBrowser = typeof window !== "undefined" && typeof document !== "undefined";

function MermaidDiagram({ code }: { code: string }) {
  if (!isBrowser) return null;  // Uses stale value
  // ...
}

// AFTER: Hook-based (request-scoped)
import { useRenderContext } from "#veryfront/rendering/environment.ts";

function MermaidDiagram({ code }: { code: string }) {
  const ctx = useRenderContext();
  if (ctx.environment !== "browser") {
    return (
      <pre className="..."><code>{code}</code></pre>
    );
  }
  // Browser-only mermaid rendering...
}
```

## Implementation Plan

| Phase | Change | Files |
|-------|--------|-------|
| 1. Create context | Add `RenderContext` type and `RenderContextProvider` | New file |
| 2. Wire into SSR | Pass context through `renderToString` pipeline | `src/rendering/` |
| 3. Migrate Head.tsx | Use context instead of global | `Head.tsx` |
| 4. Migrate providers | Use context in `LivePageContextProvider` | Provider files |
| 5. Migrate components | Use `useRenderContext()` in markdown, etc. | Component files |
| 6. Remove globals | Delete `__VERYFRONT_SSR__` from `setupSSRGlobals` | `ssr-globals/` |
| 7. Lint rule | Add ESLint rule to flag `typeof window` in non-client files | `.eslintrc` |
