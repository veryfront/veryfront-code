## Why It's Bad for Multi-Tenancy

### Hanging Fetch = Resource Leak

Each hanging fetch:
1. Holds a TCP connection open indefinitely
2. Consumes memory for the pending Promise
3. May hold semaphore permits or other resources
4. Blocks the calling async context

### In-Flight Request Maps Grow Unbounded

Several components track in-flight requests to deduplicate:

```typescript
// server/utils/domain-lookup.ts:26
const inFlightRequests = new Map<string, Promise<DomainLookupResult | null>>();

// ssr-module-loader/cache/memory.ts:35
export const globalInProgress = new Map<string, Promise<void>>();
```

If the underlying fetch hangs forever, these maps:
- Never remove the entry
- Grow unbounded over pod lifetime
- Eventually cause memory exhaustion

## Failure Scenario

### Timeline: Hanging Domain Lookup

```
T+0s:    Request for project-a.veryfront.com
         fetchDomainLookup() called
         inFlightRequests.set("project-a.veryfront.com", promise)

T+0.1s:  API server unresponsive (network issue, overload)
         fetch() hangs, no timeout configured

T+0.5s:  Another request for project-a.veryfront.com
         Finds entry in inFlightRequests
         Waits for same hanging promise

T+60s:   REQUEST_TIMEOUT_MS fires
         Handler returns 504 to client
         But fetch() is STILL pending
         inFlightRequests STILL holds promise

T+∞:     Promise never resolves
         Entry never removed from map
         Repeat for each unique domain with network issues
```

### Memory Growth Over Time

```
Hour 0:  inFlightRequests.size = 0
Hour 1:  10 hanging fetches, size = 10
Hour 2:  25 hanging fetches, size = 25
Hour 8:  200 hanging fetches, size = 200
         Each entry: ~1KB (promise + closure + string)
         Total: 200KB in just this one Map
```

Multiply by all the Maps across the codebase, and memory grows steadily until OOM.

## Solution

See **[009.0 - Timeout Handling RFC](./009.0-timeout-handling-rfc.md)** Section 5 for the `timeoutFetch` utility.

### Quick Fix for Each File

```typescript
// Before
const response = await fetch(url, { headers });

// After
import { timeoutFetch } from "#veryfront/utils/timeout-fetch.ts";

const response = await timeoutFetch(url, {
  headers,
  timeoutMs: ms(10_000),
  label: "domain-lookup",
});
```
