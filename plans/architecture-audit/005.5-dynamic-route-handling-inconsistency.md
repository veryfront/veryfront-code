## Why It's Bad for Multi-Tenancy

### Problem 1: SSG Builds Differ by Router Type

| Aspect | App Router | Pages Router |
|--------|-----------|--------------|
| Dynamic routes in build | **Skipped entirely** | Included, expanded via getStaticPaths |
| Build output | Only static routes | Static + expanded dynamic routes |
| Runtime fallback | All dynamic routes SSR | Only new dynamic routes SSR |

A project migrating from Pages Router to App Router will see dramatically different SSG behavior.

### Problem 2: No Dynamic Segment Priority

When multiple dynamic segments exist at the same level, both routers take the "first match" which is filesystem-order dependent:

```
app/
  shop/
    [category]/
      page.tsx
    [productId]/       ← Which matches /shop/electronics?
      page.tsx
```

Filesystem order is:
- Alphabetically sorted on most systems
- But not guaranteed
- Different on local vs production

### Problem 3: Catch-All Handling Differs

**App Router**: Catch-all `[...slug]` breaks the loop early
```typescript
if (isCatchAll) break;
```

**Pages Router**: Catch-all handled at file level, pattern extracted differently

## Failure Scenario

**Scenario: Dynamic route priority ambiguity**

```
Project structure:
  app/
    docs/
      [section]/
        page.tsx          # Expects: { section: "api" }
      [...path]/
        page.tsx          # Expects: { path: ["api"] }

URL: /docs/api

Behavior:
  - Filesystem lists [section] before [...path] (alphabetical)
  - First match wins: [section]
  - params = { section: "api" }

But on another system or after rename:
  - Filesystem lists [...path] first
  - First match wins: [...path]
  - params = { path: ["api"] }

INCONSISTENT BEHAVIOR BETWEEN ENVIRONMENTS!
```

**Scenario: SSG build differences**

```
Project structure (Pages Router):
  pages/
    blog/
      [slug].tsx         # Has getStaticPaths returning 100 posts

SSG Build:
  - Discovers pages/blog/[slug].tsx
  - Calls getStaticPaths()
  - Generates 100 static pages: /blog/post-1, /blog/post-2, ...

After migration to App Router:
  app/
    blog/
      [slug]/
        page.tsx         # Same getStaticPaths

SSG Build:
  - walkAppSSG skips [slug] directory entirely
  - Generates 0 static pages for blog
  - All 100 posts now SSR instead of static!
```

### Unified Dynamic Route Discovery

```typescript
interface UnifiedRoute {
  pattern: string;        // e.g., "/blog/[slug]"
  filePath: string;
  source: "app" | "pages";
  isDynamic: boolean;
  paramNames: string[];   // e.g., ["slug"]
  priority: number;       // Higher = more specific
}

// Calculate priority based on specificity
function calculatePriority(pattern: string): number {
  const parts = pattern.split("/").filter(Boolean);
  let priority = 0;

  for (const part of parts) {
    if (part.startsWith("[[...")) {
      // Optional catch-all: lowest priority
      priority += 1;
    } else if (part.startsWith("[...")) {
      // Catch-all: very low priority
      priority += 2;
    } else if (part.startsWith("[")) {
      // Dynamic segment: low priority
      priority += 10;
    } else {
      // Static segment: high priority
      priority += 100;
    }
  }

  return priority;
}

// Sort routes by priority (most specific first)
function sortRoutesByPriority(routes: UnifiedRoute[]): UnifiedRoute[] {
  return routes.sort((a, b) => b.priority - a.priority);
}
```

### Fix SSG for App Router Dynamic Routes

```typescript
async function collectAllRoutesForSSG(
  registry: RouteRegistry,
  adapter: RuntimeAdapter,
): Promise<string[]> {
  const pages: string[] = [];

  for (const route of registry.routes) {
    // Static routes: add directly
    if (!route.isDynamic) {
      pages.push(route.pattern);
      continue;
    }

    // Dynamic routes: check for getStaticPaths
    // SAME BEHAVIOR FOR BOTH ROUTERS!
    try {
      const module = await loadModule(route.filePath, adapter);
      if (typeof module.getStaticPaths === "function") {
        const { paths } = await module.getStaticPaths();
        for (const pathObj of paths) {
          const resolved = resolvePattern(route.pattern, pathObj.params);
          pages.push(resolved);
        }
      }
    } catch {
      // No getStaticPaths or error - route will be SSR
    }
  }

  return pages;
}
```

### Deterministic Matching Order

```typescript
function resolveRoute(
  slug: string,
  registry: RouteRegistry,
): UnifiedRoute | null {
  const normalizedSlug = normalizeSlug(slug);

  // Routes already sorted by priority during discovery
  // Most specific routes (more static segments) checked first
  for (const route of registry.routes) {
    if (!route.isDynamic) {
      if (route.pattern === normalizedSlug) {
        return route;
      }
      continue;
    }

    // Dynamic matching
    const params = matchDynamicRoute(route.pattern, normalizedSlug);
    if (params) {
      return route;
    }
  }

  return null;
}
```

Deno.test("dynamic route priority is deterministic", async () => {
  const routes: UnifiedRoute[] = [
    { pattern: "/docs/[...path]", priority: 0, isDynamic: true },
    { pattern: "/docs/[section]", priority: 0, isDynamic: true },
    { pattern: "/docs/api", priority: 0, isDynamic: false },
  ];

  // Calculate priorities
  for (const route of routes) {
    route.priority = calculatePriority(route.pattern);
  }

  const sorted = sortRoutesByPriority(routes);

  // Static route first
  assertEquals(sorted[0].pattern, "/docs/api");
  // Single dynamic before catch-all
  assertEquals(sorted[1].pattern, "/docs/[section]");
  // Catch-all last
  assertEquals(sorted[2].pattern, "/docs/[...path]");
