## The Problem

The import map resolution logic has several gaps that cause inconsistent behavior between what developers expect (based on browser import map spec) and what actually happens.

### 1. Dynamic Import Not Always Resolved

**transformer.ts:42-47:**
```typescript
// Dynamic imports - last pattern applied
transformedCode = transformedCode.replace(
  /import\s*\(\s*["']([^"']+)["']\s*\)/g,
  (_match, specifier) => {
    return `import("${resolve(specifier)}")`;  // No shouldResolve check!
  },
);
```

**But static imports check shouldResolve (transformer.ts:29-35):**
```typescript
transformedCode = transformedCode.replace(
  /((?:import|export)...["']([^"']+)["']/g,
  (_match, prefix, specifier) => {
    if (!shouldResolve(specifier, options)) return `${prefix}"${specifier}"`;  // CHECK HERE
    return `${prefix}"${resolve(specifier)}"`;
  },
);
```

**Inconsistency:**
```javascript
// Static import: shouldResolve checked
import { foo } from "bar";  // May be skipped based on options

// Dynamic import: ALWAYS resolved
const mod = await import("bar");  // Always resolved, ignores options
```

### 2. Scoped Imports Only Check Exact Match

**resolver.ts:52-58:**
```typescript
export function resolveImport(
  specifier: string,
  importMap: ImportMapConfig,
  scope?: string,
): string {
  const scopedImports = scope ? importMap.scopes?.[scope] : undefined;

  const scopedExact = scopedImports?.[specifier];  // ONLY exact match
  if (scopedExact) return scopedExact;

  // Falls through to global...
```

**Missing: Scoped prefix matching**

Browser import map spec supports prefix matching in scopes:

```json
{
  "imports": {
    "lodash": "https://esm.sh/lodash@4.17.21"
  },
  "scopes": {
    "/admin/": {
      "lodash/": "https://esm.sh/lodash@4.17.15/"
    }
  }
}
```

When importing from `/admin/dashboard.js`:
- `lodash` should resolve to v4.17.21 (global)
- `lodash/map` should resolve to v4.17.15/map (scoped prefix)

**Current behavior:** Scoped prefix matching NOT implemented. `lodash/map` falls through to global.

### 3. esm.sh Subpath Resolution Edge Cases

**resolver.ts:60-88:**
```typescript

    }

**Problem: File path mappings lose subpath**

```json
{

    "react": "file:///node_modules/react/index.js"

```

Input: `https://esm.sh/react@19/jsx-runtime`
Expected: `file:///node_modules/react/jsx-runtime.js`

### 4. Bare Specifier Resolution Conditional

**transformer.ts:4-17:**
```typescript
function shouldResolve(specifier: string, options?: TransformOptions): boolean {
  // esm.sh URLs always resolved
  if (specifier.startsWith("https://esm.sh/") || specifier.startsWith("http://esm.sh/")) {
    return true;
  }

  const isBare = !specifier.startsWith("http") && !specifier.startsWith("/") &&
    !specifier.startsWith(".");

  if (isBare && !options?.resolveBare) {
    return false;  // Bare specifiers skipped unless explicitly enabled!
  }

  return true;
}
```

**Default behavior skips bare specifiers:**

```javascript
// Import map: { "lodash": "https://esm.sh/lodash" }

// Input
import _ from "lodash";

// Without resolveBare: true
import _ from "lodash";  // UNCHANGED! Import map ignored!

// With resolveBare: true
import _ from "https://esm.sh/lodash";  // Correctly resolved
```

Callers must remember to pass `{ resolveBare: true }` or import maps are partially ineffective.

### 5. File Extension Fallback Incomplete

**resolver.ts:90-94:**

**Missing extensions: .ts, .tsx, .jsx, .json**

Input: `@/utils.ts`
Expected: `./src/utils.ts`
Actual: `@/utils.ts` (no match, extension not stripped)

## Why It's Bad for Multi-Tenancy

### Project-Specific Import Maps Behave Differently

Each project can have its own import map:

```typescript
// Project A import map
{
  "imports": { "lodash": "https://esm.sh/lodash@4.17.21" }
}

// Project B import map (scoped for subpaths)
{
  "imports": { "lodash": "https://cdn.example.com/lodash" },
  "scopes": {
    "/features/": { "lodash/": "https://cdn.example.com/lodash-custom/" }
  }
}
```

Project B's scoped prefix mapping doesn't work. Developer expects browser import map behavior but gets different results.

### Inconsistent Behavior Based on Call Site

```typescript
// Some callers pass resolveBare: true
transformImportsWithMap(code, importMap, scope, { resolveBare: true });

// Others don't
transformImportsWithMap(code, importMap, scope);  // Bare specifiers ignored!
```

Same import map, different results based on where transform is called.

## Failure Scenario

```
Scenario: Import Map with Scoped Dependencies
=============================================

1. Developer configures import map for different lodash in admin section:
   ```json
   {
     "imports": { "lodash": "https://esm.sh/lodash@4.17.21" },
     "scopes": {
       "/admin/": { "lodash/": "https://esm.sh/lodash@4.17.15/" }
     }
   }
   ```

2. Admin page imports lodash submodule:
   ```javascript
   // /admin/dashboard.js
   import map from "lodash/map";  // Expects 4.17.15
   ```

3. Import map transformer runs:
   - scope: "/admin/"
   - specifier: "lodash/map"
   - scopedImports["/admin/"]["lodash/map"] -> undefined (prefix not matched)
   - Falls through to global
   - importMap.imports["lodash/map"] -> undefined
   - Returns "lodash/map" unchanged!

4. Result:
   - Browser: "Cannot resolve bare specifier 'lodash/map'"
   - Or: Wrong version loaded from global esm.sh
```

## Solution

See **[011.0 - Import Rewriting RFC](./011.0-import-rewriting-rfc.md)** for the complete unified solution.

### Fix 1: Implement Prefix Matching for Scopes

```typescript
export function resolveImport(
  specifier: string,
  importMap: ImportMapConfig,
  scope?: string,
): string {
  const scopedImports = scope ? importMap.scopes?.[scope] : undefined;

  // Exact match first
  const scopedExact = scopedImports?.[specifier];
  if (scopedExact) return scopedExact;

  // PREFIX MATCHING for scoped imports
  if (scopedImports) {
    for (const [key, value] of Object.entries(scopedImports)) {
      if (key.endsWith("/") && specifier.startsWith(key)) {
        return value + specifier.slice(key.length);
      }
    }
  }

  // Then check global...
}
```

### Fix 2: Consistent shouldResolve for Dynamic Imports

```typescript
transformedCode = transformedCode.replace(
  /import\s*\(\s*["']([^"']+)["']\s*\)/g,
  (_match, specifier) => {

    if (!shouldResolve(specifier, options)) return _match;
    return `import("${resolve(specifier)}")`;

### Fix 3: Proper Subpath Handling for File Paths

```typescript
if (isFilePath) {
  // Map subpath to local path structure
  if (subpath && mapping.endsWith(".js")) {
    const dir = mapping.slice(0, mapping.lastIndexOf("/"));
    return `${dir}${subpath}.js`;
  }
  return mapping;
}
```

### Fix 4: Default resolveBare to True

```typescript
function shouldResolve(specifier: string, options?: TransformOptions): boolean {
  // ... esm.sh check ...

  const isBare = !specifier.startsWith("http") && !specifier.startsWith("/") &&
    !specifier.startsWith(".");

  // CHANGED: Default to true, not false
  if (isBare && options?.resolveBare === false) {
    return false;
  }

  return true;
}
```

## Testing

```typescript
// tests/import-map-resolution.test.ts
import { assertEquals } from "@std/assert";
import { resolveImport } from "../src/modules/import-map/resolver.ts";
import { transformImportsWithMap } from "../src/modules/import-map/transformer.ts";

Deno.test("scoped prefix matching works", () => {
  const importMap = {
    imports: { "lodash": "https://esm.sh/lodash@4.17.21" },
    scopes: {
      "/admin/": { "lodash/": "https://esm.sh/lodash@4.17.15/" }
    }
  };

  const result = resolveImport("lodash/map", importMap, "/admin/");
  assertEquals(result, "https://esm.sh/lodash@4.17.15/map");
});

Deno.test("dynamic imports respect shouldResolve", () => {
  const code = `const mod = await import("lodash");`;
  const importMap = { imports: { "lodash": "https://esm.sh/lodash" } };

  // Without resolveBare
  const result1 = transformImportsWithMap(code, importMap, undefined, { resolveBare: false });
  assertEquals(result1.includes(`import("lodash")`), true);

  // With resolveBare
  const result2 = transformImportsWithMap(code, importMap, undefined, { resolveBare: true });
  assertEquals(result2.includes(`import("https://esm.sh/lodash")`), true);
});

Deno.test("file path mapping preserves subpath", () => {
  const importMap = {
    imports: { "react": "file:///node_modules/react" }
  };

  const result = resolveImport("https://esm.sh/react@19/jsx-runtime", importMap);
  // Should include subpath
  assertEquals(result, "file:///node_modules/react/jsx-runtime");
});

Deno.test("bare specifiers resolved by default", () => {
  const code = `import _ from "lodash";`;
  const importMap = { imports: { "lodash": "https://esm.sh/lodash" } };

  // Default (no options)
  const result = transformImportsWithMap(code, importMap);
  assertEquals(result.includes("https://esm.sh/lodash"), true);
});
```
