## The Problem

### Example 1: Inline Deno Check (script-page-handling.ts)

```typescript
// src/rendering/script-page-handling.ts:310-318
function rewriteNpmImports(code: string): string {
  const isDeno = typeof (globalThis as { Deno?: unknown }).Deno !== "undefined";
  if (!isDeno) return code;

  let result = code;
  for (const { pattern, replacement } of NPM_REWRITES) {
    result = result.replace(pattern, replacement);
  }
  return result;
}
```

**Problems:**
1. Duplicates runtime detection logic (already exists in `src/platform/compat/runtime.ts`)
2. Uses inline `typeof Deno` instead of centralized `isDeno` constant
3. Business logic (npm rewriting) coupled to runtime identity

### Example 2: Module Loading Strategy (s3-storage.ts)

```typescript
// src/workflow/blob/s3-storage.ts:23-25
s3Module = isDeno
  ? await import("https://esm.sh/@aws-sdk/client-s3@3.490.0")
  : await import("@aws-sdk/client-s3");
```

**Problems:**
1. Runtime-specific import URLs scattered in business logic
2. Adding Bun support requires modifying this file
3. No centralized "optional dependency loader"

### Example 3: SSR Import Rewriter (ssr-import-rewriter.ts)

```typescript
// src/modules/server/ssr-import-rewriter.ts:18-23
function shouldKeepBareSpecifier(specifier: string): boolean {
  // npm: specifiers are only supported in Deno, not Node.js
  if (specifier.startsWith("npm:")) {
    return isDeno;
  }
  // ...
}
```

**Problem:** Transform logic knows about runtime capabilities instead of querying an adapter.

### Example 4: HTTP Cache Decisions (http-cache.ts)

```typescript
// src/transforms/esm/http-cache.ts:215-220
if (isDeno && isReactCoreUrl(normalizedUrl)) {
  logger.debug("[HTTP-CACHE] Skipping React core module for Deno");
  return null;
}

// Line 397-398
if (isDeno) {
  return specifier; // Let Deno's native npm resolution handle it
}

// Line 421-423
if (isDeno && isReactCoreUrl(resolved)) {
  return normalizeHttpUrl(resolved);
}
```

**Problem:** Caching strategy is coupled to runtime identity rather than capabilities like `supportsHttpImports` or `supportsNpmProtocol`.

## Why It's Bad

### 1. Maintenance Burden

Adding Cloudflare Workers support requires:
- Modifying `src/platform/compat/runtime.ts` (correct)
- Modifying 45+ files with `isDeno`/`isNode` checks (wrong)

### 2. Inconsistent Detection

```typescript
// src/platform/compat/runtime.ts - Canonical detection
export const isDeno: boolean = !isNode && !isBun && hasRealDeno();

// src/rendering/script-page-handling.ts - Inline detection
const isDeno = typeof (globalThis as { Deno?: unknown }).Deno !== "undefined";
// Missing: isBun check, dnt shim detection
```

The inline check doesn't exclude Bun or dnt shims, leading to potential false positives.

### 3. Identity vs Capability

Checking `isDeno` is identity-based: "Are you Deno?"

Checking `adapter.capabilities.supportsHttpImports` is capability-based: "Can you import HTTP URLs?"

Capability-based is more correct because:
- Deno behind a proxy might not support HTTP imports
- Future runtimes might add HTTP import support
- Test environments can mock capabilities

---

## Failure Scenario

**Adding Bun support to http-cache.ts:**

```
Developer reads: if (isDeno) { return specifier; }
Developer thinks: "Bun also supports npm: specifiers natively"
Developer changes to: if (isDeno || isBun) { return specifier; }

But: Bun's npm resolution has subtle differences from Deno's
Result: Module resolution breaks on Bun with cryptic errors

Better approach: if (adapter.capabilities.nativeNpmProtocol) { return specifier; }
Now Bun adapter can accurately report its npm: support level
```

---

## Solution

See **[006.0 - Environment Detection RFC](./006.0-environment-detection-rfc.md)** for the complete solution.

### Immediate Refactoring Pattern

**Before:**
```typescript
import { isDeno } from "#veryfront/platform/compat/runtime.ts";

if (isDeno) {
  return specifier; // Let Deno handle npm:
}
```

**After:**
```typescript
import { runtime } from "#veryfront/platform/adapters/registry.ts";

const adapter = await runtime.get();
if (adapter.capabilities.nativeNpmProtocol) {
  return specifier; // Runtime handles npm: natively
}
```

### New Capability Interface

```typescript
// src/platform/adapters/base.ts - Extended capabilities
interface RuntimeCapabilities {
  // Existing
  typescript: boolean;
  jsx: boolean;
  http2: boolean;
  websocket: boolean;
  workers: boolean;
  fileWatching: boolean;
  shell: boolean;
  kvStore: boolean;
  writableFs: boolean;

  // NEW: Module system capabilities
  nativeHttpImports: boolean;    // Can import https:// URLs directly
  nativeNpmProtocol: boolean;    // Can use npm: specifiers
  nativeTypeScript: boolean;     // Native TS without transpilation
}

// Deno adapter reports:
{ nativeHttpImports: true, nativeNpmProtocol: true, nativeTypeScript: true }

// Node adapter reports:
{ nativeHttpImports: false, nativeNpmProtocol: false, nativeTypeScript: false }

// Bun adapter reports:
{ nativeHttpImports: false, nativeNpmProtocol: true, nativeTypeScript: true }
```

## Files Requiring Refactoring

### Priority 1: Transform Pipeline (High Impact)

| File | Pattern | Change To |
|------|---------|-----------|
| `http-cache.ts` | `isDeno && isReactCoreUrl()` | `adapter.capabilities.nativeHttpImports && isReactCoreUrl()` |
| `http-cache.ts` | `if (isDeno) return specifier` | `if (adapter.capabilities.nativeNpmProtocol) return specifier` |
| `ssr-import-rewriter.ts` | `return isDeno` | `return adapter.capabilities.nativeNpmProtocol` |
| `react-imports.ts` | `isDeno \|\| isNode` | `adapter.capabilities.nativeTypeScript` |

### Priority 2: Module Loading (Security Sensitive)

| File | Pattern | Change To |
|------|---------|-----------|
| `s3-storage.ts` | `isDeno ? import("https://...") : import("pkg")` | `adapter.modules.loadOptionalDep("aws-s3")` |
| `redis/modules.ts` | `if (isDeno) import("deno.land/x/redis")` | `adapter.modules.loadOptionalDep("redis")` |

### Priority 3: Testing (Lower Priority)

| File | Pattern | Acceptable? |
|------|---------|-------------|
| `bdd.ts` | `if (isDeno) import("#std/testing/bdd.ts")` | YES - test framework selection |
| `isolation.ts` | `if (typeof Deno === "undefined") return null` | YES - Deno-specific isolation |
| `deno-compat.ts` | Multiple `isDeno` checks | YES - intentionally Deno-specific |
