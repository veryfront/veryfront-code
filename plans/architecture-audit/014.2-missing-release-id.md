## The Problem

### 1. Cache Key Function Throws

```typescript
// src/cache/keys.ts:110-126
export function computeContentSourceId(
  isLocalDev: boolean,
  environment: "preview" | "production",
  branch: string | null | undefined,
  releaseId: string | null | undefined,
): string {
  if (isLocalDev) {
    return `local-${branch ?? "main"}`;
  }
  if (environment === "production") {
    if (!releaseId) {
      throw new Error("Missing releaseId for production contentSourceId");
      // ^^^^^ THROWS - can crash entire request
    }
    return `release-${releaseId}`;
  }
  return `preview-${branch ?? "main"}`;
}
```

## Why It's Bad

### Failure Scenarios

1. **Domain lookup fails to return releaseId**:
   - API returns project info but `active_release_id` is null
   - Project has no releases (never published)

2. **Proxy doesn't pass header**:
   - Misconfigured proxy drops `x-release-id` header
   - Load balancer strips custom headers

3. **Environment mismatch**:
   - Domain parses as "production" but API returns preview environment
   - Proxy sets `x-environment: production` but no release exists

### Multi-Tenant Impact

When this fails for one project:
- That project gets 502 errors
- Errors logged for that project
- Other projects unaffected (good isolation)

But if the proxy is misconfigured:
- ALL production projects fail
- No graceful degradation
- Users see JSON error response instead of HTML

## Error Response Problem

The current error response is JSON, not HTML:

```json
{
  "error": "Missing releaseId for production request in proxy mode",
  "projectSlug": "example-project",
  "environment": "production"
}
```

Users see raw JSON instead of a proper error page.

## Concrete Fix

### Option A: Graceful Fallback to Preview (Recommended)

```typescript
// src/server/universal-handler/index.ts - Replace hard failure with fallback

// In standalone (non-proxy) mode without releaseId, fallback to preview
// Also in proxy mode if configured to allow fallback
const allowFallbackToPreview =
  !isProxyMode ||
  process.env.ALLOW_PRODUCTION_PREVIEW_FALLBACK === "1";

if (
  resolvedEnvironment === "production" &&
  projectSlug &&
  !releaseId &&
  !isLocalProject &&
  !allowFallbackToPreview
) {
  // Hard failure - return 502
  return new Response(...);
}

if (
  resolvedEnvironment === "production" &&
  projectSlug &&
  !releaseId &&
  !isLocalProject &&
  allowFallbackToPreview
) {
  logger.warn(
    "[universal] Missing releaseId in production, falling back to preview",
    { projectSlug, projectId, host }
  );
  resolvedEnvironment = "preview";
  // Continue with preview content
}
```

### Option B: Return Proper Error Page

```typescript
// Return HTML error page instead of JSON
import { ErrorPages } from "#veryfront/server/error-pages";

if (
  isProxyMode && resolvedEnvironment === "production" &&
  projectSlug && !releaseId && !isLocalProject
) {
  logger.error("[universal] Missing releaseId in proxy mode (production)", {
    projectSlug, projectId, environmentName, host,
  });

  const html = ErrorPages.configurationError({
    title: "Content Not Available",
    message: "This project has not been published yet.",
    details: isLocalDev ? `Missing releaseId for ${projectSlug}` : undefined,
  });

  return new Response(html, {
    status: 503,  // Service Unavailable (temporary)
    headers: {
      "Content-Type": "text/html; charset=utf-8",
      "Retry-After": "60",  // Suggest retry
    },
  });
}
```

### Option C: Proxy Validation Before Forward

```typescript
// proxy/handler.ts - Add warning before forwarding

if (scope === "production" && projectSlug && !releaseId && !isLocalProject) {
  // Log warning but still forward - let renderer decide
  logger.warn("Missing releaseId for production request", {
    projectSlug, host, scope,
    action: "forwarding to renderer with preview fallback",
  });

  // Forward as preview instead
  return {
    ...ctx,
    environment: "preview",
    contentSourceId: `preview-${ctx.parsedDomain.branch ?? "main"}`,
  };
}
```
