# 016.1: Timing Attack in Token Comparison

## Severity: HIGH

## Location

**File:** `src/security/http/auth.ts`
**Lines:** 104-124

```typescript
// VULNERABLE - String comparison leaks timing information
function validateToken(provided: string, expected: string): boolean {
  return provided === expected;  // Timing leak!
}
```

## The Problem

String comparison with `===` returns false at the first mismatched character. An attacker can measure response times to determine how many leading characters match, leaking the token byte-by-byte.

### Attack Vector

```
Token: "secret_abc123"

Attempt 1: "aXXXXXXXXXXXX" → 0.1ms (fails at char 0)
Attempt 2: "sXXXXXXXXXXXX" → 0.2ms (fails at char 1)
Attempt 3: "seXXXXXXXXXXX" → 0.3ms (fails at char 2)
...
After 13 attempts: Full token extracted
```

### Practical Exploit

1. Attacker sends many requests with incrementally correct prefixes
2. Statistical analysis of response times reveals token
3. Even sub-millisecond differences are detectable with enough samples
4. Network jitter mitigated by averaging thousands of requests

## Proposed Fix

Use constant-time comparison that always compares all bytes:

```typescript
import { timingSafeEqual } from "node:crypto";
// Or for Deno:
import { timingSafeEqual } from "https://deno.land/std/crypto/timing_safe_equal.ts";

function validateToken(provided: string, expected: string): boolean {
  // Convert to same-length buffers
  const providedBuf = new TextEncoder().encode(provided);
  const expectedBuf = new TextEncoder().encode(expected);

  // Length check must also be constant-time
  if (providedBuf.length !== expectedBuf.length) {
    // Compare against dummy to maintain constant time
    timingSafeEqual(expectedBuf, expectedBuf);
    return false;
  }

  return timingSafeEqual(providedBuf, expectedBuf);
}
```

## Blast Radius

| Affected | Impact |
|----------|--------|
| API tokens | Full token extraction |
| Session tokens | Session hijacking |
| Internal service auth | Lateral movement |

## Status: ✅ FIXED (eafa78c1)

### Resolution

Created `src/security/utils/constant-time.ts` with XOR-based constant-time comparison that always examines every byte regardless of match position. Applied to all auth comparison paths:

- `src/security/http/auth.ts` — Basic auth (line 109) and bearer auth (line 122-126)
- `src/middleware/builtin/auth.ts` — `basicAuth` (line 22) and `bearerAuth` (line 40)

Unit tests added in `src/security/utils/constant-time.test.ts`.
