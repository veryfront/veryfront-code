# 005.3 - Duplicated Route Parameters Extraction Functions

## Severity: MEDIUM

### Key Differences

| Aspect | App Router | Pages Router |
|--------|-----------|--------------|
| Base directory | `app/` | `pages/` |
| Dynamic segment location | Directory names only | Directory names OR file names |
| File extension handling | N/A (uses `page.tsx`) | Strips extension from file |
| Segment counter | `fsStatCount`, `fsReadDirCount` | `pagesStatCount`, `pagesReadDirCount` |
| Loop style | `for...of` | `for` with index |

## Why It's Bad for Multi-Tenancy

While not directly a multi-tenancy issue, the duplication causes:

1. **Inconsistent behavior**: Subtle differences in how parameters are extracted
2. **Maintenance burden**: Bug fixes must be applied to both functions
3. **Testing complexity**: Need separate tests for each router
4. **Runtime cost**: Caller must determine which function to call

### Subtle Bug Example

In Pages Router, a dynamic route can be a file:
```
pages/blog/[slug].tsx  → pattern: /blog/[slug]
```

In App Router, dynamic routes are always directories:
```
app/blog/[slug]/page.tsx  → pattern: /blog/[slug]
```

But the extraction functions handle this differently, leading to potential edge cases where the same URL produces different `params` objects.

## Failure Scenario

**Scenario: Migration from Pages Router to App Router**

```
Before (Pages Router):
  pages/blog/[slug].tsx
  URL: /blog/hello-world
  Params: { slug: "hello-world" }  ✓

After (App Router - user migrates):
  app/blog/[slug]/page.tsx
  URL: /blog/hello-world

If router detection fails (see 005.1), the wrong extraction function is called:
  extractPagesRouteParams() on App Router structure
  → Looks for pages/blog/[slug].tsx (doesn't exist)
  → Returns null
  → Route params lost!
```

### Unified Extraction Function

```typescript
// Single extraction function that works with any route pattern
function extractRouteParams(
  pattern: string,
  slug: string,
): Record<string, string | string[]> | null {
  const patternParts = pattern.split("/").filter(Boolean);
  const slugParts = slug.split("/").filter(Boolean);
  const params: Record<string, string | string[]> = {};

  let patternIdx = 0;
  let slugIdx = 0;

  while (patternIdx < patternParts.length) {
    const patternPart = patternParts[patternIdx];

    // Catch-all: [...param]
    if (patternPart.startsWith("[...") && patternPart.endsWith("]")) {
      const paramName = patternPart.slice(4, -1);
      params[paramName] = slugParts.slice(slugIdx);
      return params;
    }

    // Optional catch-all: [[...param]]
    if (patternPart.startsWith("[[...") && patternPart.endsWith("]]")) {
      const paramName = patternPart.slice(5, -2);
      params[paramName] = slugIdx < slugParts.length
        ? slugParts.slice(slugIdx)
        : [];
      return params;
    }

    // No more slug parts but pattern continues
    if (slugIdx >= slugParts.length) {
      return null;
    }

    // Dynamic segment: [param]
    if (patternPart.startsWith("[") && patternPart.endsWith("]")) {
      const paramName = patternPart.slice(1, -1);
      params[paramName] = slugParts[slugIdx];
      patternIdx++;
      slugIdx++;
      continue;
    }

    // Static segment must match exactly
    if (patternPart !== slugParts[slugIdx]) {
      return null;
    }

    patternIdx++;
    slugIdx++;
  }

  // Must consume all slug parts (unless catch-all)
  if (slugIdx !== slugParts.length) {
    return null;
  }

  return params;
}
```

### Usage with Unified Registry

```typescript
// Route discovery already knows the pattern
const registry = await getRouteRegistry(projectId, projectDir, adapter, config);
const match = await resolveRoute(slug, registry, adapter);

if (match) {
  // Pattern already extracted during discovery
  // No need for router-specific extraction!
  const params = extractRouteParams(match.route.pattern, slug);
}
```

## Testing

```typescript
// tests/unified-route-params.test.ts
import { assertEquals } from "@std/assert";
import { extractRouteParams } from "../src/routing/unified/params.ts";

Deno.test("extractRouteParams - single dynamic segment", () => {
  const params = extractRouteParams("/blog/[slug]", "/blog/hello-world");
  assertEquals(params, { slug: "hello-world" });
});

Deno.test("extractRouteParams - multiple dynamic segments", () => {
  const params = extractRouteParams("/blog/[year]/[month]/[slug]", "/blog/2024/01/hello");
  assertEquals(params, { year: "2024", month: "01", slug: "hello" });
});

Deno.test("extractRouteParams - catch-all", () => {
  const params = extractRouteParams("/docs/[...path]", "/docs/api/auth/oauth");
  assertEquals(params, { path: ["api", "auth", "oauth"] });
});

Deno.test("extractRouteParams - optional catch-all empty", () => {
  const params = extractRouteParams("/shop/[[...categories]]", "/shop");
  assertEquals(params, { categories: [] });
});

Deno.test("extractRouteParams - optional catch-all with values", () => {
  const params = extractRouteParams("/shop/[[...categories]]", "/shop/electronics/phones");
  assertEquals(params, { categories: ["electronics", "phones"] });
});

Deno.test("extractRouteParams - no match returns null", () => {
  const params = extractRouteParams("/blog/[slug]", "/about");
  assertEquals(params, null);
});

Deno.test("extractRouteParams - works identically for both router conventions", () => {
  // The pattern is already normalized during discovery
  // Whether it came from app/blog/[slug]/page.tsx or pages/blog/[slug].tsx
  // the pattern is /blog/[slug]
  const params = extractRouteParams("/blog/[slug]", "/blog/test-post");
  assertEquals(params, { slug: "test-post" });
});
```
