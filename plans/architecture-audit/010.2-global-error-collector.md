## The Problem

The `ErrorCollector` is a **global singleton** used by MCP (Model Context Protocol) to expose dev server errors to coding agents. Issues:

1. **Single global instance** shared across all projects
2. **DevError interface has no projectId** - can't identify error source
3. **Subscribers see ALL errors** - no project filtering
4. **Clear affects everyone** - one project's reset clears all errors

## Why It's Bad for Multi-Tenancy

### Scenario 1: AI Agent Sees Wrong Project's Errors

```
Timeline:
=========
t0: Developer working on Project A with Claude Code
    -> Claude calls MCP tool vf_get_errors()
    -> Returns errors from ErrorCollector

t1: Meanwhile, Project B has compile errors
    -> ErrorCollector.addCompileError() called
    -> Error stored in global collector

t2: Developer on Project A asks Claude to fix errors
    -> vf_get_errors() returns Project B's errors too!
    -> Claude confused: "I see an error in components/Chart.tsx"
    -> Developer: "I don't have a Chart.tsx file!"

t3: Claude attempts to create Chart.tsx in Project A
    -> Wrong project modified based on wrong error

Impact: AI agent acts on wrong project's errors
```

### Scenario 2: Error Count Metrics Misleading

```
Timeline:
=========
t0: Project A has 5 compile errors
t1: Project B has 3 bundle errors
t2: Admin views metrics via MCP

getErrorCollector().countByType() returns:
{
  compile: 5,    // From Project A
  bundle: 3,     // From Project B
  runtime: 0,
  hmr: 0,
  module: 0
}

Admin thinks ONE project has 8 total errors
Reality: Two different projects with separate issues
```

### Scenario 3: Subscriber Notification Leak

```typescript
// In some MCP tool handler
const collector = getErrorCollector();
collector.subscribe((error) => {
  // This callback receives ALL project errors!
  notifyUser(`New error: ${error.message}`);
});

// When ANY project has error:
// -> User gets notified even if working on different project
// -> Notification contains potentially sensitive error messages from other projects
```

## Failure Scenario

```
MCP Tool Flow:
==============
1. Developer A opens Claude Code for projectA.veryfront.local
2. Claude Code connects via MCP, subscribes to errors
3. Developer B (different machine) starts projectB.veryfront.local on same pod
4. Project B has syntax error in pages/admin.tsx
5. Error logged: getErrorCollector().addCompileError("Unexpected token", "pages/admin.tsx")
6. Developer A's Claude receives notification (via subscriber)
7. Claude: "I noticed a compile error in pages/admin.tsx - would you like me to fix it?"
8. Developer A: "What? I don't have pages/admin.tsx"

Security concern: Error messages may contain:
- File paths revealing project structure
- Variable names / code snippets
- API endpoint names
- Sensitive configuration values in error context
```

```typescript
// Project-scoped collectors
const collectorsByProject = new Map<string, ErrorCollector>();

export function getErrorCollector(projectId?: string): ErrorCollector {
  const key = projectId ?? "__default__";
  let collector = collectorsByProject.get(key);
  if (!collector) {
    collector = new ErrorCollector();
    collectorsByProject.set(key, collector);
  }
  return collector;
}

// Add projectId to DevError interface
export interface DevError {
  id: string;
  type: ErrorType;
  message: string;
  file?: string;
  line?: number;
  column?: number;
  stack?: string;
  timestamp: number;
  context?: Record<string, unknown>;
  projectId?: string;  // NEW: For filtering and debugging
}
```

## Quick Fix

If full implementation is delayed, add projectId filtering to getter:

```typescript
export function getErrorCollector(): ErrorCollector {

  globalCollector ??= new ErrorCollector();

// Add filtered getter for MCP tools
export function getProjectErrors(projectId: string): DevError[] {
  const collector = getErrorCollector();
  return collector.getAll().filter(error => {
    // Filter by file path prefix if projectId is in path
    // This is a heuristic - not perfect but helps
    if (error.file && error.file.includes(projectId)) return true;
    // Filter by context if available
    if (error.context?.projectId === projectId) return true;
    return false;
  });
}
```

## DevError Interface Enhancement

```typescript
export interface DevError {
  id: string;
  type: ErrorType;
  message: string;
  file?: string;
  line?: number;
  column?: number;
  stack?: string;
  timestamp: number;
  context?: Record<string, unknown>;
  // New fields for multi-tenancy
  projectId?: string;
  projectSlug?: string;
}

// Helper to add project context
export function addErrorWithProject(
  collector: ErrorCollector,
  error: Omit<DevError, "id" | "timestamp">,
  projectId: string,
  projectSlug?: string,
): DevError {
  return collector.add({
    ...error,
    context: {
      ...error.context,
      projectId,
      projectSlug,
    },
  });
}
```

## Testing

```typescript
// tests/error-collector-isolation.test.ts
import { assertEquals } from "@std/assert";
import { getErrorCollector, resetAllErrorCollectors } from "../src/cli/mcp/error-collector.ts";

Deno.test("error collectors are isolated by project", () => {
  resetAllErrorCollectors();

  const collectorA = getErrorCollector("project-a");
  const collectorB = getErrorCollector("project-b");

  collectorA.addCompileError("Error in A", "a.tsx");
  collectorB.addCompileError("Error in B", "b.tsx");

  assertEquals(collectorA.count, 1);
  assertEquals(collectorB.count, 1);

  // Errors are isolated
  const errorsA = collectorA.getAll();
  const errorsB = collectorB.getAll();

  assertEquals(errorsA.length, 1);
  assertEquals(errorsA[0].message, "Error in A");

  assertEquals(errorsB.length, 1);
  assertEquals(errorsB[0].message, "Error in B");
});

Deno.test("subscribers only receive project-scoped errors", () => {
  resetAllErrorCollectors();

  const collectorA = getErrorCollector("project-a");
  const collectorB = getErrorCollector("project-b");

  const receivedA: string[] = [];
  const receivedB: string[] = [];

  collectorA.subscribe((error) => receivedA.push(error.message));
  collectorB.subscribe((error) => receivedB.push(error.message));

  collectorA.addCompileError("Error A1");
  collectorB.addCompileError("Error B1");
  collectorA.addCompileError("Error A2");

  assertEquals(receivedA, ["Error A1", "Error A2"]);
  assertEquals(receivedB, ["Error B1"]);
});
```
