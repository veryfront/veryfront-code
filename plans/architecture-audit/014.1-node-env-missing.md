# 014.1: NODE_ENV Missing in Production - Unsafe Default to Development

## Severity: HIGH

## Location

**File:** `/Users/mattboon/Sites/veryfront-renderer/src/server/context/request-context.ts`
**Lines:** 8-11

## The Problem

```typescript
// Lines 8-11 - DEFAULTS TO DEVELOPMENT
export function createEnvConfig(): EnvConfig {
  const env = getEnv("NODE_ENV") ?? getEnv("DENO_ENV") ?? "development";
  return { isLocalDev: env !== "production" };
}
```

If `NODE_ENV` and `DENO_ENV` are both unset, `isLocalDev` becomes `true` even in production pods.

## Why It's Critical

The `isLocalDev` flag controls numerous behaviors:

| Behavior | When `isLocalDev=true` | Impact in Production |
|----------|----------------------|---------------------|
| Caching | Disabled | Every request fetches from API |
| Error Details | Full stack traces | Security information leakage |
| HMR | Enabled | WebSocket connections attempted |
| Dev Dashboard | Enabled | Debug endpoints exposed |
| Cache TTLs | None | No distributed cache usage |
| ETag/304 | Disabled | No conditional responses |

### Multi-Tenant Risk

In a multi-tenant pod serving multiple projects:
- ALL projects would be affected simultaneously
- Each project would experience degraded performance
- All projects would leak error details

## Concrete Fix

### Option A: Fail Fast at Startup (Recommended)

```typescript
// src/server/bootstrap.ts - Add validation
export function validateEnvironmentConfiguration(): void {
  const nodeEnv = Deno.env.get("NODE_ENV");
  const proxyMode = Deno.env.get("PROXY_MODE");

  // In proxy mode, NODE_ENV must be explicitly set
  if (proxyMode === "1" && !nodeEnv) {
    throw new Error(
      "FATAL: NODE_ENV must be set when PROXY_MODE=1. " +
      "Set NODE_ENV=production in your Kubernetes deployment."
    );
  }

  // Warn if unset outside proxy mode
  if (!nodeEnv) {
    logger.warn(
      "[Bootstrap] NODE_ENV is not set, defaulting to 'development'. " +
      "Set NODE_ENV=production for production deployments."
    );
  }
}
```

### Option B: Safe Default in Proxy Mode

```typescript
// src/server/context/request-context.ts
export function createEnvConfig(): EnvConfig {
  const env = getEnv("NODE_ENV") ?? getEnv("DENO_ENV");
  const proxyMode = getEnv("PROXY_MODE") === "1";

  // If no env is set, check if we're in proxy mode
  // Proxy mode implies production deployment
  if (!env && proxyMode) {
    logger.warn(
      "[createEnvConfig] NODE_ENV unset in proxy mode, assuming production"
    );
    return { isLocalDev: false };
  }

  // Default to development only outside proxy mode
  const effectiveEnv = env ?? "development";
  return { isLocalDev: effectiveEnv !== "production" };
}
```
