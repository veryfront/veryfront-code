# 001.0: RFC - Unified Adapter Architecture

## Target State

```
       Business Logic (ONE code path)
                   │
                   ▼
        ┌─────────────────────┐
        │  UnifiedFSAdapter   │
        │  ─────────────────  │
        │  readFile()         │
        │  exists()           │
        │  walkDirectory() ◀── NEW! All adapters implement
        │  getMetadata?()     │
        └─────────────────────┘
                   │
     ┌─────────────┼─────────────┐
     ▼             ▼             ▼
 [Local FS]   [VF API]    [GitHub]
  Adapter      Adapter     Adapter
```

**Key Invariant**: `grep -r "isVeryfrontAdapter" src/ | grep -v adapters/` = 0

---

## TL;DR

**Problem**: 3 adapters (Local FS, Veryfront API, GitHub) with scattered conditionals = 256+ code paths "works locally, breaks in production"

**Solution**: Single interface, single code path. Adapters handle differences internally.

---

## The Problem

```typescript
// This pattern appears 8+ times - each doubles code paths
if (isVeryfrontAdapter(fs)) {
  // One implementation
} else {
  // Different implementation
}
```

**Real bug**: Nested layouts (`app/dashboard/layout.tsx`) work locally, disappear in production.

## Solution

### Unified Interface

**Adapter = interface with primitives. Business logic = functions that use the interface.**

```typescript
// ═══════════════════════════════════════════════════════════════
// The adapter interface - what each adapter implements
// ═══════════════════════════════════════════════════════════════
interface UnifiedFSAdapter {
  // Core FS operations - all adapters MUST implement identically
  readFile(path: string): Promise<string>;
  readFileBytes(path: string): Promise<Uint8Array>;
  writeFile(path: string, content: string): Promise<void>;
  exists(path: string): Promise<boolean>;
  stat(path: string): Promise<FileInfo>;
  readDir(path: string): AsyncIterable<DirEntry>;
  walkDirectory(root: string, filter?: (entry: DirEntry) => boolean): AsyncIterable<string>;
  mkdir(path: string, options?: { recursive?: boolean }): Promise<void>;
  remove(path: string, options?: { recursive?: boolean }): Promise<void>;
  makeTempDir(prefix: string): Promise<string>;
  resolveFile(basePath: string): Promise<string | null>;
  watch?(paths: string | string[]): FileWatcher;  // dev mode only

  // Metadata - return undefined if not available (NOT different behavior)
  getProjectMetadata?(): { updatedAt?: string; id?: string } | undefined;
  getEntityId?(path: string): string | undefined;
  getAllSourceFiles?(): Promise<Array<{ path: string; content?: string }>>;
}

// ═══════════════════════════════════════════════════════════════
// Business logic functions - ONE implementation, works with any adapter
// ═══════════════════════════════════════════════════════════════

// Layout discovery - single code path
async function discoverLayouts(
  adapter: UnifiedFSAdapter,
  pageFilePath: string,
  config: VeryfrontConfig,
): Promise<LayoutItem[]> {
  const rootDir = config.appRouter ? join(projectDir, 'app') : join(projectDir, 'pages');
  const layouts: LayoutItem[] = [];

  for await (const file of adapter.walkDirectory(rootDir, isLayoutFile)) {
    layouts.push(toLayoutItem(file));
  }
  return layouts;
}

// Module evaluation - single code path
async function evaluateModule(
  adapter: UnifiedFSAdapter,
  path: string,
): Promise<unknown> {
  const content = await adapter.readFile(path);
  const tempDir = await adapter.makeTempDir('vf-module-');
  // ... transpile and import
}

// Cache key building - uses metadata from adapter
function buildCacheKey(
  adapter: UnifiedFSAdapter,
  slug: string,
  projectDir: string,
): string {
  const meta = adapter.getProjectMetadata?.();
  return `${meta?.id ?? hashString(projectDir)}:${slug}:${meta?.updatedAt ?? 'draft'}`;
}
```

### Before/After Examples

**1. Layout Collection** (`layout-collector.ts:262`)

```typescript
// BEFORE: Business logic branches on adapter type

const isVeryfrontAPI = !!fs && isExtendedFSAdapter(fs) && fs.isVeryfrontAdapter();

if (isVeryfrontAPI && fs && isExtendedFSAdapter(fs)) {
  return await this.collectAPILayoutConfiguration(fs.getUnderlyingAdapter());
}
return await this.collectFilesystemLayouts(pageFilePath, useAppRouter);

// AFTER: Single function, works with any adapter
return await discoverLayouts(adapter, pageFilePath, config);
```

**6. Layout Discovery** (`layout-collector.ts:262-268`)

```typescript
// BEFORE: API adapter skips directory walk entirely
if (isVeryfrontAPI) {
  // Only checks config.layout and components/layout.{ext}
  // NEVER discovers app/dashboard/layout.tsx, app/settings/layout.tsx, etc.
  return await this.collectAPILayoutConfiguration(fs.getUnderlyingAdapter());
}
// Filesystem adapter walks up from page to root, finding all layouts
return await discoverNestedLayouts(pageFilePath, rootDir, projectDir, adapter);

// AFTER: Single function uses adapter.walkDirectory() - works with any adapter
return await discoverLayouts(adapter, pageFilePath, config);
```

This is the **root cause** of "nested layouts work locally, break in production" - the API adapter never implemented `walkDirectory()`.

## Implementation

| Phase | Change |
|-------|--------|
| 1. Interface | Define `UnifiedFSAdapter`, add `walkDirectory()` to Veryfront API + GitHub adapters |
| 2. Functions | Create `discoverLayouts()`, `evaluateModule()`, `buildCacheKey()` functions |
| 3. Layout fix | Delete `collectAPILayoutConfiguration`, use single `discoverLayouts()` function |
| 4. Remove conditionals | Delete all `isVeryfrontAdapter()` / `isVirtualFilesystem()` from business logic |
| 5. Tests | Cross-adapter integration tests for all critical paths |
