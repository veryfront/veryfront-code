# 001.0: RFC - Unified Adapter Architecture

## TL;DR

**Problem**: 4 adapters with scattered conditionals = 256+ code paths = "works locally, breaks in production"

**Solution**: Single interface, single code path. Adapters handle differences internally.

---

## The Problem

```typescript
// This pattern appears 8+ times - each doubles code paths
if (isVeryfrontAdapter(fs)) {
  // One implementation
} else {
  // Different implementation
}
```

**Real bug**: Nested layouts (`app/dashboard/layout.tsx`) work locally, disappear in production.

## Divergence Points

| Location | Impact |
|----------|--------|
| `layout-collector.ts:262` | Nested layouts break in prod |
| `config/loader.ts:278` | Config evaluation differs |
| `middleware.ts:101` | Error messages differ |
| `getEntityInfo.ts:95` | Cache keys differ |
| `pipeline.ts:650` | Cache invalidation differs |

---

## Solution

### Unified Interface

```typescript
interface UnifiedFSAdapter {
  readFile(path: string): Promise<string>;
  exists(path: string): Promise<boolean>;
  stat(path: string): Promise<FileInfo>;
  readDir(path: string): AsyncIterable<DirEntry>;
  walkDirectory(root: string, filter?: FilterFn): AsyncIterable<string>;
  getProjectMetadata?(): ProjectMetadata | undefined;
}
```

### Before/After

```typescript
// BEFORE: Business logic branches on adapter type
if (isVeryfrontAdapter(fs)) {
  layouts = await collectAPILayouts(fs);
} else {
  layouts = await collectFilesystemLayouts(fs);
}

// AFTER: Single code path
layouts = await collectLayouts(adapter);
```

## Implementation

| Phase | Change |
|-------|--------|
| 1. Interface | Define `UnifiedFSAdapter`, add `walkDirectory()` to API/GitHub adapters |
| 2. Layout fix | Merge `collectAPILayoutConfiguration` + `collectFilesystemLayouts` |
| 3. Remove conditionals | Delete all `isVeryfrontAdapter()` from business logic |
| 4. Tests | Cross-adapter integration tests for all critical paths |
