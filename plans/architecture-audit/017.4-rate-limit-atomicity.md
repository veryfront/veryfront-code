# 017.4: Rate Limit Counter Race

## Severity: MEDIUM

## Location

**File:** `src/middleware/rate-limit.ts`

```typescript
// VULNERABLE - Check-then-act race condition
const counters = new Map<string, number>();

async function checkRateLimit(key: string, limit: number): Promise<boolean> {
  const current = counters.get(key) ?? 0;
  if (current >= limit) {
    return false;  // Rate limited
  }
  counters.set(key, current + 1);  // RACE: Multiple increments
  return true;
}
```

## The Problem

Read-modify-write on the counter is not atomic. Under high concurrency, multiple requests can read the same count and all increment, bypassing the limit.

### Race Window

```
Limit: 100 requests/minute
Current count: 99

t0: Request A reads count → 99
t1: Request B reads count → 99
t2: Request C reads count → 99
t3: Request A: 99 < 100 → allowed, sets 100
t4: Request B: 99 < 100 → allowed, sets 100
t5: Request C: 99 < 100 → allowed, sets 100

Result: 3 requests allowed when only 1 should be
Actual count: 100 (should be 102 if tracked correctly)
```

### At Scale

```
1000 concurrent requests at limit boundary
→ Potentially all 1000 pass through
→ Rate limit effectively bypassed
→ DoS protection fails
```

## Proposed Fix

### Option A: Atomic Increment (Recommended for Single-Node)

```typescript
const counters = new Map<string, { count: number; lock: Promise<void> | null }>();

async function checkRateLimit(key: string, limit: number): Promise<boolean> {
  let entry = counters.get(key);
  if (!entry) {
    entry = { count: 0, lock: null };
    counters.set(key, entry);
  }

  // Wait for any pending operation
  while (entry.lock) {
    await entry.lock;
  }

  // Create our lock
  let unlock: () => void;
  entry.lock = new Promise(resolve => { unlock = resolve; });

  try {
    if (entry.count >= limit) {
      return false;
    }
    entry.count++;
    return true;
  } finally {
    entry.lock = null;
    unlock!();
  }
}
```

### Option B: Redis INCR (Recommended for Multi-Node)

```typescript
import { redis } from "./redis-client";

async function checkRateLimit(key: string, limit: number, windowSeconds: number): Promise<boolean> {
  const redisKey = `ratelimit:${key}`;

  // INCR is atomic in Redis
  const count = await redis.incr(redisKey);

  // Set expiry on first increment
  if (count === 1) {
    await redis.expire(redisKey, windowSeconds);
  }

  return count <= limit;
}
```

### Option C: Token Bucket (Best for Smooth Limiting)

```typescript
interface TokenBucket {
  tokens: number;
  lastRefill: number;
}

const buckets = new Map<string, TokenBucket>();
const REFILL_RATE = 10;  // tokens per second
const BUCKET_SIZE = 100;

function checkRateLimit(key: string): boolean {
  const now = Date.now();
  let bucket = buckets.get(key);

  if (!bucket) {
    bucket = { tokens: BUCKET_SIZE, lastRefill: now };
    buckets.set(key, bucket);
  }

  // Refill tokens based on time passed
  const timePassed = (now - bucket.lastRefill) / 1000;
  bucket.tokens = Math.min(BUCKET_SIZE, bucket.tokens + timePassed * REFILL_RATE);
  bucket.lastRefill = now;

  if (bucket.tokens < 1) {
    return false;
  }

  bucket.tokens--;
  return true;
}
```
