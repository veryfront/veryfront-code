## The Problem

### Global Compiler Instance

```typescript
// Line 31-32
let compiler: Awaited<ReturnType<typeof compile>> | null = null;
let lastStylesheetHash = "";
```

The Tailwind compiler is created based on the stylesheet content hash. If two projects have different Tailwind configurations (different plugins, themes, etc.), the compiler may be configured incorrectly:

```
Timeline:
=========
t0: Project A has globals.css with @plugin "tailwindcss-typography"
    → getCompiler(stylesheetA)
    → compiler = compile(stylesheetA, { loadModule: ... })
    → lastStylesheetHash = hashString(stylesheetA)

t1: Project B has globals.css with @plugin "daisyui"
    → getCompiler(stylesheetB)
    → hashString(stylesheetB) !== lastStylesheetHash
    → Creates NEW compiler (correct behavior!)

BUT if stylesheets happen to hash to same value...

t2: Project C has different globals.css that hashes to same value as A
    → getCompiler(stylesheetC)
    → hashString(stylesheetC) === lastStylesheetHash
    → Returns compiler configured for Project A!
    → Project C's theme variables wrong
```

### Plugin Cache Pollution

```typescript
// Lines 34-35
const pluginCache = new Map<string, unknown>();
const pluginErrors = new Map<string, string>();
```

```
Timeline:
=========
t0: Project A loads plugin "tailwindcss-typography"
    → pluginCache.set("tailwindcss-typography", typographyPlugin)

t1: Project B tries to load same plugin but different version
    → pluginCache.get("tailwindcss-typography")
    → Returns A's version, not B's!

Or:

t0: Project A fails to load plugin "custom-plugin" (doesn't exist)
    → pluginErrors.set("custom-plugin", "Failed to load plugin")

t1: Project B has legitimate "custom-plugin" in node_modules
    → Checks pluginErrors.get("custom-plugin")
    → Returns error from Project A!
    → B's plugin never loads
```

## Concrete Fix

### Option A: Compiler Keyed by Content Hash AND Project

Since Tailwind compilation is deterministic (same input = same output), we can safely share compilers between projects **IF** they have the same stylesheet content. The issue is only when hash collisions occur.

```typescript
// src/html/styles-builder/tailwind-compiler.ts

// Use full stylesheet content as key (no collision risk)
const compilersByStylesheet = new Map<string, {
  compiler: Awaited<ReturnType<typeof compile>>;
  createdAt: number;
}>();

const MAX_CACHED_COMPILERS = 10;

async function getCompiler(stylesheet: string): Promise<Awaited<ReturnType<typeof compile>>> {
  // Use stylesheet content directly as key (no hash collision possible)
  const cached = compilersByStylesheet.get(stylesheet);
  if (cached) {
    return cached.compiler;
  }

  logger.debug("[tailwind] Creating new compiler for stylesheet");

  const tailwindBase = await getTailwindBaseCSS();

  const newCompiler = await compile(stylesheet, {
    base: "/",
    loadStylesheet: (id: string) => {
      if (id === "tailwindcss") {
        return Promise.resolve({ content: tailwindBase, base: "/", path: "/" });
      }
      return Promise.resolve({ content: "", base: "/", path: "/" });
    },
    loadModule: loadPlugin,
  });

  // Evict oldest if at capacity
  if (compilersByStylesheet.size >= MAX_CACHED_COMPILERS) {
    let oldestKey: string | null = null;
    let oldestTime = Infinity;
    for (const [key, entry] of compilersByStylesheet) {
      if (entry.createdAt < oldestTime) {
        oldestTime = entry.createdAt;
        oldestKey = key;
      }
    }
    if (oldestKey) compilersByStylesheet.delete(oldestKey);
  }

  compilersByStylesheet.set(stylesheet, {
    compiler: newCompiler,
    createdAt: Date.now(),
  });

  return newCompiler;
}
```

### Option B: Project-Scoped Plugin Caches

```typescript
// Project-scoped plugin caches to prevent cross-project pollution
const pluginCacheByProject = new Map<string, Map<string, unknown>>();
const pluginErrorsByProject = new Map<string, Map<string, string>>();

function getPluginCache(projectSlug: string): Map<string, unknown> {
  let cache = pluginCacheByProject.get(projectSlug);
  if (!cache) {
    cache = new Map();
    pluginCacheByProject.set(projectSlug, cache);
  }
  return cache;
}

function getPluginErrors(projectSlug: string): Map<string, string> {
  let errors = pluginErrorsByProject.get(projectSlug);
  if (!errors) {
    errors = new Map();
    pluginErrorsByProject.set(projectSlug, errors);
  }
  return errors;
}

async function loadPlugin(id: string, projectSlug: string): Promise<unknown> {
  const cache = getPluginCache(projectSlug);
  const errors = getPluginErrors(projectSlug);

  if (cache.has(id)) {
    const errorMsg = errors.get(id);
    if (errorMsg) throw new Error(errorMsg);
    return cache.get(id);
  }

  // ... load plugin logic

  cache.set(id, plugin);
  return plugin;
}
```

### Option C: Content-Addressed Compiler Cache (Recommended)

For maximum safety and efficiency, use content-addressed caching:

```typescript
// Use hash of (stylesheet + runtime context) as key
function buildCompilerKey(stylesheet: string): string {
  // Include Tailwind version to invalidate on upgrades
  const version = getTailwindVersion();
  return hashString(`${version}:${stylesheet}`);
}

const compilersByKey = new Map<string, Awaited<ReturnType<typeof compile>>>();

async function getCompiler(stylesheet: string): Promise<...> {
  const key = buildCompilerKey(stylesheet);

  const cached = compilersByKey.get(key);
  if (cached) return cached;

  // Create compiler...
  compilersByKey.set(key, compiler);
  return compiler;
}
```

## Local CSS Cache Analysis

The local CSS caches (`localCssCache`, `localCssInputsCache`) use CSS hashes as keys:

```typescript
const localCssCache = new Map<string, string>(); // hash -> css
```

This is **safe** because:
1. Same CSS content = same hash = same result
2. Content-addressed caching doesn't need project scoping

## Migration Priority

Since:
1. Project CSS cache is already properly scoped
2. Local CSS caches are content-addressed (safe)
3. Hash collisions are rare

This issue is **MEDIUM** priority. The main risk is plugin cache pollution.
