# 001.3 - Duplicated isVirtualFilesystem()

## Severity: 🟡 MEDIUM

## The Problem

Same function defined **3 times** with **different behavior**:

| File | Includes GitHub? |
|------|------------------|
| `config/loader.ts` | ✅ YES |
| `middleware.ts` | ❌ NO |
| `wrapper.ts` | ❌ NO |

## Consequence

GitHub adapter projects get:
- Config loaded via **esbuild** (virtual FS path)
- Middleware loaded via **native import** (local FS path)

**Same project, different loading strategies.**

## Fix

Single function in `src/platform/adapters/fs/detection.ts`:

```typescript
const VIRTUAL_FS_ADAPTERS = new Set([
  "VeryfrontFSAdapter",
  "MultiProjectFSAdapter",
  "GitHubFSAdapter",
]);

export function isVirtualFilesystem(adapter: RuntimeAdapter): boolean {
  return VIRTUAL_FS_ADAPTERS.has(adapter.fs?.getAdapterType());
}
```

## Value

| Before | After |
|--------|-------|
| 3 definitions | 1 definition |
| GitHub behaves inconsistently | All adapters behave consistently |
| Typos possible in string checks | Single maintained list |
