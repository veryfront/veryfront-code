# 001.3 - Duplicated isVirtualFilesystem() Functions

## Severity: 🟡 MEDIUM - Code Duplication with Subtle Differences

## The Duplication

The same function is defined **THREE times** in different files, with **subtly different implementations**:

### Definition 1: config/loader.ts:174-181

```typescript
const VIRTUAL_FS_ADAPTERS = new Set([
  "VeryfrontFSAdapter",
  "MultiProjectFSAdapter",
  "GitHubFSAdapter",
]);

function isVirtualFilesystem(adapter: RuntimeAdapter): boolean {
  const fs = adapter?.fs;
  if (!fs || typeof fs !== "object") return false;
  if (!isExtendedFSAdapter(fs)) return false;

  if (fs.isVeryfrontAdapter()) return true;
  return VIRTUAL_FS_ADAPTERS.has(fs.getAdapterType());  // ✅ Checks GitHubFSAdapter
}
```

### Definition 2: middleware.ts:82-86

```typescript
function isVirtualFilesystem(adapter: RuntimeAdapter): boolean {
  const fs = adapter?.fs;
  if (!fs || typeof fs !== "object") return false;
  return isExtendedFSAdapter(fs) && fs.isVeryfrontAdapter();  // 🔴 ONLY checks isVeryfrontAdapter
}
```

### Definition 3: wrapper.ts:67-69 (isVeryfrontAdapter method)

```typescript
isVeryfrontAdapter(): boolean {
  const name = this._fsAdapter.constructor.name;
  return name === "VeryfrontFSAdapter" || name === "MultiProjectFSAdapter";  // 🔴 NO GitHubFSAdapter
}
```

## The Differences

| Check | config/loader.ts | middleware.ts | wrapper.ts |
|-------|-----------------|---------------|------------|
| VeryfrontFSAdapter | ✅ | ✅ | ✅ |
| MultiProjectFSAdapter | ✅ | ✅ | ✅ |
| GitHubFSAdapter | ✅ | ❌ | ❌ |

**Consequence**: GitHub adapter gets different behavior in middleware loading!

## What This Means

### Scenario: GitHub Adapter Project

1. **Config loading** (`config/loader.ts`):
   - `isVirtualFilesystem()` returns `true` for GitHubFSAdapter
   - Uses esbuild transpilation path

2. **Middleware loading** (`middleware.ts`):
   - `isVirtualFilesystem()` returns `false` for GitHubFSAdapter
   - Uses native import path (different behavior!)

**Same project, two different code paths, based on function copy-paste inconsistency.**

## Why Constructor.name Is Fragile

```typescript
isVeryfrontAdapter(): boolean {
  const name = this._fsAdapter.constructor.name;
  return name === "VeryfrontFSAdapter" || name === "MultiProjectFSAdapter";
}
```

**Problems:**
1. **Minification breaks it**: Bundlers rename classes
2. **String comparison**: Typos won't be caught
3. **No interface check**: Relies on class names, not capabilities
4. **Hardcoded list**: Adding new adapters requires code changes in multiple places

## The Proper Solution

### Single Source of Truth

```typescript
// src/platform/adapters/fs/adapter-types.ts
export enum AdapterCapability {
  VirtualFilesystem = 'virtual_fs',
  ProjectMetadata = 'project_metadata',
  SourceFileAccess = 'source_files',
}

export interface CapabilityProvider {
  getCapabilities(): Set<AdapterCapability>;
}

export function hasCapability(
  adapter: RuntimeAdapter,
  capability: AdapterCapability
): boolean {
  const fs = adapter?.fs;
  if (!fs || !isExtendedFSAdapter(fs)) return false;

  const provider = fs.getUnderlyingAdapter();
  if (!('getCapabilities' in provider)) return false;

  return (provider as CapabilityProvider).getCapabilities().has(capability);
}

// Usage:
if (hasCapability(adapter, AdapterCapability.VirtualFilesystem)) {
  // transpile config with esbuild
}
```

### Or: Unified Helper

```typescript
// src/platform/adapters/fs/detection.ts
export function isVirtualFilesystem(adapter: RuntimeAdapter): boolean {
  const fs = adapter?.fs;
  if (!fs || typeof fs !== "object") return false;
  if (!isExtendedFSAdapter(fs)) return false;

  // Single list of virtual filesystem adapters
  const VIRTUAL_FS_ADAPTERS = new Set([
    "VeryfrontFSAdapter",
    "MultiProjectFSAdapter",
    "GitHubFSAdapter",
  ]);

  return VIRTUAL_FS_ADAPTERS.has(fs.getAdapterType());
}
```

**Then import this ONE function everywhere instead of copy-pasting.**

## Status: ✅ RESOLVED (2026-01-28)

### Resolution

Created single canonical `isVirtualFilesystem()` function in `src/platform/adapters/fs/wrapper.ts` that:
- Includes `VeryfrontFSAdapter`, `MultiProjectFSAdapter`, AND `GitHubFSAdapter`
- Takes `FileSystemAdapter` parameter (callers pass `adapter.fs`)
- Replaces both duplicate implementations

**Changes:**
- `src/platform/adapters/fs/wrapper.ts` — Added shared `isVirtualFilesystem()` with `VIRTUAL_FS_ADAPTERS` set
