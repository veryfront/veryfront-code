# 008.3 - Temp File Race Condition in Config Loading

## Severity: MEDIUM

## Location

**File:** `/Users/mattboon/Sites/veryfront-renderer/src/config/loader.ts`
**Lines:** 257-267, 287-298

## The Problem

Config loading creates temp files for dynamic import without deduplication:

```typescript
// src/config/loader.ts:257-267 (Virtual FS path)
const tempDir = await fs.makeTempDir({ prefix: "vf-config-" });
const tempFile = join(tempDir, "config.mjs");

try {
  await fs.writeTextFile(tempFile, js);
  const configModule = await import(`file://${tempFile}?v=${Date.now()}`);
  const userConfig = configModule.default || configModule;
  return validateAndCacheConfig(userConfig, cacheKey);
} finally {
  await fs.remove(tempDir, { recursive: true });  // Cleanup in finally
}
```

```typescript
// src/config/loader.ts:287-298 (Bun path)
const tempDir = await fs.makeTempDir({ prefix: "vf-config-" });
const tempFile = join(tempDir, `config${extension}`);

try {
  const source = await fs.readTextFile(configPath);
  await fs.writeTextFile(tempFile, source);
  const configModule = await import(`file://${tempFile}`);
  const userConfig = configModule.default || configModule;
  return validateAndCacheConfig(userConfig, cacheKey);
} finally {
  await fs.remove(tempDir, { recursive: true });
}
```

## Why It's Bad

### Race Condition 1: Concurrent Requests

When multiple requests for the same project arrive simultaneously:

```
T0: Request A starts loadConfigFromVirtualFS()
T0: Request A creates tempDir-1, writes config.mjs
T1: Request B starts loadConfigFromVirtualFS() (same project)
T1: Request B creates tempDir-2, writes config.mjs
T2: Request A's import() starts
T2: Request B's import() starts
T3: Request A's import() completes
T3: Request A enters finally{} - deletes tempDir-1
T4: Request B's import() still running...
```

Both requests create separate temp dirs, but this wastes resources.

### Race Condition 2: Cleanup Before Import Completes

The `finally` block runs after the `try` block's return, but `import()` is async:

```typescript
try {
  await fs.writeTextFile(tempFile, js);
  const configModule = await import(`file://${tempFile}?v=${Date.now()}`);
  // import() has returned, but module resolution may still be in progress
  return validateAndCacheConfig(userConfig, cacheKey);  // Return happens
} finally {
  await fs.remove(tempDir, { recursive: true });  // Cleanup happens after return
}
```

In some edge cases, the runtime may still be resolving the module when cleanup runs.

### Race Condition 3: Deno Module Cache Interaction

Deno caches modules by URL. The `?v=${Date.now()}` cache buster:

```typescript
const configModule = await import(`file://${tempFile}?v=${Date.now()}`);
```

If two requests have the same `Date.now()` (within 1ms), they could collide:

```
T0.000: Request A import(`file:///tmp/vf-config-xxx/config.mjs?v=1700000000000`)
T0.000: Request B import(`file:///tmp/vf-config-yyy/config.mjs?v=1700000000000`)
```

Both URLs have the same query param, potentially causing confusion in module cache.

## Observed Symptoms

1. Intermittent "file not found" errors during config load under high concurrency
2. Stale config values appearing briefly after changes
3. Increased latency due to duplicate temp file creation

## Multi-Tenancy Impact

In production with high request volume:
- Multiple concurrent requests for same project waste resources
- Temp file proliferation could exhaust disk space
- Intermittent failures affect user experience

## Solution

See **[008.0 - Userland Config RFC](./008.0-userland-config-rfc.md)** for the complete solution.

Key changes:
1. Add in-progress tracking to deduplicate concurrent loads
2. Use content-hash-based temp file names for stability
3. Coordinate cleanup with module cache invalidation
4. Add retry logic for transient file system errors

```typescript
const inProgress = new Map<string, Promise<VeryfrontConfig>>();

async function loadConfig(
  configPath: string,
  adapter: RuntimeAdapter,
): Promise<VeryfrontConfig> {
  const contentHash = await hashContent(await adapter.fs.readFile(configPath));
  const dedupeKey = `${configPath}:${contentHash}`;

  // Deduplicate concurrent loads
  const existing = inProgress.get(dedupeKey);
  if (existing) return existing;

  const promise = doLoadConfig(configPath, contentHash);
  inProgress.set(dedupeKey, promise);

  try {
    return await promise;
  } finally {
    inProgress.delete(dedupeKey);
  }
}
```
