# 018.4: Module Cache No Eviction

## Severity: MEDIUM

## Location

**File:** `src/module-system/`

```typescript
// LEAK - Unbounded cache growth
const moduleCache = new Map<string, Module>();

async function loadModule(path: string): Promise<Module> {
  if (moduleCache.has(path)) {
    return moduleCache.get(path)!;
  }

  const module = await fetchAndCompile(path);
  moduleCache.set(path, module);  // Never evicted
  return module;
}
```

## The Problem

Every unique module path is cached forever. In multi-tenant environments with many projects and versions, cache grows without limit.

### Growth Model

```
Single project: ~200 modules × 50KB = 10MB
Multi-tenant (100 projects): 20,000 modules = 1GB
Over time (version churn): 50,000+ entries = 2.5GB+

Cache entry size:
├── Module source: 10-100KB
├── Compiled code: 10-50KB
├── Metadata: 1-5KB
└── Total: 20-150KB per module
```

### No Natural Cleanup

```
Project A deleted → modules still cached
Version updated → old version still cached
Unused module → never evicted
Memory pressure → still no eviction
```

## Proposed Fix

### Option A: LRU Cache (Recommended)

```typescript
import { LRUCache } from "lru-cache";

const moduleCache = new LRUCache<string, Module>({
  max: 5000,  // Max entries
  maxSize: 500 * 1024 * 1024,  // 500MB max
  sizeCalculation: (module) => module.source.length + module.compiled.length,
  ttl: 1000 * 60 * 60,  // 1 hour TTL
  updateAgeOnGet: true,  // Reset TTL on access
});

async function loadModule(path: string): Promise<Module> {
  const cached = moduleCache.get(path);
  if (cached) return cached;

  const module = await fetchAndCompile(path);
  moduleCache.set(path, module);
  return module;
}
```

### Option B: Time-Based Expiry

```typescript
interface CachedModule {
  module: Module;
  timestamp: number;
}

const moduleCache = new Map<string, CachedModule>();
const TTL = 60 * 60 * 1000; // 1 hour

async function loadModule(path: string): Promise<Module> {
  const cached = moduleCache.get(path);
  if (cached && Date.now() - cached.timestamp < TTL) {
    return cached.module;
  }

  const module = await fetchAndCompile(path);
  moduleCache.set(path, { module, timestamp: Date.now() });
  return module;
}

// Periodic cleanup
setInterval(() => {
  const now = Date.now();
  for (const [key, { timestamp }] of moduleCache) {
    if (now - timestamp > TTL) {
      moduleCache.delete(key);
    }
  }
}, 5 * 60 * 1000);
```

### Option C: Project-Scoped Cache

```typescript
// Cache per project with limits
const projectCaches = new Map<string, LRUCache<string, Module>>();

function getProjectCache(projectId: string): LRUCache<string, Module> {
  if (!projectCaches.has(projectId)) {
    projectCaches.set(projectId, new LRUCache({
      max: 500,  // Per-project limit
      ttl: 1000 * 60 * 30,  // 30 min per project
    }));
  }
  return projectCaches.get(projectId)!;
}
```

## Decision Required

**D012**: Cache eviction strategy:
- A) LRU (least recently used)
- B) TTL (time-based)
- C) LRU + TTL hybrid
- D) Size-based (memory budget)

**D013**: Cache limits:
- Per-cache individual limits
- Global memory budget across all caches

## Blast Radius

| Affected | Impact |
|----------|--------|

| Multi-tenant | Faster growth |
| Version updates | Stale + new both cached |
