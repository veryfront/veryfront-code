# 001.6 - CSS Cache Key Divergence

## Severity: 馃煚 HIGH

## The Problem

CSS cache keys include `projectUpdatedAt` - but it's **only set for Veryfront API adapter**.

```typescript
// pipeline.ts:648
if (fs.isVeryfrontAdapter()) {
  projectUpdatedAt = wrappedAdapter.getProjectData?.()?.updated_at;
}
```

## Cache Keys by Adapter

| Adapter | Cache Key |
|---------|-----------|
| Veryfront API | `proj-123:production:/dashboard:2024-01-15T10:30:00Z` |
| Local FS | `default:preview:/dashboard:draft` |
| GitHub | `default:preview:/dashboard:draft` |

**Local/GitHub projects all share the same cache key!**

## Impact

- Project B serves Project A's CSS in multi-project mode
- CSS changes don't invalidate cache for non-Veryfront adapters
- Cross-project CSS pollution

## Fix

Include project directory hash in cache key:

```typescript
function getPageCssCacheKey(projectId, projectDir, environment, slug, updatedAt) {
  const project = projectId || hashString(projectDir);
  return `${project}:${environment}:${slug}:${updatedAt || Date.now()}`;
}
```

## Value

| Before | After |
|--------|-------|
| Projects share cache keys | Unique keys per project |
| Stale CSS after changes | Proper invalidation |
| Cross-project pollution | Complete isolation |
