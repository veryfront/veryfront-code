## The Problem

A single module may be parsed for imports **4+ times** during the transform pipeline:

### Transform Pipeline Flow

```
Source Code (pages/index.tsx)
       |
       v
[1] MDX Compilation (if .mdx)
    -> Regex line-by-line parse
    -> Rewrites @/ aliases for browser/SSR
       |
       v
[2] esbuild JSX->JS
    -> Full AST parse (unavoidable)
       |
       v
[3] Transform Pipeline - RESOLVE_ALIASES stage
    -> es-module-lexer parse
    -> resolvePathAliases() processes @/ imports
       |
       v
[4] Transform Pipeline - RESOLVE_BARE stage
    -> es-module-lexer parse AGAIN
    -> rewriteBareImports() processes npm packages
       |
       v
[5] SSR Module Loading (if SSR)
    -> Regex parse
    -> applySSRImportRewrites() adds query params
       |
       v
[6] HTTP Module Fetch (if external deps)
    -> Regex parse per fetched module
    -> rewriteEsmPaths() resolves relative paths
       |
       v
Final Code
```

### Performance Impact Measurement

For a typical module with 20 imports:

| Stage | Parsing Method | Approx Time |
|-------|---------------|-------------|
| MDX import rewrite | Regex (line scan) | ~0.5ms |
| resolvePathAliases | es-module-lexer | ~2ms |
| resolveCrossProjectImports | es-module-lexer | ~2ms |
| resolveRelativeImports | es-module-lexer | ~2ms |
| rewriteBareImports | es-module-lexer | ~2ms |
| SSR import rewrite | Regex | ~1ms |
| ESM path rewrite | Regex (8 patterns) | ~2ms |
| **Total** | | **~11.5ms** |

**Single unified parse would take: ~3ms**

**Overhead: ~8.5ms per module (74% waste)**

### Lexer Initialization Overhead

Each `replaceSpecifiers` call checks initialization:

```typescript
// lexer.ts:36-46
export async function initLexer(): Promise<void> {
  if (!initPromise) {
    const anyInit = init as unknown;
    initPromise = typeof anyInit === "function"
      ? (anyInit as () => Promise<void>)()
      : (anyInit as Promise<void>);
  }
  await initPromise;  // Awaited every call even when already initialized
}
```

The WASM initialization is cached, but the `await initPromise` adds microtask overhead per call.

## Why It's Bad for Multi-Tenancy

### 1. Cumulative Latency Under Load

With 50 concurrent requests, each loading 20 modules:

```
Single-tenant latency: 50 * 20 * 11.5ms = 11,500ms total parse time
With unified parsing: 50 * 20 * 3ms = 3,000ms total parse time

Wasted CPU time per batch: 8,500ms
```

### 2. GC Pressure from Intermediate Strings

Each parse pass creates new string:

```typescript
// Pass 1
let code = originalCode;
code = await resolvePathAliases(code, ...);  // New string allocated

// Pass 2
code = await resolveCrossProjectImports(code, ...);  // Another new string

// Pass 3
code = await rewriteBareImports(code, ...);  // Another new string

// Pass 4
code = applySSRImportRewrites(code, ...);  // Another new string
```

For a 50KB module, this creates ~200KB of intermediate strings per module.

### 3. CPU Contention Across Tenants

All projects share CPU time. Wasteful parsing means:
- Longer P99 latencies for all tenants
- More CPU required per request
- Higher infrastructure costs

## Solution

See **[011.0 - Import Rewriting RFC](./011.0-import-rewriting-rfc.md)** for the complete unified solution.

### Key Principle: Parse Once, Transform Once

```typescript
// Single unified transform pass
export class UnifiedImportRewriter {
  async rewrite(code: string, context: RewriteContext): Promise<string> {
    // ONE parse operation
    return replaceSpecifiers(code, (specifier, isDynamic) => {
      // ALL strategies evaluated in single pass
      for (const strategy of this.strategies) {
        if (strategy.matches(specifier, context)) {
          const result = strategy.rewrite(specifier, context);
          if (result !== null) return result;
        }
      }
      return null;
    });
  }
}
```

### Optimized Pipeline

```
Before (4+ passes):
  Parse -> @/ alias -> Parse -> Cross-project -> Parse -> Relative -> Parse -> Bare

After (1 pass):
  Parse -> [All strategies in priority order] -> Done
```

### Batch Strategy Evaluation

```typescript
// All strategies evaluated per specifier in single loop
for (const imp of imports) {
  const specifier = imp.n;

  // Fast path: HTTP URLs (most common in SSR)
  if (specifier.startsWith("http")) continue;

  // Evaluate strategies in priority order
  for (const strategy of sortedStrategies) {
    if (strategy.matches(specifier, context)) {
      const replacement = strategy.rewrite(specifier, context);
      if (replacement) {
        applyReplacement(imp, replacement);
        break; // First match wins
      }
    }
  }
}
```
