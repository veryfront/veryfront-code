Today, the capabilities of modern web applications are incredible: They run on different platforms, in different browsers, and on different screens; they can use storage, connect to media devices, request geolocation, or even run offline. 

It is amazing to realize that the core technologies in building web applications are still JavaScript and HTML—the same as 20 years ago. However, nowadays, developers usually don’t code in raw HTML and JS, but rather implement various tools, extensions, and libraries to decrease complexity, improve performance, and enhance the developer’s experience. JSX is one extension in use today. 

In this article, we will discuss what JSX is, how it works, and how to use it.


### Complexity of DOM Manipulation

From a developer's perspective, rendering modern interfaces with a lot of dynamic content using only regular JavaScript is quite a tricky task. The need to compound it with an application’s business logic adds even more complexity and results in producing tons of poorly readable code. 

Let’s say we need to code a function that creates a card (div) consisting of a title and text in a paragraph. There are multiple ways of doing that in JavaScript, but in general, the code will look like this:


```jsx
function Card({title, text}) {
  const card = document.createElement('div');
  card.classList.add('my-card');

  const h3 = document.createElement('h3');
  h3.textContent = title;

  const p = document.createElement('p');
  p.textContent = text;

  card.appendChild(h3);
  card.appendChild(p);
  return card;
}
```


Quite a lot of code, isn’t it? And we didn’t even add buttons with event handlers or any other logic, which would definitely be the case in real apps.


### Simplifying Things with JSX

But what if we were able to somehow mix HTML and JavaScript while writing UI components and then use the most suitable parts of each to have cleaner code and a more effective development process? 

It would be great if we could achieve the same “card” functionality by writing code like this:


```jsx
function Card({title, text}) {
  return (
    <div className="my-card">
      <h3>{ title }</h3>
      <p>{ text }</p>
    </div>
  )
}
```


Looks a bit weird, but it is very clear and developer-friendly. Fortunately, this is real and the above code is a valid JSX, which is an extension to JavaScript syntax.

With JSX, developers can write HTML constructions inside of JavaScript as well as write JavaScript expressions inside HTML. Look at this short example below:


```jsx
function getInitials(fullName) {
  const [first, last] = fullName.split(" ");
  return first[0] + last[0];
}

function Initials({ fullName }) {
  return <div>{ getInitials(fullName) }</div>;
}
```


Finally, of course, it is also possible to embed components into other components:


```jsx
function ComponentA() { ... }

function ComponentB() { ... }

function Page() {
  return (
    <div>
      <ComponentA />
      <ComponentB />
    </div>
  )
}
```



#### JSX vs. React Clarification

Before diving a bit deeper into technical details, it would be good to explain that React and JSX are not interchangeable technologies; they are not concurrents or alternatives to each other. Both were created by Facebook and designed to be used together, where React is a library for developing user interfaces and JSX is simply a syntax extension. 

You can actually use React via coding in pure JavaScript, but from a practical point of view, it doesn’t make any sense. It is also possible to configure JSX in a way to make it work with other libraries instead of React. However, such scenarios are out of the scope of this article and are, in any event, very uncommon.


### Transpilation

So, we see that JSX is cool and comfortable to work with for developing interfaces, unlike coding in JS or HTML. But do web browsers support this syntax? 

The answer is NO. Browsers still work only with valid HTML, JS, and CSS code, meaning the JSX code needs to first be converted into pure and valid JS. This conversion is called **transpilation**, the process of producing source code from another source code. 

[Babel](https://babeljs.io/) is a transpiler capable of transforming JSX code as well as code written using other syntax extensions into browser-compatible JS. After transpilation of our “Card” component, the output code should look like this:


```jsx
import { jsx as _jsx } from "react/jsx-runtime";
import { jsxs as _jsxs } from "react/jsx-runtime";

function Card({ title, text }) {
  return _jsxs("div", {
    className: "my-card",
    children: [_jsx("h3", {
      children: title
    }),_jsx("p", {
      children: text
    })]
  });
}
```


**Note:** If we were to use an older version of React, then the output would slightly [differ](https://new-jsx-transform.netlify.app/), and the “React.createElement” function would be used instead of “jsx” and “jsxs”.


### Writing JSX

Although it is useful to know how JSX works under the hood, it is even more important to know how to use it properly and effectively to resolve corresponding tasks in development.


#### CSS in JSX

Applying styles is an essential part of UI development, and there are multiple approaches in working with stylings: from quite basic methods like importing CSS or using inline styles to slightly more advanced ways like CSS modules or [styled components](https://styled-components.com/). But for now, understanding the basic ways is more than enough.

The first option is to simply import CSS files the same way we do it for JS files:


```jsx
import './Card.css';
function Card {
  return (<div>...</div>);
}
```


Another way is to not use CSS files at all, but create style objects right in JSX, for example:


```jsx
const cardStyles = {
  container: {
    border: '1px solid black',
    padding: 20
  },
  text: {
    fontSize: 14
  }
}

function Card() {
  return (<div style={cardStyles.container}>...</div>);
}
```


Of course, we could skip creating a separate variable for styles and put the same style object right into the “style” attribute, but this may decrease code readability if there are many CSS properties.


#### Component Libraries

It is worth noting that in real-life projects, it is very common to use component libraries like Chakra UI, Material UI, Ant Design, and others. Libraries allow developers to write less layout and style-related code, avoid reinventing the wheel, and save time. Usually, all the components exposed by the libraries are extremely responsive and work well in all browsers, but you can still opt to customize the style if necessary.

#### Props and Arguments

When we use components in other components, we might want to pass some values inside of that child component because usually components are dynamic and their content depends on some input. This is easy, and you need to know only two things:



* The function representing the component expects the first argument to be an object consisting of all properties to be passed from the parent component.
* Passing properties into components is done in a very similar way to how we set attribute values for HTML elements.

Copy and paste the following code into the Veryfront editor. It includes code for the “Card” and “Container” components where the second one will be rendered in live preview:


```jsx
export function Card({title, text}) {
  const style = {
    border: '1px solid black',
    padding: 5,
    margin: 5
  };
  return (
    <div style={style}>
      <h3>{ title }</h3>
      <p>{ text }</p>
    </div>
  )
}
export const title = 'The title';
export const text = 'Text in the card';

<div>
  <Card title={title} text={text} />
  <Card title="Another title" text="Another text in card" />
</div>
```


Now, we see two blocks with title and text in each. Look at how we use the “Card” component. As you can see, we can pass variables using constructs similar to HTML attributes, but put variables inside curly braces. Of course, you may use quotation marks instead, like for regular HTML attributes, but that way only string literal can be passed.


#### Iterating Iterables in JSX

It is possible to iterate an array directly in the JSX code in order to render lists or tables. Let’s now change our code so the amount of cards rendered depends on the passed data and is not hardcoded in the template. 

You only need to change the code in the following way:


```jsx
const cardsData = [
  {title: 'First card', text: 'Text of the first card'},
  {title: 'Seconds card', text: 'Text of the second card'},
  {title: 'Third card', text: 'Text of the third card'},
]
<div>
  {
    cardsData.map((data) => <Card title={data.title} text={data.text}/>)
  }
</div>
```


Since we want to write a JavaScript expression in the template, put it into the curly braces. Using the default “map” function, above we convert an array of data into an array of Card components, which gets rendered into three cards.

Actually, this expression can be shortened using the [spread](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_syntax#spread_in_object_literals) syntax:


```jsx
cardsData.map((data) => <Card {...data}/>)
```


Finally, since “Card” is actually a function, we can make this expression even shorter and simply instruct JavaScript to execute it for every item in the “cardsData” array:


```jsx
cardsData.map(Card)
```



#### Event Handlers

Adding event handlers for various events like clicks, keyboard input, element focus, etc. is super common in any web application. In vanilla JavaScript/HTML development, it can be done via the “addEventListener” function or by using a corresponding attribute in the HTML, for example, “onclick”. 

Adding event handlers in React looks very similar, but in order to prevent conflicts with default attributes, we need to use camel-case when assigning handlers via attributes. So “onclick” becomes “onClick” in React.  Let’s change our “Card” component’s output (return statement) to this:


```jsx
return (
  <div style={style} onClick={(event) => alert(title)}>
    <h3>{ title }</h3>
    <p>{ text }</p>
  </div>
)
```


Now, if we click on our cards, we will see an alert dialog with the cards’ title inside. Note that we are able to write functions right in the template as well as simply set a corresponding function name into curly braces.
