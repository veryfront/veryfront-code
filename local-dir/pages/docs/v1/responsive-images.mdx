<H1>Responsive images</H1>

### Overview
The [Responsive Image](https://responsive-image.veryfront.com) library provides the `ResponsiveImage` component. It provides a flexible way to display images that are responsive by default, but also allows for extensive customization based on the requirements of your application.

### Usage
import { ResponsiveImage } from "@/shared/ui/ResponsiveImage"

<ResponsiveImage
  src="https://images.unsplash.com/photo-1663334484805-97e6c9e9eb1c"
  alt="King's Cross Station tunnel, London"
  sizes="100vw"
/>

```mdx
import { ResponsiveImage } from "@/shared/ui/ResponsiveImage"

<ResponsiveImage
  src="https://images.unsplash.com/photo-1663334484805-97e6c9e9eb1c"
  alt="King's Cross Station tunnel, London"
  sizes="100vw"
/>
```

### Props
- `src` (required): A string representing the URL of the image.
- `alt` (optional): A string that represents the alternative text to display if the image cannot be loaded or if the user is using a screen reader. This attribute helps improve accessibility and user experience.
- `width` (optional): Width of the image.
- `height` (optional): Height of the image.
- `fill` (optional): Boolean. If true, the image will fill its container.
- `quality` (optional): Quality of the image. It's an integer between 1 and 100.
- `priority` (optional): Boolean. If true, the image will be preloaded.
- `loading` (optional): Loading strategy. If 'lazy', the image will be lazily loaded.
- `placeholder` (optional): One of 'empty', 'blur'. It decides the placeholder used when the image is loading.
- `blurDataURL` (optional): A string with a base64-encoded image.
- `unoptimized` (optional): Boolean. If true, the image will be served as is, without optimization.
- `onLoadingComplete` (optional): Function that is called when the image finishes loading.
- `className` (optional): String of classes applied to the image.
- `style` (optional): CSS style object applied to the image.
- `loader` (optional): Function that generates the URLs of the image. You can define your own loader function to further control how the image URL is formed, especially useful when integrating with image services that have specific URL structures.

Here you can see the default `loader` function which integrates with Veryfront's image proxy:
```javascript
const imageProxyUrl = 'https://images.veryfront.com/imgproxy'

export const defaultLoader: ImageLoader = ({ src, width, quality, params, extension }) => {
  const args = [`q:${quality || 75}`, `w:${width}`]

  if (params?.length) {
    args.push(...params)
  }

  const argsString = args.join('/')
  const base64Src = Buffer.from(src).toString('base64')
  const fileExtension = extension ?? '.webp'

  return `${imageProxyUrl}/${argsString}/${base64Src}${fileExtension}`
}
```
