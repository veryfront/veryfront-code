<H1>Page context</H1>

### What Is Page Context?

You can use to Page Context share data between pages, child components, and wrapper components (App and Layout Components).

You can define `context` values in the frontmatter and consume them anywhere in your app by using the `UsePageContext` provider component:

```mdx
---
context:
  myVariable: Hello, World!
---

import { UsePageContext } from 'https://core.veryfront.com/@components/index'

<UsePageContext>
  {context => (
    <Text>{context.myVariable}</Text>
  )}
</UsePageContext>
```

### How Does Context Work?

The best way to explain how to use `context` is with an example.

Let’s say you want to define a default meta title that you use for all your pages and that you can optionally customize for each Page. To do so, proceed as follows:

Open your App Component in the Live Editor.

  src="https://storage.googleapis.com/veryfront-web/docs-v2/builder-ui-app-component-editor.png"
  alt="Builder UI App Component Editor"
  width="1920"
  height="1080"
  sizes="(max-width: 764px) 100vw, (max-width: 1023px) 764px, (max-width: 1400px) 50vw,  764px"
/>
Define the `metaTitle` `context` variable and give it a default value in the frontmatter of the App Component.
You can now set the default page `title` in the App Component via the `UsePageContext` provider.

```mdx
---
context:
  metaTitle: My Website
---

import { UsePageContext } from 'https://core.veryfront.com/@components/index'
import { Head } from 'https://core.veryfront.com/@components/index'

<UsePageContext>
  {context => (
    <Head>
      <title>{context.metaTitle}</title>
    </Head>
  )}
</UsePageContext>
```

<ResponsiveImage
  src="https://storage.googleapis.com/veryfront-web/docs-v2/page-preview-default-page-title.png"
  alt="Page Preview Default Page Title"
/>
You can also access the value of metaTitle in any other component via the UsePageContext provider.

```jsx
import { UsePageContext } from 'https://core.veryfront.com/@components/index'

<UsePageContext>
  {context => (
    <>{context.metaTitle}</>
  )}
</UsePageContext>
```

Now each Page uses the defined default value. If you want to use a different value, you can overwrite the default `metaTitle` value in the frontmatter of the respective Page:

```markdown
---
context:
  metaTitle: About Us
---
```

### Context precedence rules

- `context` values defined in a Page overwrite those defined in its parent Layout Component and App Component.
- `context` values defined in a Layout Component overwrite those defined in its parent App Component.
- You can also access (but not overwrite) the value of `context` in any other component via the UsePageContext provider.
