<H1>Configure globals.css</H1>

Veryfront is using [PostCSS](https://postcss.org/) and [Tailwind CSS](https://tailwindcss.com) to generate styles which are inlined on the page.

One of the key aspects of this process is the usage of the `globals.css` file. The content of the `globals.css` file is processed by PostCSS.

Use cases for the `globals.css` file are:
- Configure custom CSS (e.g. CSS variables)
- Configure Tailwind CSS / postcss plugins

### Default configuration

In situations where no explicit `globals.css` file is defined, Veryfront employs a default CSS content snippet to ensure a fallback option. The default content includes the following Tailwind CSS directives:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### Advanced Configuration

Here's an example of an advanced `globals.css` file:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  :root {
    --background: 0 0% 100%;
    --foreground: 222.2 47.4% 11.2%;

    --muted: 210 40% 96.1%;
    --muted-foreground: 215.4 16.3% 46.9%;

    --popover: 0 0% 100%;
    --popover-foreground: 222.2 47.4% 11.2%;

    --card: 0 0% 100%;
    --card-foreground: 222.2 47.4% 11.2%;

    --border: 214.3 31.8% 91.4%;
    --input: 214.3 31.8% 91.4%;

    --primary: 222.2 47.4% 11.2%;
    --primary-foreground: 210 40% 98%;

    --secondary: 210 40% 96.1%;
    --secondary-foreground: 222.2 47.4% 11.2%;

    --accent: 210 40% 96.1%;
    --accent-foreground: 222.2 47.4% 11.2%;

    --destructive: 0 100% 50%;
    --destructive-foreground: 210 40% 98%;

    --ring: 215 20.2% 65.1%;

    --radius: 0.5rem;
  }

  .dark {
    --background: 224 71% 4%;
    --foreground: 213 31% 91%;

    --muted: 223 47% 11%;
    --muted-foreground: 215.4 16.3% 56.9%;

    --popover: 224 71% 4%;
    --popover-foreground: 215 20.2% 65.1%;

    --card: 224 71% 4%;
    --card-foreground: 213 31% 91%;

    --border: 216 34% 17%;
    --input: 216 34% 17%;

    --primary: 210 40% 98%;
    --primary-foreground: 222.2 47.4% 1.2%;

    --secondary: 222.2 47.4% 11.2%;
    --secondary-foreground: 210 40% 98%;

    --accent: 216 34% 17%;
    --accent-foreground: 210 40% 98%;

    --destructive: 0 63% 31%;
    --destructive-foreground: 210 40% 98%;

    --ring: 216 34% 17%;

    --radius: 0.5rem;
  }
}

@layer base {
  * {
    @apply border-border;
  }
  body {
    @apply bg-background text-foreground;
    font-feature-settings: "rlig" 1, "calt" 1;
  }
}

/* For Webkit-based browsers (Chrome, Safari and Opera) */
.scrollbar-hide::-webkit-scrollbar {
    display: none;
}

/* For IE, Edge and Firefox */
.scrollbar-hide {
    -ms-overflow-style: none;  /* IE and Edge */
    scrollbar-width: none;  /* Firefox */
}
```

Let's break down what each part of the advanced configuration code is doing:

1. The code begins with three directives `@tailwind base;`, `@tailwind components;`, and `@tailwind utilities;`. These directives are used in conjunction with Tailwind CSS.

2. The `@layer base` rule establishes a new layer called "base" for organizing CSS declarations. Inside this layer, various CSS custom properties (also known as CSS variables) are defined. These custom properties store specific color values and other settings to be used across the application.

3. The `.dark` selector represents a CSS class that applies styles specifically for a dark mode theme. It redefines the custom properties defined in the previous :root selector, providing different color values for the dark mode.

4. Another `@layer base` rule is present, and it targets all elements (*). The `@apply` directive is used to apply the `border-border` utility class from Tailwind CSS to all elements. This adds a border to all elements using the predefined border style defined by Tailwind CSS.

5. The `body` selector applies additional styles to the `<body>` element. It applies a background color (`bg-background`) and text color (`text-foreground`) using the utility classes from Tailwind CSS. Additionally, it sets the `font-feature-settings` property to enable specific font features like "rlig" (required ligatures) and "calt" (contextual alternates).

6. The subsequent CSS rules target the scrollbar of the web page. The `.scrollbar-hide::-webkit-scrollbar` selector is used to hide the scrollbar in Webkit-based browsers (Chrome, Safari, and Opera) by setting `display: none`. The `.scrollbar-hide` selector with `-ms-overflow-style` and `scrollbar-width` properties targets IE, Edge, and Firefox to hide the scrollbar by disabling specific scrollbar styles.

Overall, this CSS code snippet demonstrates the usage of Tailwind CSS directives, custom properties, utility classes, and additional styles to define a consistent visual style and handle specific browser behaviors related to scrollbars.
