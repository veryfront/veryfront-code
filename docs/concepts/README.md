# Veryfront concepts

This folder holds public explanation docs for Veryfront Code. Concepts help
readers understand how parts of the framework relate, why boundaries exist, and
how to reason about trade-offs.

Use `index.md` as the public section overview. Keep `README.md` for
repo-maintainer notes only.

## What belongs here

A page belongs in `docs/concepts/` when it:

- Explains why a Veryfront concept exists or how concepts relate.
- Builds a mental model for choosing, reviewing, or debugging behavior.
- Avoids step-by-step procedures and detailed API catalogs.
- Links to guides for tasks and API reference for exact imports.
